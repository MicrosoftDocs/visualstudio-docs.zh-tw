# <a name="test-area-1-add-toopen-from-source-control"></a>測試區域 1: 從原始碼管理新增/開啟
此原始程式管理外掛程式測試區域包括將解決方案或專案置於原始程式碼管理之下,並從原始程式碼管理中檢索它們。

## <a name="command-menu-access"></a>命令選單存取
 測試案例[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]中使用了以下整合式開發環境選單路徑:

- 對於[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)],從原始碼管理開啟:**檔案**、**開啟**、**專案**/**解決方案**;看看位置[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]。

- 對此原始碼管理外掛程式,從原始碼管理開啟:**檔案**,**原始碼管理**, 從**原始碼管理開啟**。

- 新增來源碼管理:**檔案**,**原始碼管理**,**加入解決方案到原始碼管理檔案**,**原始碼管理**,**將選擇的專案加入到原始碼管理**。

- 捷徑表(專案/解決方案),**將解決方案新增到原始碼管理**。

- 從原始碼管理加入:**檔案**,**原始碼管理**,**從原始碼管理新增專案**。

- 對於[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)],從原始碼進行增加也可從**檔案**、**添加**、**現有專案**;看看位置[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]。

  > [!NOTE]
  > 在此測試中可以使用本地檔案或本地 IIS(Web 伺服器)的路徑。

## <a name="expected-behavior"></a>預期行為

- 對於每個受支援的項目類型,用戶應該能夠"添加到"和"從"源控制。

- 將專案添加到原始碼管理時,將創建相應的\< *ProjectName*>.vspscc 檔案(專案提示檔)。 它包含排除檔案清單和連接資訊。 不要刪除此文件,因為它包含特定於項目的資訊。

- 將解決方案添加到原始碼管理時,將創建相應的\<*解決方案名稱*>.vssscc (三重 S) 檔案。 文字檔包含連接資訊和排除檔案清單,類似於專案提示檔。 此檔是臨時的,僅存在於原始程式碼管理資料庫中。

- 當從原始碼管理開啟解決方案時,\<僅在原始程式碼管理資料庫中存在的*解決方案名稱*>.vsscc (雙 S) 檔在暫存檔中在本地創建。 此檔包含從解決方案連接資料夾到解決方案檔的路徑。 此檔是臨時的,當"從原始程式碼管理打開"操作完成後,本地複本將被刪除。

- 將專案添加到原始程式碼管理後,可以對它執行任何原始程式碼管理操作(簽出、獲取等)。

## <a name="test-cases"></a>測試案例
 以下是「從原始碼管理添加到/開放」測試區域的特定測試案例。

### <a name="case-1a-add-solution-to-source-control"></a>案例 1a:向原始碼管理新增解決方案
 此測試用例側重於向原始程式碼管理添加解決方案。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|將包含用戶端項目的解決方案新增到原始碼管理|1. 創建用戶端專案。<br />2. 將解決方案添加到原始碼管理(**檔案**,**源碼管理**,**向源碼管理添加解決方案**)。|解決方案/專案已添加到原始程式碼管理中。|
|將包含檔案系統或本地 IIS Web 專案的解決方案加入到原始碼管理|1. 創建檔案系統或本地 IIS Web 專案(使用"流覽"按鈕指向專案的位置;路徑確定創建了哪種類型的 Web 專案)。<br />2. 將解決方案添加到原始碼管理(**檔案**,**源碼管理**,**向源碼管理添加解決方案**)。|解決方案/專案已添加到原始程式碼管理中。|
|將包含遠端站台 Web 專案的解決方案加入到原始碼管理|1. 創建遠端網站專案。<br />2. 將解決方案添加到原始碼管理(**檔案**,**源碼管理**,**向源碼管理添加解決方案**)。<br />3. 在「前頁訪問」警告對話框中按下 **「確定**」 。|解決方案已添加到原始碼管理中。<br /><br /> 遠端網站專案不受原始程式碼管理。 (遠端網站項目必須從自己的 IIS 伺服器進行控制。|
|使用**將選定專案添加到原始碼管理**,將單個專案解決方案添加到原始程式碼管理。|1. 創建單個專案解決方案。<br />2. 僅將解決方案添加到原始碼管理作為選擇(**檔案**、**原始碼管理**,**將選擇專案新增到原始碼管理**)。 如果此步驟成功,請繼續執行下一步。<br />3. 將專案加入來源碼管理作為選擇(**檔案**,**源碼管理**,**將選擇專案加入來源代碼管理**)。<br />4. 點擊「**是**」將專案添加到同一位置。<br />5. 點擊 **「簽出****」以進行編輯**對話框。|`Result from Step 2:`<br /><br /> 專案和專案中的所有檔都有簽出的原始程式碼管理指示器,ToolTip 顯示"不受原始程式碼控制"。<br /><br /> `Result from Step 5:`<br /><br /> 專案和解決方案檔位於原始程式碼管理中的同一資料夾中。|
|取消向原始碼管理新增解決方案|1. 創建單個專案解決方案。<br />2. 嘗試將專案和解決方案添加到原始碼管理。 如果此步驟成功,請繼續執行下一步。<br />3. 在原始程式碼管理系統後取消。|`Result from Step 2:`<br /><br /> 設定專案位置來源控制「對話框僅顯示一次。<br /><br /> `Result from Step 3:`<br /><br /> 專案添加已取消,專案/解決方案不受原始程式碼管理,所有添加到原始程式碼管理功能表仍然可用。|

### <a name="case-1b-open-solution-from-source-control"></a>案例1b. 原始程式碼管理的開放解決方案
 此測試用例側重於從原始碼管理打開解決方案。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|從原始碼管理開啟包含用戶端項目的解決方案|1. 創建用戶端專案。<br />2. 將解決方案添加到原始碼管理。<br />3. 關閉解決方案。<br />4. 將解決方案從原始程式碼管理打開到新位置。|從原始碼管理打開的解決方案/專案。|
|從原始碼管理開啟包含本地端或 IIS Web 專案的解決方案|1. 創建本地或 IIS Web 專案。<br />2. 將解決方案添加到原始碼管理。<br />3. 關閉解決方案。<br />4. 將解決方案從原始程式碼管理打開到新位置。|從原始碼管理打開的解決方案/專案。|
|從原始碼管理開啟包含遠端網站項目的解決方案|1. 創建遠端網站專案。<br />2. 將解決方案添加到原始碼管理。 如果此步驟成功,請繼續執行下一步。<br />3. 關閉解決方案。<br />4. 將解決方案從原始程式碼管理打開到新位置。|`Result from Step 2:`<br /><br /> 遠端網站網站不受原始程式碼管理。<br /><br /> `Result from Step 4:`<br /><br /> 解決方案從原始程式碼管理打開。<br /><br /> 遠端網站專案已載入,但不受原始程式碼管理。|

### <a name="case-1c-add-solution-from-source-control"></a>案例 1c:從原始碼管理新增解決方案
 此測試用例側重於從原始碼管理添加解決方案。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|新增至空解決方案 = 單一專案解決方案|1. 創建單個專案解決方案。<br />2. 將解決方案添加到原始碼管理。<br />3. 關閉解決方案。<br />4. 創建第二個空解決方案。<br />5. 從原始碼管理(**檔案**、**原始碼管理**,**從原始碼管理添加專案**)添加以前控制的解決方案。|添加的項目將顯示在**解決方案資源管理器**中並簽入。|
|使用單一專案新增解決方案 + 單個項目|1. 使用單個專案創建解決方案。<br />2. 將解決方案添加到原始碼管理。<br />3. 關閉解決方案。<br />4. 創建第二個空解決方案。<br />5. 從原始碼管理(**檔案**、**原始碼管理**,**從原始碼管理添加專案**)添加以前控制的解決方案。|添加的項目將顯示在**解決方案資源管理器**中並簽入。|
|以選擇新增到解決方案 - 解決方案新增到原始碼管理|1. 使用專案創建解決方案。<br />2. 僅將解決方案添加到原始程式碼管理作為選擇。 如果此步驟成功,請繼續執行下一步。<br />3. 關閉解決方案。<br />4. 創建新解決方案。<br />5. 從原始碼管理(**檔案**、**原始碼管理**,**從原始碼管理添加專案**)添加以前控制的解決方案。|`Result from Step 2:`<br /><br /> 專案不受原始程式碼管理。<br /><br /> `Result from Step 5:`<br /><br /> 如果第一個解決方案具有解決方案項,則無法從原始程式碼管理中添加它們,因此它們不會顯示。<br /><br /> 來自第一個解決方案的項目顯示為不可用。|
