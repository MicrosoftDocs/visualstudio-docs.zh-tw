# <a name="test-area-1-add-toopen-from-source-control"></a>測試區域1：在原始檔控制中加入/開啟
這個原始檔控制外掛程式測試區域涵蓋將方案或專案放在原始檔控制之下，並從原始檔控制中抓取。

## <a name="command-menu-access"></a>命令功能表存取
 下列 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 整合式開發環境功能表路徑用於測試案例中：

- 針對 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]，從原始檔控制開啟 **： [** 檔案]、[**開啟**]、[**專案**/**方案**]、查看 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 位置。

- 針對其他原始檔控制外掛程式，從原始檔控制開啟： [檔案]、[**原始檔控制** **]、[** **從原始檔控制開啟**]。

- 加入至原始檔控制 **：檔案**、**原始檔控制**、**將方案加入至原始**檔控制檔案、**原始檔控制**、**將選取的專案加入至原始檔控制**。

- 快捷方式功能表（[專案/方案]），**將方案加入至原始檔控制**。

- 從原始檔控制加入 **： [** 檔案]、[原始檔**控制**]、[**從原始檔控制加入專案**]。

- 對於 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]，您也可以從 [檔案]、[**新增**]、[**現有專案** **] 中，** 從原始檔控制加入。查看 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 位置。

  > [!NOTE]
  > 本機檔案或本機 IIS （web 伺服器）的路徑可以在這項測試中使用。

## <a name="expected-behavior"></a>預期的行為

- 針對每個支援的專案類型，使用者應該能夠「加入」和「從」原始檔控制中開啟。

- 將專案加入至原始檔控制時，會建立對應的 \<*專案名稱*> manager.csproj.vspscc 檔案（專案提示檔案）。 其中包含排除檔案清單和連接資訊。 請勿刪除此檔案，因為它包含專案的特定資訊。

- 將方案加入至原始檔控制時，會建立對應*的 \< 方案*>. studio.vssscc （三秒）檔案。 此文字檔包含連接資訊和排除檔案清單，類似于專案提示檔案。 這個檔案是暫時性的，而且只存在於原始檔控制資料庫中。

- 從原始檔控制開啟方案時，只有存在於原始檔控制資料庫中的 \< 程式*名稱*> studio.vsscc （double）檔案會在本機建立于暫存檔案中。 此檔案包含從解決方案連接資料夾到方案檔的路徑。 這個檔案是暫時性的，而且本機複本會在「從原始檔控制開啟」作業完成時刪除。

- 將專案加入至原始檔控制之後，您可以對其執行任何原始檔控制動作（簽出、取得等等）。

## <a name="test-cases"></a>測試案例
 以下是 [新增至]/[從原始檔控制開啟] 測試區域的特定測試案例。

### <a name="case-1a-add-solution-to-source-control"></a>案例1a：將方案加入至原始檔控制
 此測試案例著重于將方案加入至原始檔控制。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|將包含用戶端專案的方案加入至原始檔控制|1. 建立用戶端專案。<br />2. 將方案加入至原始檔控制 **（[** 檔案]、[**原始檔控制**]、[**將方案加入至原始檔控制**]）。|方案/專案已加入至原始檔控制。|
|將包含檔案系統或本機 IIS Web 專案的方案加入至原始檔控制|1. 建立檔案系統或本機 IIS Web 專案（使用 [流覽] 按鈕指向專案的位置，路徑會決定要建立的 Web 專案類型）。<br />2. 將方案加入至原始檔控制 **（[** 檔案]、[**原始檔控制**]、[**將方案加入至原始檔控制**]）。|方案/專案已加入至原始檔控制。|
|將包含遠端網站 Web 專案的方案加入至原始檔控制|1. 建立遠端網站 Web 專案。<br />2. 將方案加入至原始檔控制 **（[** 檔案]、[**原始檔控制**]、[**將方案加入至原始檔控制**]）。<br />3. 按一下 [FrontPage Access 警告] 對話方塊中的 **[確定]** 。|方案已加入至原始檔控制。<br /><br /> 遠端網站專案不在原始檔控制之下。 （遠端網站專案必須透過自己的 IIS 伺服器來控制）。|
|使用 [**將選取的專案加入至原始檔控制**]，將單一專案方案加入至原始檔控制。|1. 建立單一專案方案。<br />2. 僅將方案加入至原始檔控制做為選取**範圍（檔案**、原始檔**控制**、**將選取的專案加入至原始檔控制**）。 如果此步驟成功，請繼續進行下一個步驟。<br />3. 將專案新增至原始檔控制做為選取**範圍（檔案**、**原始檔控制**、**將選取的專案加入至原始檔控制**）。<br />4. 按一下 **[是]** ，將專案加入至相同的位置。<br />5. 按一下 [簽出**以編輯**] 對話方塊中的 [**簽出**]。|`Result from Step 2:`<br /><br /> 專案和專案中的所有檔案都有已簽出的原始檔控制指標，而工具提示會顯示「不在原始檔控制之下」。<br /><br /> `Result from Step 5:`<br /><br /> 專案和方案檔位於原始檔控制的相同資料夾中。|
|取消將方案加入至原始檔控制|1. 建立單一專案方案。<br />2. 嘗試將專案和方案加入至原始檔控制。 如果此步驟成功，請繼續進行下一個步驟。<br />3. 在原始檔控制系統中之後取消。|`Result from Step 2:`<br /><br /> [設定專案位置原始檔控制] 對話方塊只會出現一次。<br /><br /> `Result from Step 3:`<br /><br /> 專案新增已取消，專案/方案不在原始檔控制之下，而且所有 [加入至原始檔控制] 功能表仍然可以使用。|

### <a name="case-1b-open-solution-from-source-control"></a>案例1b。 從原始檔控制開啟方案
 此測試案例著重于從原始檔控制開啟方案。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|從原始檔控制開啟包含用戶端專案的方案|1. 建立用戶端專案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 從原始檔控制開啟方案到新的位置。|從原始檔控制開啟的方案/專案。|
|從原始檔控制開啟包含本機或 IIS Web 專案的方案|1. 建立本機或 IIS Web 專案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 從原始檔控制開啟方案到新的位置。|從原始檔控制開啟的方案/專案。|
|從原始檔控制開啟包含遠端網站 Web 專案的方案|1. 建立遠端網站 Web 專案。<br />2. 將方案加入至原始檔控制。 如果此步驟成功，請繼續進行下一個步驟。<br />3. 關閉方案。<br />4. 從原始檔控制開啟方案到新的位置。|`Result from Step 2:`<br /><br /> 遠端網站網站不在原始檔控制之下。<br /><br /> `Result from Step 4:`<br /><br /> 從原始檔控制開啟的方案。<br /><br /> 已載入遠端網站專案，但它不在原始檔控制之下。|

### <a name="case-1c-add-solution-from-source-control"></a>Case 1c：從原始檔控制加入方案
 此測試案例著重于從原始檔控制加入方案。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|新增至空白方案-單一專案方案|1. 建立單一專案方案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 建立第二個空白解決方案。<br />5. 從原始檔控制新增先前控制的方案 **（檔案**、**原始檔控制**、**從原始檔控制加入專案**）。|新增的專案會出現在**方案總管**中，並已簽入。|
|使用單一專案加入方案-單一專案|1. 建立具有單一專案的方案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 建立第二個空白解決方案。<br />5. 從原始檔控制新增先前控制的方案 **（檔案**、**原始檔控制**、**從原始檔控制加入專案**）。|新增的專案會出現在**方案總管**中，並已簽入。|
|加入方案-依選取專案新增至原始檔控制的方案|1. 建立包含專案的方案。<br />2. 僅將方案加入至原始檔控制做為選取專案。 如果此步驟成功，請繼續進行下一個步驟。<br />3. 關閉方案。<br />4. 建立新的方案。<br />5. 從原始檔控制新增先前控制的方案 **（檔案**、**原始檔控制**、**從原始檔控制加入專案**）。|`Result from Step 2:`<br /><br /> 專案不在原始檔控制之下。<br /><br /> `Result from Step 5:`<br /><br /> 如果第一個方案有方案專案，就無法從原始檔控制加入，因此不會出現。<br /><br /> 第一個方案中的專案顯示為無法使用。|
