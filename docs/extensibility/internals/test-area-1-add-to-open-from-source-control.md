# <a name="test-area-1-add-toopen-from-source-control"></a>測試區域 1： 加入開啟從原始檔控制
這個原始檔控制外掛程式測試區域涵蓋放在原始檔控制的專案或解決方案，並擷取原始檔控制。  
  
## <a name="command-menu-access"></a>命令功能表存取  
 下列[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]測試案例中使用整合式的開發環境功能表路徑：  
  
-   如[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]，從原始檔控制開啟：**檔案**，**開啟**，**專案**/**方案**; 尋找中[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]位置。  
  
-   針對其他原始檔控制外掛程式，請從原始檔控制開啟：**檔案**，**原始檔控制**，**從原始檔控制開啟**。  
  
-   加入原始檔控制：**檔案**，**原始檔控制**，**將方案加入至原始檔控制檔案**，**原始檔控制**，**新增選取專案加入原始檔控制**。  
  
-   捷徑功能表 （專案/方案）**將方案加入至原始檔控制**。  
  
-   從原始檔控制加入：**檔案**，**原始檔控制**，**加入的專案，從原始檔控制**。  
  
-   如[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]，加入從原始檔控制項也會提供**檔案**，**新增**，**現有專案**; 尋找在[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]位置。  
  
    > [!NOTE]
    >  這項測試中可用的本機檔案或本機 IIS （web 伺服器） 的路徑。  
  
## <a name="expected-behavior"></a>預期的行為  
  
-   針對每個支援的專案類型，使用者應該能夠使用 「 加入 」 和 「 開啟從 「 原始檔控制。  
  
-   當專案加入原始檔控制，相對應\< *ProjectName*> 建立.vspscc 檔案 （專案提示檔案）。 它包含排除的檔案清單和連接資訊。 請勿刪除此檔案，因為它包含專案的特定資訊。  
  
-   當方案加入原始檔控制，相對應\< *SolutionName*> 建立.vssscc (triple S) 檔案。 文字檔案包含連接資訊和排除的檔案清單，類似於專案提示檔案。 這個檔案是暫時的且只存在於原始檔控制資料庫。  
  
-   從原始檔控制開啟方案時\< *SolutionName*> 只存在於原始檔控制資料庫中，.vsscc (double S) 檔案在本機建立的暫存檔案中。 此檔案包含方案的 [連線] 資料夾的方案檔的路徑。 這個檔案是暫時的 「 從原始檔控制開啟 」 作業完成時，會刪除本機複本。  
  
-   加入至原始檔控制專案之後，您可以執行任何原始檔控制動作 （檢查出 Get，以及等等）。  
  
## <a name="test-cases"></a>測試案例  
 以下是特定的測試案例的 新增 / 開啟從原始檔控制測試區域。  
  
### <a name="case-1a-add-solution-to-source-control"></a>案例 1： 將方案加入至原始檔控制  
 此測試案例著重在將方案加入至原始檔控制。  
  
|動作|測試步驟|若要確認預期的結果|  
|------------|----------------|--------------------------------|  
|加入方案，其中包含用戶端專案加入原始檔控制|1.建立用戶端專案。<br />2.將方案加入原始檔控制 (**檔案**，**原始檔控制**，**將方案加入至原始檔控制**)。|方案/專案已加入至原始檔控制。|  
|加入方案，其中包含檔案系統或本機 IIS Web 專案加入原始檔控制|1.建立檔案系統或本機 IIS Web 專案 (使用瀏覽按鈕，以指向專案的位置; 路徑會決定建立的 Web 專案類型為何)。<br />2.將方案加入原始檔控制 (**檔案**，**原始檔控制**，**將方案加入至原始檔控制**)。|方案/專案已加入至原始檔控制。|  
|加入方案，其中包含遠端站台 Web 專案加入原始檔控制|1.建立遠端站台 Web 專案。<br />2.將方案加入原始檔控制 (**檔案**，**原始檔控制**，**將方案加入至原始檔控制**)。<br />3.按一下**確定**FrontPage 存取 [警告] 對話方塊中。|方案已加入至原始檔控制。<br /><br /> 遠端站台的專案不在原始檔控制中。 （從自己的 IIS 伺服器必須控制遠端站台的專案）。|  
|將單一專案方案加入原始檔控制使用**加入所選專案加入原始檔控制**。|1.建立單一專案的方案。<br />2.將只有方案加入至原始檔控制選項 (**檔案**，**原始檔控制**，**加入所選專案加入原始檔控制**)。 如果這個步驟成功，請繼續下一個步驟。<br />3.為選取的原始檔控制加入專案 (**檔案**，**原始檔控制**，**加入所選專案加入原始檔控制**)。<br />4.按一下**是**將專案加入至相同的位置。<br />5.按一下**簽出**中**簽出以編輯** 對話方塊。|`Result from Step 2:`<br /><br /> 專案和專案中的所有檔案已簽出原始檔控制的指標，並 「 不在原始檔控制 」 會顯示工具提示。<br /><br /> `Result from Step 5:`<br /><br /> 專案和方案檔案位於原始檔控制中的相同資料夾中。|  
|取消將方案加入原始檔控制 5d|1.建立單一專案的方案。<br />2.嘗試加入原始檔控制專案和方案。 如果這個步驟成功，請繼續下一個步驟。<br />3.您是在原始檔控制系統之後取消。|`Result from Step 2:`<br /><br /> 設定專案位置原始檔控制對話方塊只出現一次。<br /><br /> `Result from Step 3:`<br /><br /> 專案取消加入專案/方案不是原始檔控制下，所有加入至原始檔控制功能表仍然可以使用。|  
  
### <a name="case-1b-open-solution-from-source-control"></a>案例 1b。 從原始檔控制開啟方案  
 此測試案例會著重於從原始檔控制開啟方案。  
  
|動作|測試步驟|若要確認預期的結果|

|開啟包含從原始檔控制用戶端專案的方案|1.建立用戶端專案。<br />2.將方案加入原始檔控制。<br />3.關閉方案。<br />4.從原始檔控制開啟方案，到新位置。|從原始檔控制開啟方案/專案。|  
|開啟包含本機或從原始檔控制的 IIS Web 專案的方案|1.建立本機或 IIS Web 專案。<br />2.將方案加入原始檔控制。<br />3.關閉方案。<br />4.從原始檔控制開啟方案，到新位置。|從原始檔控制開啟方案/專案。|  
|開啟包含從原始檔控制的遠端站台 Web 專案的方案|1.建立遠端站台 Web 專案。<br />2.將方案加入原始檔控制。 如果這個步驟成功，請繼續下一個步驟。<br />3.關閉方案。<br />4.從原始檔控制開啟方案，到新位置。|`Result from Step 2:`<br /><br /> 遠端站台的網站不在原始檔控制中。<br /><br /> `Result from Step 4:`<br /><br /> 從原始檔控制開啟方案。<br /><br /> 遠端站台的專案已載入，但是它不在原始檔控制之下。|  
  
### <a name="case-1c-add-solution-from-source-control"></a>案例 1 c： 將方案加入原始檔控制中  
 此測試案例會著重於從原始檔控制新增解決方案。  
  
|動作|測試步驟|若要確認預期的結果|  
|------------|----------------|--------------------------------|  
|加入空白的方案 — 一個單一專案的方案|1.建立單一專案的方案。<br />2.將方案加入原始檔控制。<br />3.關閉方案。<br />4.建立第二個空白方案。<br />5.從原始檔控制加入先前受控制的方案 (**檔案**，**原始檔控制**，**加入的專案，從原始檔控制**)。|加入的專案會出現在**方案總管 中**和簽入。|  
|將加入單一專案的方案，在單一專案|1.建立單一專案的方案。<br />2.將方案加入原始檔控制。<br />3.關閉方案。<br />4.建立第二個空白方案。<br />5.從原始檔控制加入先前受控制的方案 (**檔案**，**原始檔控制**，**加入的專案，從原始檔控制**)。|加入的專案會出現在**方案總管 中**和簽入。|  
|將加入至方案，方案加入原始檔控制選取範圍|1.建立方案與專案。<br />2.只將方案加入至原始檔控制選項。 如果這個步驟成功，請繼續下一個步驟。<br />3.關閉方案。<br />4.建立新的方案。<br />5.從原始檔控制加入先前受控制的方案 (**檔案**，**原始檔控制**，**加入的專案，從原始檔控制**)。|`Result from Step 2:`<br /><br /> 專案不在原始檔控制中。<br /><br /> `Result from Step 5:`<br /><br /> 如果第一個方案具有方案項目，它們無法加入從原始檔控制，因此它們不會出現。<br /><br /> 從第一個方案的專案會顯示為無法使用。|
