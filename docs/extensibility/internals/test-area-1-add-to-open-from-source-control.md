# <a name="test-area-1-add-toopen-from-source-control"></a>測試區域 1：新增至原始檔控制/從原始檔控制開啟
此原始檔控制外掛程式測試區域涵蓋將方案或專案放置在原始檔控制之下，並從原始檔控制中進行抓取。

## <a name="command-menu-access"></a>命令功能表存取
 下列是 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 在測試案例中使用的整合式開發環境功能表路徑：

- 針對 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] ，從原始檔控制開啟： [檔案]、[**開啟**]、[**專案**方案 **]、** / **Solution**查看 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 位置。

- 針對其他原始檔控制外掛程式，請從原始檔控制開啟 **：檔案**、 **原始**檔控制、 **從原始檔控制開啟**。

- 加入至原始檔控制 **：檔案**、 **原始檔控制**、 **將方案加入至原始檔控制**檔案、原始檔 **控制**、 **將選取的專案加入至原始檔控制**。

-  (專案/方案) 的快捷方式功能表， **將方案加入至原始檔控制**。

- 從原始檔控制新增 **： [** 檔案]、[原始檔 **控制**]、[ **從原始檔控制加入專案**]。

- 若是 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] ，也可以從 [檔案]、[**加入**] 和 [**現有專案** **] 中**使用 [從原始檔控制加入]; 在位置中尋找 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] 。

  > [!NOTE]
  > 本機檔案或本機 IIS (web 伺服器的路徑) 可以在此測試中使用。

## <a name="expected-behavior"></a>預期行為

- 對於每個支援的專案類型，使用者都應該能夠「加入」和「從」原始檔控制中開啟。

- 當專案加入至原始檔控制時， \<*ProjectName*> 會 (專案提示檔中建立對應的 manager.csproj.vspscc 檔) 。 它包含排除檔案清單和連接資訊。 請勿刪除此檔案，因為它包含專案的特定資訊。

- 將方案加入至原始檔控制時， \<*SolutionName*> 會建立對應的 >.vssscc (三秒的) 檔案。 文字檔包含連接資訊和排除檔案清單，類似于專案提示檔。 這個檔案是暫時性的，而且只存在於原始檔控制資料庫中。

- 從原始檔控制開啟方案時， \<*SolutionName*> 會在本機的暫存檔案中建立一個 studio.vsscc (雙) 檔案，該檔案只存在於原始檔控制資料庫中。 此檔案包含從方案連接資料夾到方案檔的路徑。 這個檔案是暫時性的，而且當「從原始檔控制開啟」作業完成時，就會刪除本機複本。

- 將專案加入至原始檔控制之後，您可以在其上執行任何原始檔控制動作 (簽出、取得) 等等。

## <a name="test-cases"></a>測試案例
 以下是 [從原始檔控制加入/開啟] 測試區域的特定測試案例。

### <a name="case-1a-add-solution-to-source-control"></a>案例1a：將方案加入至原始檔控制
 此測試案例著重于將方案加入至原始檔控制。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|將包含用戶端專案的方案加入至原始檔控制|1. 建立用戶端專案。<br />2. 將方案加入至原始檔控制 ** (檔案**、 **原始檔控制**、 **將方案加入至原始檔控制**) 。|方案/專案已加入至原始檔控制。|
|將包含檔案系統或本機 IIS Web 專案的方案加入至原始檔控制|1. 建立檔案系統或本機 IIS Web 專案 (使用 [流覽] 按鈕指向專案的位置;路徑會決定) 建立哪種類型的 Web 專案。<br />2. 將方案加入至原始檔控制 ** (檔案**、 **原始檔控制**、 **將方案加入至原始檔控制**) 。|方案/專案已加入至原始檔控制。|
|將包含遠端網站 Web 專案的方案加入至原始檔控制|1. 建立遠端網站 Web 專案。<br />2. 將方案加入至原始檔控制 ** (檔案**、 **原始檔控制**、 **將方案加入至原始檔控制**) 。<br />3. 在 [FrontPage Access 警告] 對話方塊中按一下 **[確定** ]。|方案已新增至原始檔控制。<br /><br /> 遠端網站專案不在原始檔控制之下。  (遠端網站專案必須由自己的 IIS 伺服器控制。 ) |
|使用 [ **將選取的專案加入至原始**檔控制]，將單一專案方案加入至原始檔控制。|1. 建立單一專案方案。<br />2. 將方案加入至原始檔 **控制中做**為選取專案 (檔案、 **原始檔控制**、 **將選取的專案加入至原始檔控制**) 。 如果此步驟成功，請繼續下一個步驟。<br />3. 將專案加入至原始檔控制做為選取 **專案 (檔案**、 **原始檔控制**、 **將選取的專案加入至原始檔控制**) 。<br />4. 按一下 [ **是]** ，將專案加入至相同的位置。<br />5. 按一下 [簽出**進行編輯**] 對話方塊中的 [**簽**出]。|`Result from Step 2:`<br /><br /> 專案和專案中的所有檔案都有已簽出的原始檔控制指標，而工具提示會顯示「不在原始檔控制之下」。<br /><br /> `Result from Step 5:`<br /><br /> 專案和方案檔位於原始檔控制的相同資料夾中。|
|取消將方案加入至原始檔控制|1. 建立單一專案方案。<br />2. 嘗試將專案和方案加入至原始檔控制。 如果此步驟成功，請繼續下一個步驟。<br />3. 在原始檔控制系統中之後取消。|`Result from Step 2:`<br /><br /> [設定專案位置原始檔控制] 對話方塊只會出現一次。<br /><br /> `Result from Step 3:`<br /><br /> 已取消專案加入，專案/方案不在原始檔控制下，而且所有新增至原始檔控制功能表仍可供使用。|

### <a name="case-1b-open-solution-from-source-control"></a>案例1b。 從原始檔控制開啟方案
 此測試案例著重于從原始檔控制開啟方案。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|從原始檔控制開啟包含用戶端專案的方案|1. 建立用戶端專案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 將方案從原始檔控制開啟至新位置。|從原始檔控制開啟的方案/專案。|
|從原始檔控制開啟包含本機或 IIS Web 專案的方案|1. 建立本機或 IIS Web 專案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 將方案從原始檔控制開啟至新位置。|從原始檔控制開啟的方案/專案。|
|從原始檔控制開啟包含遠端網站 Web 專案的方案|1. 建立遠端網站 Web 專案。<br />2. 將方案加入至原始檔控制。 如果此步驟成功，請繼續下一個步驟。<br />3. 關閉方案。<br />4. 將方案從原始檔控制開啟至新位置。|`Result from Step 2:`<br /><br /> 遠端網站網頁不在原始檔控制之下。<br /><br /> `Result from Step 4:`<br /><br /> 從原始檔控制開啟的方案。<br /><br /> 遠端網站專案已載入，但不在原始檔控制之下。|

### <a name="case-1c-add-solution-from-source-control"></a>案例1c：從原始檔控制加入方案
 此測試案例著重于從原始檔控制加入方案。

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|新增至空白方案-單一專案方案|1. 建立單一專案方案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 建立第二個空白的方案。<br />5. 從原始檔控制加入先前控制的方案 ** (檔案**、 **原始**檔控制、 **從原始檔控制加入專案**) 。|新增的專案會出現在 **方案總管** 中，並簽入。|
|使用單一專案加入方案-單一專案|1. 建立具有單一專案的方案。<br />2. 將方案加入至原始檔控制。<br />3. 關閉方案。<br />4. 建立第二個空白的方案。<br />5. 從原始檔控制加入先前控制的方案 ** (檔案**、 **原始**檔控制、 **從原始檔控制加入專案**) 。|新增的專案會出現在 **方案總管** 中，並簽入。|
|加入至方案-依選取專案加入至原始檔控制的方案|1. 建立具有專案的方案。<br />2. 將方案加入至原始檔控制做為選取專案。 如果此步驟成功，請繼續下一個步驟。<br />3. 關閉方案。<br />4. 建立新的方案。<br />5. 從原始檔控制加入先前控制的方案 ** (檔案**、 **原始**檔控制、 **從原始檔控制加入專案**) 。|`Result from Step 2:`<br /><br /> 專案不在原始檔控制之下。<br /><br /> `Result from Step 5:`<br /><br /> 如果第一個方案有方案專案，則無法從原始檔控制加入，因此不會出現這些專案。<br /><br /> 第一個解決方案的專案顯示為無法使用。|
