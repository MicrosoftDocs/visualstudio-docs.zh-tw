# <a name="test-area-2-get-from-source-control"></a>測試區域 2︰從原始檔控制中取得
此測試區域涵蓋用於通過 Get 命令從版本存儲檢索項的測試用例。 這些測試用例可以應用於本地和 Web 專案。

## <a name="command-menu-access"></a>命令選單存取
 測試用例[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]中使用了以下集成開發環境功能表路徑。

##### <a name="get-latest-version"></a>取得最新版本:

- **檔案**,**原始碼管理**,**取得最新版本**.

- **檔案**,**取得最新版本**.

- 捷徑選單,**取得最新版本**。

- 取得:**檔案**,**原始碼管理**,**取得**.

## <a name="expected-behavior"></a>預期行為

##### <a name="get-latest-version"></a>取得最新版本:
 從版本存儲對專案的最新版本執行靜默(無 UI)檢索。

##### <a name="get"></a>取得：
 顯示 **「獲取」** 對話框,允許使用者對將檢索的檔集進行更改,並修改影響檔檢索方式的選項。

## <a name="test-cases"></a>測試案例

|動作|測試步驟|要驗證的預期結果|
|------------|----------------|--------------------------------|
|取得本地不存在的檔案的最新版本|1. 創建專案。<br />2. 向專案添加專案。<br />3. 將專案置於原始程式碼管理之下。<br />4. 刪除項目的本地複本。<br />5. 取得專案的最新版本(快捷選單,**取得最新版本**)。|專案檔在本地檢索。|
|取得本地端不存在的檔案|1. 創建專案。<br />2. 向專案添加專案。<br />3. 將專案置於原始程式碼管理之下。<br />4. 刪除項目的本地複本。<br />5. 取得項目(**檔案**,**原始碼管理**,**取得**\<專案>)。|專案檔在本地檢索。|
|取得已單獨簽出並在本地修改的檔案|1. 創建專案。<br />2. 向專案添加專案。<br />3. 將專案置於原始程式碼管理之下。<br />4. 專門查看專案專案。<br />5. 修改本地複本。<br />6. 取得專案的最新版本(**檔案**,獲取專案>**的**\<最新版本)。 如果此步驟成功,請繼續執行下一步。<br />7. 按一下警告對話方塊中的 **「替換**」 按鈕。|**步驟 6 的最終結果**`:`<br /><br /> 警告對話方塊指示檔已簽出。<br /><br /> **步驟 7 的最終結果:**<br /><br /> 修改後的本地檔將由版本存儲中的原始版本替換。<br /><br /> 檔是讀/寫。|
|取得與取代在本地籤出、共用及修改的檔案|1. 創建新專案。<br />2. 向專案添加專案。<br />3. 將專案置於原始程式碼管理之下。<br />4. 簽出專案項目作為共用。<br />5. 修改本地複本。<br />6. 取得專案的最新版本(**檔案**,獲取專案>**的**\<最新版本)。 如果此步驟成功,請繼續執行下一步。<br />7. 在警告對話方塊中按下 **「替換**」。。|**步驟 6 的結果:**<br /><br /> 警告對話方塊指示檔已簽出。<br /><br /> **步驟 7 的結果:**<br /><br /> 修改後的本地檔將由版本存儲中的原始版本替換。<br /><br /> 檔是讀/寫。|
|取得本地存在的檔案,與版本儲存中的最新版本相同|1. 創建新專案。<br />2. 向專案添加專案。<br />3. 將專案置於原始程式碼管理之下。<br />4. 取得項目(**檔案**,**原始碼管理**,**取得**\<專案>)。|本地檔保持不變。|
|透過專案取得解決方案|1. 使用一個專案創建解決方案。<br />2. 將解決方案置於原始程式碼管理之下。<br />3. 在本地刪除所有項目檔。<br />4. 取得解決方案(**檔案**,**源碼管理**,**取得**)。|所有已刪除的檔都將在本地還原。|
