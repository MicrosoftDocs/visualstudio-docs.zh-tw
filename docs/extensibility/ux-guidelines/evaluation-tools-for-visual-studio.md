# <a name="evaluation-tools-for-visual-studio"></a>適用於 Visual Studio 的評估工具
## <a name="craftsmanship-checklist-for-visual-studio"></a>視覺工作室的工藝清單
 使用此檢查表可評估使用者體驗品質,瞭解視覺和交互詳細資訊。

### <a name="overview"></a>概觀

- 驗證所有命令是否導致反饋,告知使用者其命令已執行。

- 驗證所有 UI 元素和控制檔在所有主題和高對比度模式下是否可見。

- 驗證非活動選擇和活動選擇在標準和高對比度模式下始終有區別。

- 驗證焦點是否始終可見且明顯。

### <a name="performance"></a>效能

- 驗證如果命令需要超過一秒才能完成,則顯示某種"忙"指示器。

- 驗證如果命令需要超過 10 秒才能完成,將顯示一個顯式進度欄,確定(首選)或不確定。

### <a name="ui-text"></a>UI 文字

- 驗證所有標籤是否都是句子或標題大小寫,並且沒有文本是完全小寫。

    ||正確|錯誤|
    |-|-------------|---------------|
    |**命令文字(所有)**|判刑案例:<br /><br /> **目錄名稱:**|目錄名稱:|
    |**按鈕文字(用戶端)**|標題案例:<br /><br /> **【 設定為預設值 】**|設定為預設值|
    |**按鈕文字(線上)**|判刑案例:<br /><br /> **【 設定為預設值 】**||

- 驗證除組標題和按鈕之外的所有標籤都以冒號結尾,並在與其配對的控制項之前結束。

- 驗證啟動 UI 以在省略號擷取使用者輸入端的按鈕、命令和指令連結 **[...]**.

  範例：

  - 對話框上的 **[高級...]** 按鈕。

  - "工具"選單(**工具>選項**)下的命令選項不應得到省略號,因為啟動對話方塊本身是命令的意圖。

- 驗證 UI 是否不包含縮寫,但行業標準術語除外。 例如,HTML 和 TCP/IP 都不需要拼寫出來,儘管 OOM(記憶體不足)和 PII(個人身份資訊)應該。

### <a name="keyboard-access"></a>存取鍵盤

- 驗證是否有使用鍵盤完成每個任務的方法。 通常,這是通過每個控件的鍵盤訪問來完成的,但對於某些高度可視的區域,諸如訪問代碼視圖等解決方法是可以接受的。

- 驗證是否可以按邏輯順序(從左至右和從上到下)對控件進行選項卡式檢查。 雖然這對於大多數控件都是最佳做法,但並非所有控制件都需要此方法。 例如,驗證單選按鈕控制項是否位於具有單個製表位的組中。

- 驗證所有控制項是否具有標籤,並且每個分頁都有助記符(例外情況包括一些可能遵循選項卡中標記控制項的非標記控制項)。

- 驗證不存在助記符衝突。

### <a name="fonts"></a>字型

- 驗證所有字體(面部、大小、顏色)是否一致使用並維護層次結構。

- 驗證所有 UI 元素是否使用環境字型服務。 (請參閱[可視化工作室的字型和格式](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md))

     要檢查服務是否正在使用,請轉到**工具>選項>字型和顏色**。 在「設定」下拉下,選擇「環境字體」並將字體更改為風格上有所不同的內容(如哈靈頓或漫畫聖人),並將大小設置為 12 磅。 然後按一下 [確定]。 您可能需要重新啟動 IDE,但大多數 UI 將立即更改。 即使在重新啟動時也不會拾取字體更改的區域不使用環境字型。

- 驗證作為服務派生的字型(例如,粗體或放大文本)在環境字體大小更改時保留其大小和格式與"正常"文本相關。

- 驗證由於字體放大,沒有剪切錯誤。 剪切的字體可能是固定高度控制項或固定高度容器的結果。

### <a name="dialogs"></a>對話方塊

- 驗證對話框標題是否與啟動對話框的標題的命令相同。

- 驗證所有標準控制項是否與作業系統一致:背景顏色是標準控制項,任何控制項都不應具有特殊的重新範本化樣式,使其看起來不同於標準控制項。

- 驗證表單中的邊距應為 12 像素,並且應顯示均勻且一致。

- 驗證對話框是否顯示在 IDE 外殼或生成對話框的視窗中居中。

- 有用時,對話方塊應可調整大小。 對於可調整大小的對話框,請驗證調整大小時,適當的控制項必須調整大小,而對話方塊的其他部分保持不變。

- 驗證可調整大小的對話方塊是否保留任何使用者調整的大小(大小、位置、對話方塊控制項的擴展等)。

- 驗證標題列中沒有圖示。

- 驗證標題列中沒有「最小化」和「最大化」按鈕。

#### <a name="dialog-operation-buttons-vs-client-only"></a>對話框操作按鈕(只有限制 VS 用戶端)

- 檢查操作按鍵是否按此順序排列:**確定**, 取消、 套用 、**取消**、**套用**。

- 驗證 **「確定****」和「取消」** 按鈕是否為標準大小:75x23 圖元。

- 驗證 **"確定****"和"取消"** 按鈕的大小是否相等,而不考慮字串長度。

- 如果操作按鈕上的標籤要求按鈕比標準寬,請驗證相應的 **「取消」** 按鈕的大小是否相同。

- 驗證按鈕和相關控制項之間是否有6 像素填充。

- 驗證 **「確定**」和 **「取消」** 按鈕沒有助記符(由帶下劃線字母定義的存取鍵)。

- 驗證一個按鈕(通常**為"確定**")預設具有焦點。

- 驗證**Esc**是否取消對話框

- 如果焦點不在處理 Enter 的控制項中,請驗證**Enter**是否執行預設按鈕。

- 驗證 **「確定**」和 **「取消」** 按鈕是否位於對話框的右下角。 在極少數例外情況中,它們垂直堆疊在右上角是可以接受的。

- 僅當對話方塊中的其他按鈕處於水準對齊時,才驗證垂直配置是否使用。

### <a name="control-standards"></a>控制標準

#### <a name="general"></a>一般

- 驗證是否盡可能有良好的默認值,以加快使用者交互,並引導用戶獲得安全或共同的結果。

- 驗證標準控件是否以相同的方式操作,以便用戶根據以前的體驗知道將會發生什麼。

#### <a name="label-controls"></a>標籤控制項

- 驗證每個控制項是否具有標籤,並且每個標籤在視覺上與其控制項配對(通常在 4-6 像素範圍內),並且比其他控制項更接近其相應的控制項。

- 如果定位在上方並水準居中,則驗證標註是否與控制項左邊緣對齊左,以便標籤的基線與輸入文本的基線對齊(如果位於左側)。

- 驗證如果多個堆疊標籤和輸入控制項位於控制的左側,則標籤向左刷新,與對話框邊緣的距離相等,切勿向右刷新,並且與控制項的距離相等。 對應均勻分佈,除非它們需要額外的間距來指示分組。

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>輸入控制器 (文字框與組合框)

- 驗證使用預設環境字型時,文本框、組合框和按鈕的顯示高度均為 23 圖元。

- 如果使用提示文本,請驗證顏色是否設置為`Environment.ControlEditHintText`使用顏色服務。

- 如果該欄位是必須識別為該欄位的必填欄位,請驗證:

  - 背景設定為`Environment.ControlEditRequiredBackground`,前景設定為`Environment.ControlEditRequiredHintText`

  - 控制項中有顯示為 **「必需>」\<** 的提示文字

#### <a name="button-controls"></a>按鈕控制項

- 驗證按鈕的最小大小是否為 75x23 圖元,除非容納較長的文本。

- 驗證按鈕的左右邊距是否為 3-5 圖元,以及內容填充。

- 可以使用一個小方形按鈕,其中只有省略號 **[...] 而不是** **[Browse...]** 按鈕(或類似功能)。 如果使用,請驗證按鈕的大小是否為 23x23。

- 如果對話框中有多個 **[Browse...]** 按鈕,則驗證縮短的版本(僅限省略號 **[...]**) 是否用於所有內容。

- 驗證省略**號 [...]** 按鈕沒有助記符。 當焦點位於其旁邊的輸入控件上時,一個選項卡應將焦點移動到橢圓按鈕。

- 驗證啟動捕獲更多使用者輸入的輔助 UI 的按鈕、命令和命令連結必須以省略號 **[...]** 結尾。

#### <a name="hyperlinks"></a>超連結

- 驗證超連結控制件在啟動時是否從不閃爍紅色。 這是一個指示,表明顏色服務未使用

- 認證使用的 VS 顏色是否:

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 驗證超連結是否顯示為藍色,除非嵌入段落,否則沒有下劃線。

#### <a name="check-boxes"></a>核取方塊

- 如果複選框具有多行文本,請驗證該框是否與第一行文本對齊,而不是在所有行中垂直居中。

- 驗證複選框始終指示二進位選擇,並且不導航使用者或打開新視窗或頁面。

- 如果複選框顯示與輸入控制項相關的選項,請驗證該選項位於該控制件的左側和非常接近的位置,以指示其關係。

- 驗證複選框**是否從未**用作啟用對話框或頁面的全部內容的方法。

#### <a name="group-boxes"></a>群組盒

- 驗證對話框中不包含包含對話框全部內容的單個組框。

- 驗證每個組框中至少有兩個控件。

- 對話框中很少應該有兩個以上組框。

- 驗證沒有嵌套組框。

### <a name="icons"></a>圖示

- 驗證圖示在深色主題中是否正確反轉。

- 驗證所有圖示是否基於核心概念。

- 驗證每個圖示是否不同、易於識別且不包含兩個以上概念(沒有狀態修改器/語言)。

- 驗證基本圖示是否顯示在空間內的中心。

- 驗證所有圖示在高對比度模式下是否清晰。

- 驗證使用的任何顏色是否符合顏色使用標準。

- 驗證圖示周圍沒有光暈(邊框)。 (如果存在,光暈應與相鄰 UI 的背景顏色匹配)。

### <a name="touch-enabled-ui"></a>開啟觸控的 UI

- 驗證互動式控制項是否足夠大,易於觸控 - 最小**23x23 像素**大小

- 驗證最常用的控制項大小是否至少**為 40x40 像素**。

- 驗證互動式控制器之間至少有**5 像素的間距**
