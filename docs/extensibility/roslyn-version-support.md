# <a name="net-compiler-platform-package-version-reference"></a>.NET 編譯器平臺套件版本引用

下表顯示了 Visual Studio 的不同版本支援哪些[.NET 編譯器平臺 (Roslyn) 套件](https://www.nuget.org/packages/Microsoft.Net.Compilers/)版本。

例如,為了確保自定義分析器適用於 Visual Studio 2017 的所有版本,它應該針對 Microsoft.Net.編譯器的版本 2.0。

| 羅斯林包版本 | 受支援的最小視覺工作室版本 |
| - | - |
| 3.x | Visual Studio 2019 |
| 2.10.0 | 視覺工作室 2017 版本 15.9 |
| 2.9.0 | Visual Studio 2017 15.8 版 |
| 2.8.2 | Visual Studio 2017 15.7 版 |
| 2.7.0 | Visual Studio 2017 15.6 版 |
| 2.6.1 | Visual Studio 2017 15.5 版 |
| 2.4.0 | Visual Studio 2017 15.4 版 |
| 2.3.2 | Visual Studio 2017 15.3 版 |
| 2.2.0 | Visual Studio 2017 15.2 版 |
| 2.1.0 | Visual Studio 2017 15.1 版 |
| 2.0.0 | 視覺工作室 2017 RTM |
| 1.3.2 | 視覺工作室 2015 更新 3 |
| 1.2.2 | 視覺工作室 2015 更新 2 |
| 1.1.1 | 視覺化工作室 2015 更新 1 |
| 1.0.1 | 視覺工作室 2015 RTM |

> [!TIP]
> 對於受支援的最低視覺工作室版本為 Visual Studio 2017 版本的 Roslyn 軟體包,也支援 Visual Studio 2019 的所有版本,因為它們來得較晚。
