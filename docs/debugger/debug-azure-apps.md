# <a name="debug-azure-services-in-visual-studio"></a>在 Visual Studio 中將 Azure 服務進行 Debug 錯

您可以在不同的案例中使用 Visual Studio 來將 Azure 服務進行 debug 錯：

- 若要對裝載于 Azure App Service 中的生產應用程式進行偵錯工具，請參閱 [使用快照偵錯工具來偵測即時 ASP.NET 應用程式](../debugger/debug-live-azure-applications.md)。

- 若要對裝載于 Azure 虛擬機器或 Azure 虛擬機器擴展集的生產環境應用程式進行偵錯工具，請參閱 [使用快照偵錯工具來即時偵測 azure 虛擬機器和 azure 虛擬機器擴展集](../debugger/debug-live-azure-virtual-machines.md)。

- 若要對裝載于 Azure Kubernetes Service 中的生產應用程式進行偵錯工具，請參閱 [使用快照偵錯工具來即時 ASP.NET Azure Kubernetes Services](../debugger/debug-live-azure-kubernetes.md)。

- 若要使用 Application Insights 和裝載于 Azure App Service 或 Service Fabric 中的生產應用程式，請參閱 [.net 應用程式例外狀況的偵錯工具快照](/azure/application-insights/app-insights-snapshot-debugger)。

- 若要在 IIS (Azure App Service 或 Azure VM) 上進行遠端偵錯 ASP.NET，請參閱 [azure 上的遠端偵錯程式 ASP.NET](remote-debugging-azure.md)。

- 若要在 Azure Service Fabric 上遠端偵錯 ASP.NET，請參閱 [debug a remote Service Fabric 應用程式](/azure/service-fabric/service-fabric-debugging-your-application#debug-a-remote-service-fabric-application)
