# <a name="debugging-web-applications"></a>偵錯 Web 應用程式

本文提供的連結可協助您偵測不同類型的 web 應用程式。

- 對 ASP.NET 應用程式進行偵錯

  如需啟用 [!INCLUDE[vstecasp](../code-quality/includes/vstecasp_md.md)] 應用程式之偵測的必要步驟，請參閱[Debug ASP.NET applications](how-to-enable-debugging-for-aspnet-applications.md)。

- 在 ASP.NET 中調試用戶端腳本

  如需必要步驟，請參閱[Microsoft Edge 中的偵錯工具 JavaScript](https://devblogs.microsoft.com/visualstudio/debug-javascript-in-microsoft-edge-from-visual-studio/)和這[篇 Google Chrome](https://devblogs.microsoft.com/aspnet/client-side-debugging-of-asp-net-projects-in-google-chrome)的文章。

- Debug JavaScript 和 TypeScript

  請參閱[Debug JavaScript 和 TypeScript](../javascript/debug-nodejs.md)。

- 檢視指令碼文件

  您可以在 [方案總管] 查看伺服器端與用戶端指令碼文件清單並加以檢視。 您可以從 [ **方案總管**] 開啟任何指令碼文件。 如需詳細資訊，請參閱 [How to: View Script Documents](../debugger/how-to-view-script-documents.md)。

- 偵錯 AJAX 指令碼應用程式

  啟用 AJAX 功能的 Web 應用程式會使用大量指令碼，在偵錯時特別困難。 如需 AJAX 偵錯工具技術的相關資訊，請參閱[偵錯工具和追蹤 Ajax 應用程式總覽](https://msdn.microsoft.com/Library/92684ea0-7bb4-4a34-9203-3aa6394ce375)。
