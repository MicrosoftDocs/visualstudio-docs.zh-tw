# <a name="debugging-web-applications"></a>偵錯 Web 應用程式

本文提供的連結可協助您進行不同類型的 web 應用程式的偵錯工具。

- 對 ASP.NET 應用程式進行偵錯

  如需啟用應用程式的偵錯工具所需的步驟 [!INCLUDE[vstecasp](../code-quality/includes/vstecasp_md.md)] ，請參閱 [Debug ASP.NET applications](how-to-enable-debugging-for-aspnet-applications.md)。

- ASP.NET 中的 Debug 用戶端腳本

  如需必要步驟，請參閱 [Microsoft Edge 中](https://devblogs.microsoft.com/visualstudio/debug-javascript-in-microsoft-edge-from-visual-studio/) 的 blog 偵錯工具和 [Google Chrome 的這篇文章](https://devblogs.microsoft.com/aspnet/client-side-debugging-of-asp-net-projects-in-google-chrome)。

- 調試 JavaScript 和 TypeScript

  請參閱 [Debug JavaScript 和 TypeScript](../javascript/debug-nodejs.md)。

- 檢視指令碼文件

  您可以在 [方案總管] 查看伺服器端與用戶端指令碼文件清單並加以檢視。 您可以從 [ **方案總管**] 開啟任何指令碼文件。 如需詳細資訊，請參閱 [How to: View Script Documents](../debugger/how-to-view-script-documents.md)。

- 偵錯 AJAX 指令碼應用程式

  啟用 AJAX 功能的 Web 應用程式會使用大量指令碼，在偵錯時特別困難。 如需 AJAX 偵錯工具技術的詳細資訊，請參閱 [偵錯工具和追蹤 Ajax 應用程式總覽](/previous-versions/bb398817(v=vs.140))。
