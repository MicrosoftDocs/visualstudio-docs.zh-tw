# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>使用 .Pkgdef 檔案修改 Isolated Shell
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

.Pkgdef 檔案支援您可用來自訂獨立 shell 應用程式的設定。 它會指定在電腦上安裝應用程式時所建立的值，以及在啟動應用程式時由 Visual Studio shell 所參考的值。 這些設定會根據適用的登錄機碼組織在檔案中。  

> [!WARNING]
> 請注意，當 Visual Studio 啟動時，不會掃描未在 VSPackage 的 extension.vsixmanifest 檔案中宣告的 .pkgdef 檔案。  

 .Pkgdef 檔案包含每個以機碼（或子機碼）識別的區段， `[$RootKey$]` `[$RootKey$\` *subkey* `]` 其中 $RootKey $ 是應用程式的根金鑰。  

 每個區段都包含名稱/值組，其格式如下： `"` *ValueName* `"=` *值*。  

 值可以是以引號括住的字串，或是表示為 dword 的32位整數。 值具有下列條件約束：  

- 例如，所有 dword 值都採用十六進位格式 `dword:00000001` 。  

   若為布林值，1代表 true，0代表 false。  

- 所有 GUID 字串都是登錄格式，例如 `"{00000000-0000-0000-0000-000000000000}"` 。  

- 所有可當地語系化的資源識別碼都有 "@*resourceID*" 或 "#*resourceid*" 格式，其中 *resourceID* 是應用程式 UI 套件中的資源識別碼，例如 `"@102"` 。 應用程式 UI 封裝是 AppLocalizationPackage 設定中所參考的封裝。  

  例如，  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 您可以將批註新增至 .pkgdef 檔案。 單行批註有兩個斜線作為前兩個字元。  

 如需替代字串的清單，請參閱 [中使用的替代字串。.Pkgdef 和。.Pkgundef](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md)檔案。  

 下列各節說明特定的登錄值，這些值會影響 Visual Studio shell 在隔離模式中的行為。 您也可以在此檔案中為應用程式定義額外的登錄值。  

> [!NOTE]
> 如果 .pkgdef 檔案中未提供設定，則登錄中不會進行對應的專案。

## <a name="binding-path-settings"></a>系結路徑設定  
 [$RootKey $ \BindingPaths \\ {00000000-0000-0000-0000-000000000000} ] 鍵包含 shell 檢查元件的目錄清單。 這些目錄會加入至 shell 探查應用程式私用元件的目錄清單。  

 依預設，不會在 .pkgdef 檔案中加入任何系結路徑專案。 但是，Visual Studio 安裝目錄的下列子目錄會自動新增至登錄中的應用程式系結路徑清單。  

- Common7\IDE\  

- Common7\IDE \\ \PrivateAssemblies  

- Common7\IDE \\ \PublicAssemblies  

  若要將目錄新增至系結路徑，請新增 "*directoryName*" = "" 格式的專案，其中 *directoryName* 是絕對路徑。 例如，  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>封裝功能表項目設定  
 [$RootKey $ \Menus] 登錄機碼會定義應用程式的 UI 資源檔。  

 功能表項目值的格式為 "{*vsUiPackageGuid*}" = "、 *resourceId*、 *versionNumber*"，其中 *VSUIPACKAGEGUID* 是應用程式 UI 套件的 GUID， *resourceId* 是包含 UI 元素之 CTMENU 資源的資源識別碼，而 *versionNumber* 則是 CTMENU 資源的虛擬版本號碼。 如需詳細資訊，請參閱 [註冊 Interop 元件命令處理](../extensibility/internals/registering-interop-assembly-command-handlers.md)程式。  

 依預設，會在應用程式 UI 套件的 .pkgdef 檔案中建立功能表項目專案。  

 針對每個提供功能表項目的封裝，並將它當作應用程式的一部分散發，加入封裝的功能表項目專案。
