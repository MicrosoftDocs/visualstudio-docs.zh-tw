# <a name="share-models-and-exporting-diagrams"></a>共用模型及匯出圖表
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

在 Visual Studio 中，有多種方式可與同事共用模型和圖表。  
  
 若要查看哪些 Visual Studio 版本支援這項功能，請參閱 [Version support for architecture and modeling tools](../modeling/what-s-new-for-design-in-visual-studio.md#VersionSupport)。  
  
## <a name="working-on-a-model-as-a-team"></a>以小組型態處理模型  
 Visual Studio 的使用者可以建立和編輯模型、分層圖和有向圖形。  
  
 不同的使用者可以同時處理 UML 模型，但應該在不同的套件上處理。  
  
 建議您使用 [!INCLUDE[esprscc](../includes/esprscc-md.md)]。  
  
 如需詳細資訊，請參閱 <<c0> [ 管理模型與版本控制下的圖表](../modeling/manage-models-and-diagrams-under-version-control.md)。  
  
 如需組織中型或大型專案中的模型的建議，請參閱[建構模型方案](../modeling/structure-your-modeling-solution.md)。  
  
## <a name="reading-and-reviewing-models"></a>讀取和檢閱模型  
 使用某些版本的 Visual Studio （這無法建立模型） 的小組成員可以讀取模型，模型化圖表和有向的圖形，但無法加以編輯。  若要查看哪些 Visual Studio 版本支援這項功能，請參閱 [Version support for architecture and modeling tools](../modeling/what-s-new-for-design-in-visual-studio.md#VersionSupport)。  
  
 UML 圖表或分層圖只能在建立所在的模型中開啟。 若要提供圖表的存取，即必須提供所有模型化專案檔案的存取，或其複本的存取。  
  
 如需詳細資訊，請參閱 <<c0> [ 讀取模型和其他 Visual Studio 版本中的圖表](../modeling/read-models-and-diagrams-in-other-visual-studio-editions.md)。  
  
## <a name="using-diagrams-in-documents-and-presentations"></a>在文件和簡報中使用圖表  
 您可以將部分的圖表複製到文件或簡報。 您也可以列印圖表，例如列印為 XPS 檔。  
  
 如需詳細資訊，請參閱 <<c0> [ 將圖表匯出為影像](../modeling/export-diagrams-as-images.md)。
