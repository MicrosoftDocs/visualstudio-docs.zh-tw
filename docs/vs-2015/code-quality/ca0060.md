# <a name="ca0060"></a>CA0060

無法載入間接參考的元件。

如果分析工具無法載入在分析下由元件間接參考的元件，就會發生這個警告。 「間接參考」是指您的其中一個分析元件的元件參考所參考的參考元件。 例如，如果程式碼分析 (FxCop) 正在分析元件 A、元件 A 參考元件 B，而元件 B 參考元件 C，但 A 元件 B 參考元件 c，則元件 C 是間接參考，而元件 B 是直接參考。

這可能會導致在其他地方發生錯誤 CA0001，因為如果發生這種情況，可能會發生許多未預期的狀態。 如果您使用 Visual Studio 中的 managed 程式碼分析工具，以下是一些可能會導致 CA0060 警告的原因：

- 輸入元件已被覆寫，現在指向參考 Visual Studio 專案檔中參考清單以外之其他元件的元件。

- 如果專案已修改為在 FxCopCmd.exe 執行之前叫用自訂群組建步驟，而且自訂群組建程式新增更多對輸入元件的參考，並產生警告 CA0060。

- 在 c + + 中，您可以在 VCPROJ 中使用: Vcproj forcedusing 來參考元件，並覆寫命令列以傳遞/FU 或 \<Some.dll> 僅在來源中新增 #using。

- 具有部分私用參考的協力廠商元件，而您不需要這些元件來編譯和執行程式碼。

在所有情況下，這個問題的解決方法是將遺漏的參考加入 Visual Studio 專案本身。
