# <a name="c6214"></a>C6214
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
警告 C6214：在語義不同的整數類型之間轉換： HRESULT 轉換成布林類型

此警告表示正在 `HRESULT` 轉換成布林型別。 成功值 (`S_OK` 等於0的) `HRESULT` 。 不過，0表示布林值類型的失敗。 將轉換 `HRESULT` 成布林型別，然後在測試運算式中使用它，將會產生不正確的結果。 有時候，如果 `HRESULT` 儲存在布林值變數中，就會發生這個警告。 任何使用布林值變數來測試 `HRESULT` 成功或失敗的比較，可能會導致不正確的結果。

## <a name="example"></a>範例

下列程式碼會產生這個警告：

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);
  if ((BOOL)hr) // warning 6214
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

若要更正這個警告，請使用下列程式碼：

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (SUCCEEDED(hr))
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

針對這個警告， `SCODE` 類型相當於 `HRESULT` 。

通常，您 `SUCCEEDED` `FAILED` 應該使用或宏來測試的值 `HRESULT` 。

如需詳細資訊，請參閱下列其中一個主題：

[成功](/windows/desktop/api/winerror/nf-winerror-succeeded)

[失敗](/windows/desktop/api/winerror/nf-winerror-failed)

若要利用新式 c + + 記憶體配置方法，請使用 c + + 標準範本庫 (STL) 所提供的機制。 這些包含 [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、 [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)和 [向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱 [智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) 和 [c + + 標準程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
