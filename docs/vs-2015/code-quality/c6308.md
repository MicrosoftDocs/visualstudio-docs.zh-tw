# <a name="c6308"></a>C6308
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6308: 'realloc' 可能會傳回 null 指標： 指派了 null 指標給\<變數 >，做為引數傳遞至 'realloc'，會造成原始記憶體區塊遺漏  
  
 這則警告表示記憶體流失，是使用不正確的配置函式的結果。 堆積配置函式請勿釋放的傳遞的緩衝區，如果重新配置失敗。 若要更正該缺失，重新配置函式的結果指派給暫存，並取代原始指標重新配置成功之後。  
  
## <a name="example"></a>範例  
 下列範例程式碼會產生這個警告：  
  
```cpp  
#include <malloc.h>  
#include <windows.h>  
  
void f( )  
{  
  char *x;  
  x = (char *) malloc(10);  
  if (x != NULL)  
  {  
    x = (char *) realloc(x, 512);  
    // code...  
    free(x);  
  }     
}  
```  
  
 若要更正這個警告，請使用下列程式碼：  
  
```cpp  
#include <malloc.h>  
#include <windows.h>  
  
void f()  
{  
  char *x, *tmp;  
  
  x = (char *) malloc(10);  
  
  if (x != NULL)  
  {  
    tmp = (char *) realloc(x,512);  
    if (tmp != NULL)   
    {  
      x = tmp;  
    }  
    free(x);  
  }  
}  
```  
  
 這個警告可能會產生雜訊，如果當時重新配置函式的結果指派的即時的別名，以配置緩衝區。  
  
 若要完全避免這類的問題，請使用所提供的機制C++標準範本庫 (STL)。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，並[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)並[ C++ Standard 程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。</c0>
