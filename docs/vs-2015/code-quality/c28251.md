# <a name="c28251"></a>C28251
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C28251：函數的注釋不一致：這個實例發生錯誤  
  
 此警告會參考批註中的錯誤，並反映函式宣告上的批註必須符合定義上的注釋，除非 `typedef` 涉及函數。 在此情況下，函式 `typedef` 會針對宣告和定義取得為決定性。  
  
 請注意，批註通常會實作為宏，而一個宏通常會產生數個低層級的附注。 針對每個不相符的低層級批註報告這個警告，因此單一不相符的注釋宏可能會產生一些不相符的低層級批註。 最好只比較宣告和定義原始程式碼，以確保它們相同。  (不會報告附注順序中的簡單差異。 )   
  
 這項比較一律會在找到的第一個宣告與目前的宣告之間進行。 如果有其他宣告，則會以兩個群組來檢查每個宣告。 您目前無法進行成對的比較，雖然可以識別出兩個以上的宣告/定義。  上方的 *文字* 欄位包含批註清單，這些批註的 (在兩個實例之間的相對低層級) 不同。  
  
 此警告訊息會顯示傳送至編譯器的基礎程式碼文字，而不是用來實際將批註插入原始程式碼中的宏， (使用) 時使用宏的情況。 一般而言，您不需要瞭解低層級的附注，但您應該要知道批註在錯誤訊息中報告的行號之間會回報為不一致。 大部分情況下，檢查原始程式碼會讓它清楚知道有不一致的原因。
