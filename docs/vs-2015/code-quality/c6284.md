# <a name="c6284"></a>C6284
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6284：在呼叫中需要字串時，物件傳遞為參數 '% d ' \<function> 。  
  
 此警告表示格式字串會指定字串，例如 `%s` 或的規格 `printf` `scanf` ，但改為傳遞 c + + 物件。  
  
 此瑕疵可能會產生不正確的輸出或損毀。  
  
 這則訊息通常是因為將執行某些字串類型的 c + + 物件（例如， `std::string` `CComBSTR` 或 `bstr_t` ）傳遞到 C `printf` 樣式的呼叫而回報。 視 c + + 類別的執行而定，也就是如果定義了適當的 cast 運算子，在需要 C 字串時，c + + 字串物件通常可以明確地使用;不過，因為 `printf` -樣式函式的參數本質上不具類型，所以不會轉換成字串。  
  
 視物件而定，可能會適當地將運算子插入 `static_cast` 適當的字串類型，例如 `char *` 或，或呼叫會傳回字串的成員函式（ `TCHAR``*` 例如， `c_str()` 在的實例上） `std::string` 。  
  
## <a name="example"></a>範例  
 下列程式碼會產生此警告，因為 `CComBSTR` 會傳遞至函式 `sprintf` ：  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf(buff,"%ws",bstrValue);   
}  
```  
  
 下列程式碼會使用靜態轉換來修正此警告：  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));  
}  
```
