# <a name="c6211"></a>C6211
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6211:發生記憶體溢位\<指標 > 因為發生例外狀況。 請考慮使用區域 catch 區塊來清除記憶體  
  
 這則警告表示已配置的記憶體不會被釋放時擲回例外狀況。 路徑結尾的陳述式可能會擲回例外狀況。 「 分析器 」 會檢查這個條件時，才`_Analysis_mode_(_Analysis_local_leak_checks_)`指定 SAL 註釋。 根據預設，此註解被指定的 Windows 核心模式 （驅動程式） 的程式碼。 如需 SAL 註釋的詳細資訊，請參閱[使用 SAL 註釋減少 C /C++程式碼缺失](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這則警告，因為例外狀況可能在第二個配置時擲回，因而流失第一個配置，或者例外狀況可能會在程式碼中 "`code ...`" 註解所表示的某處擲回，因而流失了兩個配置。  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4   
#include <sal.h>  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
void f( )  
{  
    char *p1 = new char[10];  
    char *p2 = new char[10];  
  
    // code ...  
  
    delete[] p2;  
    delete[] p1;  
}  
  
```  
  
 若要使用相同的配置方法並修正這個問題，請加入例外狀況處理常式：  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <new>  
#include <iostream>  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f()  
{  
    char *p1 = nullptr;   
    char *p2 = nullptr;  
  
    try  
    {  
        p1 = new char[10];  
        p2 = new char[10];  
  
        // code ...  
  
        delete [] p2;  
        delete [] p1;  
    }  
    catch (const bad_alloc& ba)  
    {  
        cout << ba.what() << endl;  
        delete [] p2;  
        delete [] p1;  
    }  
    // code ...  
}  
  
```  
  
 若要避免這類可能的流失發生，請使用 C++ 標準範本庫 (STL) 提供的機制。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，並[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)並[ C++ Standard 程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。</c0>  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <vector>  
#include <memory>  
  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f( )  
{  
    // use 10-element vectors in place of char[10]  
    vector<char> v1;  
    vector<char> v2;  
  
    for (int i=0; i<10; ++i) {  
        v1.push_back('a');  
        v2.push_back('b');  
    }  
    // code ...  
  
    // use unique_ptr if you still want char[10]  
    unique_ptr<char[]> a1(new char[10]);  
    unique_ptr<char[]> a2(new char[10]);  
  
    // code ...  
  
    // No need for delete; vector and unique_ptr   
    // clean up when out of scope.  
}  
  
```
