# <a name="c6250"></a>C6250
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6250：在沒有 MEM_RELEASE 旗標的情況下呼叫 \<函式 > VirtualFree 可能會釋放記憶體，而不是位址描述項（Vad）;導致位址空間流失的結果  
  
 此警告表示呼叫不含 `MEM_RELEASE` 旗標的 `VirtualFree` 只會解除頁面的分頁，而且不會釋放它們。 若要 offerreclaim11 取消認可和發行頁面，請在 `VirtualFree`的呼叫中使用 `MEM_RELEASE` 旗標。 如果區域中的任何頁面已認可，函式會先解除，然後再釋放它們。 在此作業之後，頁面會處於 [免費] 狀態。 如果您指定此旗標，`dwSize` 必須為零，而且 `lpAddress` 必須指向保留區域時，`VirtualAlloc` 函式所傳回的基底位址。 如果不符合上述任一條件，函數就會失敗。  
  
 如果您的程式碼稍後藉由使用 `MEM_RELEASE` 旗標來呼叫 `VirtualFree`，就可以忽略此警告。  
  
 如需詳細資訊，請參閱[VirtualAlloc](https://msdn.microsoft.com/library/aa366887.aspx)和[VirtualFree](https://msdn.microsoft.com/library/aa366892.aspx)。  
  
 在記憶體流失和例外狀況方面，使用 VirtualAlloc 和 VirtualFree 有許多陷阱。 若要避免這類流失和例外狀況問題，請使用C++標準範本程式庫（STL）提供的機制。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、 [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)和[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱[智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)和[ C++標準程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。  
  
## <a name="example"></a>範例  
 下列範例程式碼會產生這個警告：  
  
```  
#include <windows.h>  
#include <stdio.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo(&sSysInfo);    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS);       
  //  
  // code to access memory   
  // ...  
  
  if (lpvBase != NULL)  
  {  
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages  
      {  
        puts ("MEM_DECOMMIT Succeeded");  
      }  
      else  
      {  
       puts("MEM_DECOMMIT failed");  
      }  
  }  
  else  
  {  
    puts("lpvBase == NULL");  
  }  
}  
```  
  
 若要修正這個警告，請使用下列範例程式碼：  
  
```  
#include <windows.h>  
#include <stdio.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo(&sSysInfo);    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS);       
  //  
  // code to access memory   
  // ...  
  
  if (lpvBase != NULL)  
  {  
   if (VirtualFree(lpvBase, 0,MEM_RELEASE )) // decommit & release   
   {  
     // code ...  
   }  
   else  
   {  
     puts("MEM_RELEASE failed");  
   }  
  }  
  else   
  {  
    puts("lpvBase == Null ");  
    // code...  
  }  
}  
```
