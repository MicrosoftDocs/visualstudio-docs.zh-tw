# <a name="c6511"></a>C6511
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6511：不正確注釋： MustCheck 屬性必須是 [是] 或 [否]  
  
> [!NOTE]
> 只有使用已淘汰版本的原始程式碼批註語言 (SAL) 的程式碼才會發生這個警告。 建議您移植程式碼，以使用最新版本的 SAL。 如需詳細資訊，請參閱 [使用 SAL 注釋減少 C/c + + 程式碼](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)缺失。  
  
 此警告表示指定的 MustCheck 屬性值無效。 這個屬性唯一有效的值為： Yes 和 No。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
[returnvalue:SA_Post(MustCheck=SA_Maybe)] int f();  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
[returnvalue:Post(MustCheck=Maybe)] int f();  
```  
  
 若要更正這個警告，請在下列程式碼中使用 MustCheck 屬性的有效值：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
[returnvalue:SA_Post(MustCheck=SA_Yes)] int f();  
  
// C++    
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
[returnvalue:Post(MustCheck=Yes)] int f();  
```
