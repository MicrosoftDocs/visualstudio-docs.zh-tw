# <a name="c6516"></a>C6516
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6516：不正確注釋：未針對屬性指定屬性 \<name>  
  
> [!NOTE]
> 只有使用已淘汰版本的原始程式碼批註語言 (SAL) 的程式碼才會發生這個警告。 建議您移植程式碼，以使用最新版本的 SAL。 如需詳細資訊，請參閱 [使用 SAL 注釋減少 C/c + + 程式碼](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)缺失。  
  
 此警告表示屬性中未指定任何屬性，或指定的屬性無效;因此，無法將屬性視為完成。  
  
## <a name="example"></a>範例  
 下列程式碼會產生此警告，因為 Deref = 1 只會指定間接取值層級，但這項資訊只會協助分析工具：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f([SA_Pre(Deref=1)] char* pc);  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre(Deref=1)] char* pc);  
  
```  
  
 若要更正這個警告，必須有另一個屬性（例如 Access），才能向分析工具指出哪些專案必須在取消參考的專案上強制執行。 下列程式碼會修正此警告：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f([SA_Pre(Deref=1, Access=SA_Read)] char* pc);   
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre(Deref=1, Access=Read)] char* pc);  
```
