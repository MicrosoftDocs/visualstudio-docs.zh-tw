# <a name="c6516"></a>C6516
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6516： 無效的註釋： 沒有針對指定的屬性\<名稱 > 屬性  
  
> [!NOTE]
> 只在使用已被取代的版本的原始程式碼註釋語言 (SAL) 的程式碼中，會發生這個警告。 我們建議您將移植程式碼以使用最新版的 SAL。 如需詳細資訊，請參閱 <<c0> [ 使用 SAL 註釋減少 C /C++程式碼缺失](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)。</c0>  
  
 這則警告指出未指定屬性的屬性中，或是指定的屬性無效;因此，您無法將屬性視為為完成。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告，因為 Deref = 1 只會指定的層級的間接取值，但這項資訊就不會協助分析工具：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f([SA_Pre(Deref=1)] char* pc);  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre(Deref=1)] char* pc);  
  
```  
  
 若要更正這個警告，另一個屬性，例如存取，才能指出分析工具功能必須解除參考的項目上強制執行。 下列程式碼會更正這個警告：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f([SA_Pre(Deref=1, Access=SA_Read)] char* pc);   
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre(Deref=1, Access=Read)] char* pc);  
```
