# <a name="c6297"></a>C6297
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6297：算術溢位： 32-位值已移位，然後轉換為64位值。 結果可能不是預期的值  
  
 此警告表示不正確的行為，因為整數升階規則和類型大於通常執行算術的型別。  
  
 在此情況下，32位值已左移，而該移位的結果已轉換為64位值。 如果排班溢位32位的值，則會遺失 bits。  
  
 如果您不想要遺失位，請將值轉換成64位數量後再移位。 如果您想要遺失位、執行適當的轉換成不帶正負號的 long 或 short 型別，或遮罩移位的結果，將會消除這個警告，並讓程式碼的意圖更清楚。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
void f(int i)  
{  
  unsigned __int64 x;  
  
  x = i << 34;  
  // code   
}  
```  
  
 若要更正這個警告，請使用下列程式碼：  
  
```  
void f(int i)  
{  
  unsigned __int64 x;  
  // code  
  x = ((unsigned __int64)i) << 34;  
}  
```
