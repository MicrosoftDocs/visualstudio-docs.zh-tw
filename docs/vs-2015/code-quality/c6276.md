# <a name="c6276"></a>C6276
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6276：在語義不同的字串類型之間轉換： char * 可 wchar_t \* 。 使用不正確字串可能會導致未定義的行為  
  
 此警告表示從 ANSI 字串轉換 `char_t*` 為 UNICODE 字串 ()  () 可能不正確 `wchar_t *` 。 由於 UNICODE 字串的字元大小為2個位元組，因此這項轉換可能會產生未正確終止的字串。 使用具有 wcs * 函數程式庫的這類字串可能會造成緩衝區溢位和存取違規。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <windows.h>  
VOID f()  
{  
   WCHAR szBuffer[8];  
   LPWSTR pSrc;  
  
   pSrc = (LPWSTR)"a";  
   wcscpy(szBuffer, pSrc);   
}  
```  
  
 下列程式碼會附加字母 L 以將 ASCII 字元表示為寬字元，以修正這個警告：  
  
```  
#include <windows.h>  
  
VOID f()  
{  
   WCHAR szBuffer[8];  
   LPWSTR pSrc;  
  
   pSrc = L"a";  
   wcscpy(szBuffer, pSrc);  
}  
```  
  
 下列程式碼會使用安全字串操作函式 `wcscpy_s` 來修正這個警告：  
  
```  
#include <windows.h>  
  
VOID f()  
{  
   WCHAR szBuffer[8];  
   LPWSTR pSrc;  
   pSrc = L"a";  
   wcscpy_s(szBuffer,8,pSrc);   
}  
```
