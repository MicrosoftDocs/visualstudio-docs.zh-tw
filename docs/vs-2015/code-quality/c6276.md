# <a name="c6276"></a>C6276
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

本主題的最新的版本可從[C6276](https://docs.microsoft.com/visualstudio/code-quality/c6276)。  
  
警告 C6276： 語意不相同的字串類型之間轉型： char * 為 wchar_t\*。 使用無效的字串可能會導致未定義的行為  
  
 這則警告表示可能有不正確的型別轉換，從 ANSI 字串 (`char_t*`) 的 UNICODE 字串 (`wchar_t *`)。 UNICODE 字串有 2 個位元組的字元大小，因為這項轉換可能會產生不正確地終止的字串。 Wcs * 程式庫函式中使用這類字串可能會導致緩衝區滿溢和存取違規。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <windows.h>  
VOID f()  
{  
   WCHAR szBuffer[8];  
   LPWSTR pSrc;  
  
   pSrc = (LPWSTR)"a";  
   wcscpy(szBuffer, pSrc);   
}  
```  
  
 下列程式碼會藉由附加字母 L 代表以寬字元的 ASCII 字元，更正這個警告：  
  
```  
#include <windows.h>  
  
VOID f()  
{  
   WCHAR szBuffer[8];  
   LPWSTR pSrc;  
  
   pSrc = L"a";  
   wcscpy(szBuffer, pSrc);  
}  
```  
  
 下列程式碼會使用安全的字串操作函式， `wcscpy_s`，若要更正這個警告：  
  
```  
#include <windows.h>  
  
VOID f()  
{  
   WCHAR szBuffer[8];  
   LPWSTR pSrc;  
   pSrc = L"a";  
   wcscpy_s(szBuffer,8,pSrc);   
}  
```
