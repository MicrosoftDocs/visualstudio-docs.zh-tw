# <a name="c6332"></a>C6332
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6332：不正確參數：將零當做 dwFreeType 參數傳遞給 \<函式 > 不允許使用。 這會導致此呼叫失敗  
  
 此警告表示將不正確參數傳遞至 VirtualFree 或 VirtualFreeEx。 VirtualFree 和 VirtualFreeEx 都會拒絕 dwFreeType 參數零。 DwFreeType 參數可以是 MEM_DECOMMIT 或 MEM_RELEASE。 不過，不能在相同的呼叫中同時使用 MEM_DECOMMIT 和 MEM_RELEASE 值。 此外，請確定不會忽略 VirtualFree 函數的傳回值。  
  
## <a name="example"></a>範例  
 下列程式碼會產生此警告，因為傳遞給 VirtualFree 函數的參數無效：  
  
```  
#include <windows.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  
  bSuccess = VirtualFree( lpvBase, 0, 0 );   
  // code ...  
}  
```  
  
 若要修正這個警告，請修改 VirtualFree 函式的呼叫，如下列程式碼所示：  
  
```  
#include <windows.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  
  bSuccess = VirtualFree( lpvBase, 0, MEM_RELEASE );  
  // code ...  
}   
```  
  
 在記憶體流失和例外狀況方面，使用 VirtualAlloc 和 VirtualFree 有許多陷阱。 若要避免這類流失和例外狀況問題，請使用C++標準範本程式庫（STL）提供的機制。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、 [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)和[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱[智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)和[ C++標準程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
