# <a name="c28303"></a>C28303
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C28303：針對 c + + 參考-參數 <parameter_name>， `_Deref_` 在上找到不明確的運算子 \<annotation> 。  
  
 此警告類似于警告 C28302，而且會在參數上使用額外的層級時回報 `_Deref_` 。  
  
 處理參考參數時，SAL2 不需要使用額外的層級 `_Deref_` 。 這個特定的注釋並不明確，因為要標注的取值層級。 可能需要使用 `_At_` 來參考要標注的特定物件。  
  
## <a name="example"></a>範例  
 下列程式碼會產生此警告，因為使用 `__deref_out_ecount(n)` 不明確：  
  
```  
void ref(__deref_out_ecount(n) int **&buff, int &n)  
```  
  
 上述批註可解釋為：  
  
- 陣列的參考 (n) 指向整數的指標 (SAL1 轉譯)   
  
- 指向陣列 (n) 整數的指標的參考 (SAL2 轉譯)   
  
  下列任一項都可以更正此警告：  
  
```  
void ref(_Out_writes_(n) int **&buff, int &n)  
// or  
_At_(*buff), _Out_writes(n)) void ref(int **&buff, int &n)  
  
```
