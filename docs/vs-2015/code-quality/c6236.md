# <a name="c6236"></a>C6236
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6236： (\<expression> &#124;&#124; \<non-zero constant>) 一律為非零的常數  
  
 此警告表示在測試內容中所發生的邏輯 OR 運算右側偵測到非零的常數值，而不是一個。 邏輯上，這表示測試是多餘的，而且可以安全地移除。 另外，它也會建議程式設計人員可能想要使用不同的運算子，例如，相等 (`==`) 、位和 (`&`) 或位 XOR (`^`) 運算子，以測試特定的值或旗標。  
  
 當非零常數為1時，不會針對常見的用法產生此警告，因為它在編譯時期會使用選擇性啟用程式碼路徑。 但是，如果非零常數是由評估為1的運算式形成，例如 1 + 0，則會產生警告。  
  
## <a name="example"></a>範例  
 這段程式碼會說明警告 C6236 的顯示方式。 因為不 `INPUT_TYPE` 是0，所以運算式 `n || INPUT_TYPE` 一律為非零，且不會 `else` 執行子句。 但 `INPUT_TYPE` 它是一個不是一個值的常數，這表示它是要做比較的值：  
  
```  
#define INPUT_TYPE 2  
#include <stdio.h>  
  
void f( int n )  
{  
   if ( n || INPUT_TYPE ) // analysis warning C6236  
   {  
      puts( "Always gets here" );  
   }  
   else  
   {  
      puts( "Never enters here" );  
   }  
}  
```  
  
 下列程式碼會改為使用位 AND (`&`) 運算子來測試是否 `INPUT_TYPE` 已設定輸入參數的位 `n` ：  
  
```  
#define INPUT_TYPE 2  
#include <stdio.h>  
  
void f( int n )  
{  
   if ( n & INPUT_TYPE ) // no warning  
   {  
      puts( "Bitwise-AND comparison is true" );  
   }  
   else  
   {  
      puts( "Bitwise-AND comparison is false" );  
   }  
}  
```
