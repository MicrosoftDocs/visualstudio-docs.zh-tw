# <a name="c6215"></a>C6215
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6215：在語義不同的整數類型之間轉換：布林值類型為 HRESULT  
  
 此警告表示布林值正在轉換成 `HRESULT` 。 布林值型別以非零的值表示成功，而成功的 (`S_OK`) `HRESULT` 則是以0的值表示。 將布林型別轉換成 `HRESULT` ，然後在測試運算式中使用它，將會產生不正確的結果。  
  
 當布林值當做或宏的引數使用時，通常會發生這種警告 `SUCCEEDED` `FAILED` ，這會明確地將其引數轉換為 `HRESULT` 。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
void f( REFGUID riid1, REFGUID riid2 )  
{  
  if (SUCCEEDED( IsEqual( riid1, riid2 ) )) //warning 6215  
  {  
    // success code ...  
  }  
  else  
  {  
    // failure code ...  
  }  
}  
```  
  
 一般而言， `SUCCEEDED` 或 `FAILED` 宏只能套用至 `HRESULT` 。  
  
 若要更正這個警告，請使用下列程式碼：  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
void f( REFGUID riid1, REFGUID riid2 )  
{  
  if (IsEqual( riid1, riid2 ) == TRUE)   
  {  
    // code for riid1 == riid2  
  }  
  else  
  {  
    // code for riid1 != riid2  
  }  
}  
```  
  
 如需詳細資訊，請參閱 [成功宏](https://msdn.microsoft.com/library/ms687197.aspx) 和 [失敗宏](https://msdn.microsoft.com/library/ms693474.aspx)
