# <a name="c6503"></a>C6503
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6503: 無效的註釋: 參考和陣列不能標記為 Null=Yes 或 Null=Maybe  
  
> [!NOTE]
> 只有使用已淘汰版本的原始程式碼批註語言 (SAL) 的程式碼才會發生這個警告。 建議您移植程式碼，以使用最新版本的 SAL。 如需詳細資訊，請參閱 [使用 SAL 注釋減少 C/c + + 程式碼](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)缺失。  
  
 這則警告指出，參考或陣列類型上未正確使用 Null 屬性。 參考或陣列類型會保存物件的位址，而且必須指向有效的物件。 因為參考和陣列類型不能是 null，所以您必須移除 Null 屬性或將 Null 屬性值設定為 No，以便修正錯誤。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```cpp  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
class Point  
{  
public:  
    //  members  
};  
  
void f([Pre(Null=Yes)] Point& pt);  
```  
  
 若要更正這個警告，請將 Null 屬性設定為 [否]，如下列程式碼所示：  
  
```cpp  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
  
class Point  
{  
public:  
    //  members  
};   
void f([Pre(Null=No)] Point& pt);  
```
