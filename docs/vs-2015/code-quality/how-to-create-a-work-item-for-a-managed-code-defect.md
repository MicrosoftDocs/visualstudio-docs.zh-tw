# <a name="how-to-create-a-work-item-for-a-managed-code-defect"></a>如何：為 Managed 程式碼的缺失建立工作項目
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

您可以使用工作專案追蹤功能來記錄中的工作專案 [!INCLUDE[vsPreLong](../includes/vsprelong-md.md)] 。 若要使用此功能，您的專案必須是中 Team 專案的一部分 [!INCLUDE[esprfound](../includes/esprfound-md.md)] 。

### <a name="to-create-a-work-item-for-managed-code-defect"></a>若要建立 managed 程式碼缺失的工作專案

1. 在 [程式 **代碼分析** ] 視窗中，選取警告。

2. 選擇 [ **動作**]，然後選擇 [ **建立工作專案** ]，並選擇要建立的工作專案類型。

     系統會為您建立新的工作專案，以指定瑕疵資訊。

### <a name="to-create-a-work-item-for-multiple-managed-code-defects"></a>若要建立多個受控碼瑕疵的工作專案

1. 在 [ **錯誤清單**] 中，選取多個警告，然後以滑鼠右鍵按一下警告。

2. 指向 [ **建立工作專案** ]，然後按一下要建立的工作專案類型。

     系統會為所有選取的警告建立單一工作專案，讓您指定錯誤資訊。
