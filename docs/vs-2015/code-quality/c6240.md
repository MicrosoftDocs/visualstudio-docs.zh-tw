# <a name="c6240"></a>C6240
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6240： (\<expression>  &&  \<non-zero constant>) 一律會評估為的結果 \<expression> 。 您是否想要使用位 and 運算子？  
  
 此警告表示在測試內容中所發生的邏輯 and 運算右側，偵測到非零的常數值，而不是一個。 例如，運算式 `(n && 3)` 會縮減為 `(!!n)` ，也就是的布林值 `n` 。  
  
 這項警告通常表示嘗試檢查位元遮罩，其中應使用位 AND (`&`) 運算子。 如果非零的常數評估為1，則不會產生它，因為它會使用選擇性地選擇程式碼路徑。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <stdio.h>  
#define INPUT_TYPE 2  
  
void f(int n)  
{  
   if (n && INPUT_TYPE)   
   {  
      puts("boolean value of !!n is true");  
   }  
   else  
   {  
      puts("boolean value of !!n is false");  
   }  
}  
  
```  
  
 若要更正這個警告，請使用位 AND 運算子，如下列程式碼所示：  
  
```  
#include <stdio.h>  
#define INPUT_TYPE 2  
  
void f(int n)  
{  
   if ( (n & INPUT_TYPE) )  
   {  
      puts("bitmask true");  
   }  
   else  
   {  
      puts("bitmak false");  
   }  
}  
```
