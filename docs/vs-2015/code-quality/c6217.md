# <a name="c6217"></a>C6217

[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
警告 C6217：語義不同的整數類型之間的隱含轉換：使用 ' not ' 測試 HRESULT。 請考慮改為使用 [SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded) 或 [FAILED](/windows/desktop/api/winerror/nf-winerror-failed) 宏。

此警告表示 `HRESULT` 正在使用 not (`!`) 運算子測試。  (`S_OK` 中) 的成功 `HRESULT` 是以0的值表示。 不過，0表示布林值類型的失敗。 `HRESULT`使用 not 運算子測試 (`!`) 來判斷要執行的程式碼區塊，可能會導致下列錯誤的程式碼路徑。 這會導致不想要的結果。

## <a name="example"></a>範例

下列程式碼會產生此警告，因為 not 運算子用來判斷值的成功或失敗 `HRESULT` 。 在此情況下，因為執行失敗碼，所以會執行錯誤的程式碼路徑 `( !hr )` ：

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (!hr)
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

若要更正這個警告，下列程式碼會使用 `FAILED` 宏來尋找失敗：

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (FAILED(hr))
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

針對這個警告， `SCODE` 類型相當於 `HRESULT` 。

HRESULT 的一般成功值 (`S_OK`) 是 `false` 測試為布林值時。

若要確認是否 `HRESULT` 成功，請改為使用 `SUCCEEDED` 宏。
