# <a name="c26110"></a>C26110
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C26110：呼叫端無法在呼叫函式之前保留鎖定 \<lock> \<func> 。  
  
 當需要鎖定時，請務必明確指出函式本身或其呼叫端是否應取得鎖定。 當批註違規時，就會發出警告 C26110 `_Requires_lock_held_` 。  
  
## <a name="example"></a>範例  
 在下列範例中，由於函式上的注釋 `_Requires_lock_held_` 指出的呼叫端必須在 `LockRequired` `LockRequired` 呼叫之前取得鎖定，因此會產生警告 C26110 `LockRequired` 。 若沒有此注釋， `LockRequired` 則必須先取得鎖定，才能存取鎖定所保護的任何共用資料。  
  
```  
  
typedef struct _DATA   
{  
    CRITICAL_SECTION cs;  
    int d;  
} DATA;  
  
_Requires_lock_held_(p->cs)  
  
void LockRequired(DATA* p)  
{  
    p->d = 0;  
}  
  
void LockNotHeld(DATA* p)   
{   
    LockRequired(p); // Warning C26110   
}  
  
```
