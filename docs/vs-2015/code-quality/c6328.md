# <a name="c6328"></a>C6328
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6328：大小不符： \<type> \<number> 當呼叫時， \<type> 必須以參數形式傳遞 \<function>  
  
 針對系列名稱 xxx 中的 C 執行時間字元型常式 `is` *xxx* `()` ，傳遞類型的引數 `char` 可能會產生無法預期的結果。 例如，值大於 `char` 且類型為 `0x7F` 的 SBCS 或 MBCS 單一位元組字元為負值。 如果 `char` 傳遞，編譯器可能會將值轉換成帶正負號 `int` 或已簽署的 `long` 。 此值可由編譯器進行簽署，並產生非預期的結果。 例如， `isspace` 接受類型的引數 `int` ，但其輸入引數的有效值範圍為：  
  
 `0 <= c <= 255`和特殊值 `EOF` 。  
  
## <a name="example"></a>範例  
 根據預設，`char` 在 Visual C++ 中是帶正負號的類型，所以 char 類型之變數的值範圍是 `-128 <= c <= 127`。 因此，如果您執行了下列動作：  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( c );  
    // code ...  
}  
  
```  
  
 `c` 會以-37 的帶正負號的符號擴充 `int` ，此值不在 [isspace](https://msdn.microsoft.com/library/3ac254c4-92aa-4238-882a-3a3fb576cdf5)的有效範圍內。  
  
 若要更正這個問題，您可以使用 `static_cast`，如下列程式碼所示：  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( static_cast<unsigned char> (c) );  
    // code ...  
}  
  
```  
  
 警告 C6328 專門用來捕捉此錯誤。 對於 7 位元 ASCII 範圍的字元，轉換並不需要，不過該範圍以外的字元可能會產生無法預期的結果，最嚴重可達程式錯誤和終止。
