# <a name="c6328"></a>C6328
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6328：大小不符：當呼叫 \<函式需要 > 類型 \<時，\<類型 > 當做參數傳遞 \<號碼 > >  
  
 針對系列名稱中的 C 執行時間字元型常式 `is`*xxx*`()`，傳遞類型 `char` 的引數可能會產生無法預期的結果。 例如，值大於 `char` 且類型為 `0x7F` 的 SBCS 或 MBCS 單一位元組字元為負值。 如果傳遞 `char`，編譯器可能會將值轉換成已簽署的 `int` 或已簽署的 `long`。 編譯器會對此值進行正負號擴充，但產生非預期的結果。 例如，`isspace` 接受 `int`類型的引數。不過，其輸入引數值的有效範圍為：  
  
 `0 <= c <= 255`，加上 `EOF`的特殊值。  
  
## <a name="example"></a>範例  
 根據預設，`char` 在 Visual C++ 中是帶正負號的類型，所以 char 類型之變數的值範圍是 `-128 <= c <= 127`。 因此，如果您執行了下列動作：  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( c );  
    // code ...  
}  
  
```  
  
 `c` 會以-37 的值延伸到已簽署的 `int`，而此值不在[isspace](https://msdn.microsoft.com/library/3ac254c4-92aa-4238-882a-3a3fb576cdf5)的有效範圍內。  
  
 若要更正這個問題，您可以使用 `static_cast`，如下列程式碼所示：  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( static_cast<unsigned char> (c) );  
    // code ...  
}  
  
```  
  
 警告 C6328 會特別存在，以攔截此 bug。 對於 7 位元 ASCII 範圍的字元，轉換並不需要，不過該範圍以外的字元可能會產生無法預期的結果，最嚴重可達程式錯誤和終止。
