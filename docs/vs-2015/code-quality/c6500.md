# <a name="c6500"></a>C6500
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6500： 無效的附註： 值\<名稱 > 屬性無效  
  
> [!NOTE]
> 只在使用已被取代的版本的原始程式碼註釋語言 (SAL) 的程式碼中，會發生這個警告。 我們建議您將移植程式碼以使用最新版的 SAL。 如需詳細資訊，請參閱 <<c0> [ 使用 SAL 註釋減少 C /C++程式碼缺失](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)。</c0>  
  
 這個警告表示註解中使用的屬性值無效。 比方說，它可能會發生如果 Deref 屬性用於取值層級但是不正確，或如果您使用大於 size_t ElementSize 等屬性的常數值。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告，因為取值層級但是不正確會在前置條件：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f( [SA_Pre( Deref=2, Access=SA_Read )] char buffer[] );  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
  
void f( [Pre( Deref=2, Access=Read )] char buffer[] );  
```  
  
 若要更正這個警告，請指定正確的層級的取值，如下列範例程式碼所示：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f( [SA_Pre( Deref=1, Access=SA_Read )] char buffer[] );  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
  
void f( [Pre( Deref=1, Access=Read )] char buffer[] );  
```  
  
 前置和後置條件會產生這個警告。
