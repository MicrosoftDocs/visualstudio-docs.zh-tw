# <a name="c6244"></a>C6244
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6244：的區域宣告會 \<variable> 隱藏先前 \<line> 的宣告 \<file>  
  
 此警告表示宣告與外部範圍的宣告具有相同的名稱，並隱藏先前的宣告。 您將無法從區域範圍內部參考先前的宣告。 上述宣告的任何預期用法最後都會使用區域宣告，此警告只會識別範圍重迭，而不是存留期重迭。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <stdlib.h>  
#pragma warning(push)  
  
// disable warning C4101: unreferenced local variable   
#pragma warning(disable: 4101)   
  
int i;  
void f();  
void (*pf)();  
  
void test()  
{  
    // Hide global int with local function pointer  
    void (*i)();  //Warning: 6244  
  
     // Hide global function pointer with an int  
    int pf;     //Warning: 6244  
}  
#pragma warning(pop)  
```  
  
 若要更正這個警告，請使用下列範例程式碼：  
  
```  
#include <stdlib.h>  
#pragma warning(push)  
// disable warning C4101: unreferenced local variable   
#pragma warning(disable: 4101)   
  
int g_i;          // modified global variable name  
void g_f();       // modified global function name  
void (*f_pf)();   // modified global function pointer name  
  
void test()  
{  
    void (*i)();  
    int pf;  
}  
#pragma warning(pop)  
```  
  
 處理記憶體配置時，請參閱程式碼以判斷配置是否儲存在一個變數中，並由另一個變數釋放。
