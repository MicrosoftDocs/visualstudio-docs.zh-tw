# <a name="c6268"></a>C6268
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6268：不正確的作業順序： (\<TYPE1>) # B2 \<TYPE2>) x + y。  (中可能缺少括弧 \<TYPE1>) # A2 # B3 \<TYPE2>) x + y)   
  
 此警告表示複雜的 cast 運算式在執行指標算術時可能會牽涉到優先順序問題。 因為 cast 群組比二元運算子更緊密，所以結果可能不是程式設計人員想要的結果。 在某些情況下，此瑕疵會導致不正確的行為或程式損毀。  
  
 在下列運算式中：  
  
 `(char *)p + offset`  
  
 位移會解讀為字元的位移;不過，運算式如下：  
  
 `(int *)(char *)p + offset`  
  
 相當於：  
  
 `((int *)(char *)p) + offset`  
  
 因此，位移會被解釋為整數中的位移。 換句話說，它相當於：  
  
 `(int *)((char *)p + (offset * sizeof(int)))`  
  
 這不太可能是程式設計人員所要的。  
  
 根據兩種類型的相對大小，這可能會導致緩衝區溢位。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
void f(int *p, int offset_in_bytes)  
{  
    int *ptr;  
    ptr = (int *)(char *)p + offset_in_bytes;  
    // code ...  
}  
  
```  
  
 若要更正這個警告，請使用下列程式碼：  
  
```  
void f(int *p, int offset_in_bytes)  
{  
    int *ptr;  
    ptr = (int *)((char *)p + offset_in_bytes);  
    // code ...  
}  
```
