# <a name="c28104"></a>C28104
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C28104：未取得函式結束之前應取得的資源  
  
 一種函式，其目的是要在結束之前取得資源，而不需要取得資源。 此警告表示函式已加上批註， `__drv_acquiresResource` 但未傳回實際取得資源的功能。 如果此函式是包裝函式，則透過函式的路徑不會到達包裝的函式。 如果無法連線到包裝的函式是因為函式傳回錯誤，而未實際取得資源，您可能需要使用條件式注釋 (`__drv_when`) 。  
  
 如果此函式實際實作為資源的取得，則 PFD 可能無法偵測到已取得資源。 在此情況下，請使用 `#pragma` 警告來抑制錯誤。 您可能會將放 `#pragma` 在開始函式主體之前的行上 `{` 。 呼叫的函式仍然需要注釋，但程式碼分析工具將無法偵測是否已取得資源。  
  
## <a name="example"></a>範例  
  
```  
__drv_acquireResourceGlobal(HWLock, lockid)  
void GetHardwareLock(lockid)  
#pragma warning (suppress: 28104)  
{  
   // code to implement a hardware lock (which the Code Analysis tool can't recognize)  
}  
```
