# <a name="c6269"></a>C6269
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6269：可能不正確的作業順序：已忽略取值  
  
 此警告表示會忽略指標取值的結果，這會引發第一次將指標取值的原因問題。  
  
 編譯器將會正確地將不必要的取值優化。 不過，在某些情況下，此瑕疵可能會反映優先順序或邏輯錯誤。  
  
 這項瑕疵的一個常見原因是下列格式的運算式語句：  
  
```  
*p++;  
```  
  
 如果這個語句的目的只是要遞增指標 `p` ，則不需要取值。不過，如果目的是要遞增指向的位置 `p` ，則程式不會如預期般運作，因為  \* p + + 結構會被解讀為 `\* (p++)` 。 而不是 `(*p)++` 。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <windows.h>  
  
void f( int *p )  
{  
    // code ...  
  if( p != NULL )  
    *p++;  
    // code ...  
}  
```  
  
 若要更正這個警告，請使用括弧，如下列程式碼所示：  
  
```  
#include <windows.h>  
  
void f( int *p )  
{  
    // code ...  
  if( p != NULL )  
    (*p)++;  
    // code ...  
}  
```
