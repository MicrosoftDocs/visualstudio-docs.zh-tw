# <a name="c6501"></a>C6501
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6501：批註衝突： \<name> 屬性與先前指定的屬性衝突  
  
> [!NOTE]
> 只有使用已淘汰版本的原始程式碼批註語言 (SAL) 的程式碼才會發生這個警告。 建議您移植程式碼，以使用最新版本的 SAL。 如需詳細資訊，請參閱 [使用 SAL 注釋減少 C/c + + 程式碼](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)缺失。  
  
 此警告表示批註中有衝突的屬性。 當有多個服務類似目的的屬性用來標注參數或傳回值時，通常就會發生這種情況。 若要更正警告，您必須選擇最能滿足您需求的屬性。  
  
## <a name="example"></a>範例  
 下列程式碼會產生此警告，因為 ValidElementsConst 和 ValidBytesConst 都提供允許讀取有效資料的機制：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void fd([SA_Pre(ValidElementsConst =4, ValidBytesConst =4)] char pch[]);  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f( [Pre(ValidElementsConst=4, ValidBytesConst=4 )] char pch[] );  
```  
  
 若要更正這個警告，請使用最適當的屬性，如下列程式碼所示：  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f( [SA_Pre(ValidElementsConst=4)] char pch[] );  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f( [Pre(ValidElementsConst=4)] char pch[] );  
```
