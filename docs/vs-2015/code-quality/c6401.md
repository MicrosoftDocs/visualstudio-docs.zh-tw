# <a name="c6401"></a>C6401
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6401： \<function name> 在預設地區設定中使用，以執行與常數位串不區分大小寫的比較 \< string name> 。 在非英文地區設定中產生非預期的結果  
  
 此警告表示在指定預設地區設定時，會執行與常數位串不區分大小寫的比較;通常，與地區設定無關的比較是預期的。  
  
 這項瑕疵的一般結果是非英文說話地區設定中不正確的行為。 例如，在土耳其文中，".gif" 將不會符合 "。GIF ";在越南文中，"LogIn" 將不會符合 "LOGIN"。  
  
 此函 `CompareString` 式會採用地區設定作為引數; 不過，傳遞預設地區設定（例如常數 `LOCALE_USER_DEFAULT` ）將會在不同的地區設定中造成不同的行為，視使用者的預設值而定。 通常，與常數位符串的不區分大小寫比較，應以與地區設定無關的比較來執行。  
  
 若要在 Windows XP 上執行與地區設定無關的比較 `CompareString` ，第一個參數應該是常數 `LOCALE_INVARIANT` ; 例如，若要執行與地區設定無關的測試，以判斷是否 `pString` 相符 file1.gif 忽略大寫/小寫差異，請使用如下的呼叫：  
  
```  
CompareString(LOCALE_INVARIANT,  
              NORM_IGNORECASE,  
              pString,  
              -1,  
              TEXT("file1.gif"),  
              -1) == CSTR_EQUAL   
```  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
include <windows.h>  
  
int fd(char *ext)  
{  
  return (CompareString(LOCALE_USER_DEFAULT,  
                        NORM_IGNORECASE,  
                        ext,   
                        -1,   
                        TEXT("gif"),  
                        -1) == 2);  
}  
```  
  
 若要更正這個警告，請使用下列程式碼：  
  
```  
include <windows.h>  
int f(char *ext)  
{  
  return (CompareString(LOCALE_INVARIANT,  
                        NORM_IGNORECASE,  
                        ext,   
                        -1,   
                        TEXT("gif"),  
                        -1) == 2);  
  
}  
```
