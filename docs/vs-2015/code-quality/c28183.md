# <a name="c28183"></a>C28183
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C28183：引數可能是一個值，而且是在指標中找到之值的複本。  
  
 此警告表示目前內容中的這個值未預期。 當值做為 `NULL` 引數傳遞至不允許的函式時，通常會出現此警告。 實際在指定的變數中找到此值，而引數是該變數的複本。  
  
 程式碼分析工具會在第一個點回報此警告，讓它可以明確判斷指標是， `NULL` 還是包含不合法的值。 不過，通常會發生錯誤，這通常會發生在追蹤中。 發生這種情況時，程式碼分析工具也會提供第一個可能實例的行號，通常位於無法明確判斷警告是否適當的位置。 在這些情況下，發生這種情況的先前位置會附加至警告訊息。 一般來說，程式碼變更應該在該行號的或之前發生，而不是在報告點發生。  
  
## <a name="example"></a>範例  
 在下列範例中，程式碼分析工具會判斷語句中的 *s* 為 Null `if` ，而且會採用的主體 `if` 。 然後指標 *會* 指派給 *t* ，然後 *t* 會以可進行取值的方式使用 `NULL` 。  
  
```  
#include <windows.h>  
  
int fun2(char *s)  
{  
    char *t;  
    if (s == NULL) {  
       //... but s is unchanged   
    }  
  
    t = s;  
  
    return lstrlenA(t);  
}  
```  
  
 在此程式碼範例中，程式碼分析工具會報告下列警告：  
  
 **d:\sample\testfile.cpp(38)：警告 C28183：'t' 可以是 '0'，而且是 's' 中所找到值的複本：這種情況並未遵守函式 'lstrlenA' 的規格：行：31、32、36、38。**
