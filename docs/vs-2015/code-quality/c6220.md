# <a name="c6220"></a>C6220
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6220-語意不相同的整數類型之間發生不當隱含轉換： 將 HRESULT 比作-1。 請考慮改用 SUCCEEDED 或 FAILED 巨集  
  
 這個警告指出`HRESULT`與其比較使用明確的非`HRESULT`-1，這不是語式正確的值`HRESULT`。 中的失敗`HRESULT`(`E_FAIL`) 不以-1。 因此，隱含轉型的`HRESULT`為整數會產生不正確的值，而可能導致錯誤的結果。  
  
## <a name="example"></a>範例  
 在大部分情況下，這項警告因錯誤地預期的程式碼的函式，應該會傳回`HRESULT`改為使用-1 做為錯誤值傳回整數。 下列程式碼範例會產生這個警告：  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == -1)  
  {  
    // failure code ...  
    return E_FAIL;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 建議您最好使用`SUCCEEDED`或是`FAILED`巨集來測試 windows 7 `HRESULT`。 若要更正這個警告，請使用下列程式碼：  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return E_FAIL;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 這個警告，`SCODE`類型相當於`HRESULT`。  
  
 若要檢查是否有特定的明確比較適合`HRESULT`值，例如， `E_FAIL`。 否則，請使用`SUCCEEDED`或`FAILED`巨集。  
  
 如需詳細資訊，請參閱 < [SUCCEEDED 巨集](http://go.microsoft.com/fwlink/?LinkId=92738)並[FAILED 巨集](http://go.microsoft.com/fwlink/?LinkId=180875)。  
  
 請注意，使用 malloc 和自由 （和相關的動態記憶體配置 Api） 方面的記憶體流失和例外狀況的許多問題。 若要避免這種流失和例外狀況的問題完全，使用所提供的機制C++標準範本庫 (STL)。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，並[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)並[ C++ Standard 程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。</c0>
