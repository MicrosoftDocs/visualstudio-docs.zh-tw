# <a name="c6011"></a>C6011
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6011：將 Null 指標取值 \<名稱 >  
  
 此警告表示正在取值 null 指標。 如果指標值無效，則結果會是未定義的。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告，因為如果沒有足夠的記憶體可用，則對 malloc 的呼叫可能會傳回 null：  
  
```  
#include <malloc.h>  
  
void f( )  
{   
  char *p = ( char * ) malloc( 10 );  
  *p = '\0';  
  
  // code ...  
 free( p );  
}  
```  
  
 若要修正這個警告，請檢查指標的 null 值，如下列程式碼所示：  
  
```  
#include <malloc.h>  
void f( )  
{  
  char *p = ( char * )malloc ( 10 );  
  if ( p )   
  {  
    *p = '\0';  
    // code ...  
  
    free( p );  
  }  
}  
```  
  
 您必須先在前置條件中使用 Null 屬性來配置參數的函式內的記憶體，然後再對參數進行引用。 下列程式碼會產生警告 C6011，因為在未先配置記憶體的情況下，會嘗試對函式內的 null 指標（`pc`）進行取值：  
  
```  
#include <sal.h>  
using namespace vc_attributes;  
void f([Pre(Null=Yes)] char* pc)  
{  
  *pc='\0'; // warning C6011 - pc is null  
  // code ...  
}  
```  
  
 在記憶體流失和例外狀況方面，使用 malloc 和 free 有許多陷阱。 若要避免這類流失和例外狀況問題，請使用C++標準範本程式庫（STL）提供的機制。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、 [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)和[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱[智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)和[ C++標準程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
