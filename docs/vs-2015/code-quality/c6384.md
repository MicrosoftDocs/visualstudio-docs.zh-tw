# <a name="c6384"></a>C6384
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6384：以另一個值除以指標  
  
 此警告表示大小計算可能不正確。 若要計算陣列中的元素數目，一個有時候會將陣列的大小除以第一個元素的大小;不過，當陣列實際上是指標時，結果通常會與預期的不同。  
  
 如果指標是函式參數，而且未傳遞緩衝區的大小，就無法計算可用的緩衝區上限。 當指標在本機配置時，應該使用配置中使用的大小。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, sizeof dest  / sizeof dest[0]);   
  }  
}  
```  
  
 若要修正這個警告，請傳遞緩衝區大小，如下列程式碼所示：  
  
```  
#include <windows.h>  
#include <TCHAR.h>  
  
#define SIZE 15  
  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
    _tcsncpy(dest, src, SIZE);  
  }  
}  
```  
  
 若要使用安全字串函數 _tcsncpy_s 更正這個警告，請使用下列程式碼：  
  
```  
void f( )  
{  
  LPTSTR dest = new TCHAR[SIZE];  
  char src [SIZE] = "Hello, World!!";  
  if (dest)  
  {  
   _tcsncpy_s(dest, SIZE, src, SIZE);  
  }  
}   
```  
  
 請注意，在記憶體流失和例外狀況方面，使用 new 和 delete 有許多陷阱。 若要避免這類流失和例外狀況問題，請使用C++標準範本程式庫（STL）提供的機制。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、 [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)和[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱[智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)和[ C++標準程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
