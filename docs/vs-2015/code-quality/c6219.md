# <a name="c6219"></a>C6219
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
警告 C6219:語意不相同的整數類型之間發生不當隱含轉換： 將 HRESULT 比作 1 或 TRUE。 請考慮使用[SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)或是[失敗](/windows/desktop/api/winerror/nf-winerror-failed)巨集改為

這則警告指出`HRESULT`與其比較使用明確的非`HRESULT`值一 (1)。 這項比較不可能導致不正確的結果，因為一般成功值`HRESULT`(`S_OK`) 為 0。 如果您比較此值，它會隱含地轉換為 false 布林型別。

## <a name="example"></a>範例

下列程式碼會產生這個警告，因為`CoGetMalloc`會傳回`HRESULT`，然後相較於`TRUE`:

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (hr == TRUE)
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

大部分的情況下，比較的程式碼造成這項警告`HRESULT`為布林值。 最好是使用`SUCCEEDED`或是`FAILED`巨集來測試 windows 7 `HRESULT`。 若要更正這個警告，請使用下列程式碼：

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

這個警告，`SCODE`型別會被視為`HRESULT`。

善用`malloc`和`free`（及相關的動態記憶體的 Api） 做為記憶體流失和例外狀況的原因有許多的陷阱。 若要避免這種流失和例外狀況的問題，請使用所提供的指標和容器類別C++標準程式庫。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，並[向量](/cpp/standard-library/vector)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](/cpp/cpp/smart-pointers-modern-cpp)並[ C++ Standard 程式庫](/cpp/standard-library/cpp-standard-library-reference)。</c0>
