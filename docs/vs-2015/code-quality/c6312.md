# <a name="c6312"></a>C6312
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6312：可能的無限迴圈：在 try-except 的例外狀況篩選條件運算式中使用常數 EXCEPTION_CONTINUE_EXECUTION  
  
 此警告表示在 `EXCEPTION_CONTINUE_EXECUTION` 結構化例外狀況處理常式的篩選條件運算式中，使用常數 (或另一個評估為-1) 的常數。 使用常數值 `EXCEPTION_CONTINUE_EXECUTION` 可能會導致無限迴圈。 例如，如果硬體引發例外狀況，則會重新開機造成例外狀況的指令。 如果造成例外狀況的位址仍然不正確，則會發生另一個例外狀況，並以相同的方式處理。 這會造成無限迴圈。  
  
 明確的呼叫 `RaiseException` 不會直接造成無限迴圈，但會繼續執行受保護區塊中的程式碼。 這可能是非預期的，如果 `RaiseException` 用來避免取值不正確指標，可能會導致無限迴圈。  
  
 通常， `EXCEPTION_CONTINUE_EXECUTION` 只有在篩選運算式中呼叫的函式才會傳回，這有機會修正造成例外狀況或基礎記憶體的指標。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告：  
  
```  
#include <excpt.h>  
#include <stdio.h>  
#include <windows.h>  
  
void f (char *ptr)  
{  
  __try   
  {  
    // exception occurs if the caller passes null ptr  
    // code...     
   *ptr = '\0';  
  }   
  __except (EXCEPTION_CONTINUE_EXECUTION)  
    // When EXCEPTION_CONTINUE_EXECUTION is used, the handler  
    //  block of the structured exception handler is not executed.   
  {  
    puts("This block is never executed");  
  }  
}  
```  
  
 若要更正這個警告，請使用下列程式碼：  
  
```  
#include <excpt.h>  
#include <stdio.h>  
#include <windows.h>  
  
void f (char *ptr)  
{  
  __try   
  {  
    // exception occurs if the caller passes null ptr  
    // code...  
    *ptr = '\0';  
  }   
  __except (GetExceptionCode()== EXCEPTION_ACCESS_VIOLATION ?   
               EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH)   
  {  
    puts("Error Occurred");  
  }  
}  
```
