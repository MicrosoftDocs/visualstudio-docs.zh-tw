# <a name="c6221"></a>C6221
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6221：語義不同整數類型之間的隱含轉換：比較 HRESULT 與整數。 請考慮改為使用 SUCCEEDED 或 FAILED 宏  
  
 此警告表示正在將 `HRESULT` 與零以外的整數進行比較。 成功的 `HRESULT` （`S_OK`）是以0表示。 因此，將 `HRESULT` 隱含轉換成整數會產生不正確的值，而且可能會導致錯誤的結果。 這通常是因為錯誤地預期函式在實際傳回 `HRESULT`時傳回整數。  
  
## <a name="example"></a>範例  
 下列程式碼會藉由比較 `HRESULT` 與整數值來產生此警告：  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == 4)  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 若要修正這個警告，下列程式碼會使用 `FAILED` 宏：  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 針對此警告，`SCODE` 類型相當於 `HRESULT`。  
  
 如需詳細資訊，請參閱[SUCCEEDED 宏](https://go.microsoft.com/fwlink/?LinkId=92738)和[FAILED 宏](https://go.microsoft.com/fwlink/?LinkId=180875)。  
  
 請注意，在記憶體流失和例外狀況方面，使用 malloc 和 free （和相關的動態記憶體配置 Api）有許多陷阱。 若要避免這類流失和例外狀況問題，請使用C++標準範本程式庫（STL）提供的機制。 這些機制包括 [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、[unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a) 和 [vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱[智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)和[ C++標準程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。
