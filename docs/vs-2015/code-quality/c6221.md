# <a name="c6221"></a>C6221
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6221:語意不相同的整數類型之間發生不當隱含轉換： 將 HRESULT 比作整數。 請考慮改用 SUCCEEDED 或 FAILED 巨集  
  
 這個警告指出`HRESULT`所比較為非零的整數。 在順利`HRESULT`(`S_OK`) 由為 0。 因此，隱含轉型的`HRESULT`為整數會產生不正確的值，而可能導致錯誤的結果。 它通常發生錯誤地預期函式來傳回整數，但實際上卻傳回`HRESULT`。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告，藉由比較`HRESULT`針對整數值：  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == 4)  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 若要更正這個警告，下列程式碼會使用`FAILED`巨集：  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 這個警告，`SCODE`類型相當於`HRESULT`。  
  
 如需詳細資訊，請參閱 < [SUCCEEDED 巨集](http://go.microsoft.com/fwlink/?LinkId=92738)並[FAILED 巨集](http://go.microsoft.com/fwlink/?LinkId=180875)。  
  
 請注意，使用 malloc 和自由 （和相關的動態記憶體配置 Api） 方面的記憶體流失和例外狀況的許多問題。 若要避免這種流失和例外狀況的問題完全，使用所提供的機制C++標準範本庫 (STL)。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，並[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)並[ C++ Standard 程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。</c0>
