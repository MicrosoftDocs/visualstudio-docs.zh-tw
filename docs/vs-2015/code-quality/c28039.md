# <a name="c28039"></a>C28039
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C28039：實際參數的類型 \<operand> 應該與類型完全相符 \<typename>  
  
 當列舉正式未傳遞列舉的成員，但也可以用於其他類型時，通常會報告此警告。  
  
 由於 C 可交換使用列舉，並且與常數交換，因此很容易就能將錯誤的列舉值傳遞至函式，而不會發生錯誤。  
  
 若是列舉型別，如果列舉參數的型別是以標注的 `_Enum_is_bitflag_` ，則可在參數上使用算術。 否則參數的類型必須完全正確。 如果常數是絕對必要的，則可能也適用警告 C28137。  
  
 這個規則也可以用於其他參數類型;請參閱函式檔，以瞭解類型必須完全相符的原因。
