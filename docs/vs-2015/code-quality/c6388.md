# <a name="c6388"></a>C6388
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6388:\<引數 > 可能無法\<的值 >： 這樣會違反函式的規格\<函式名稱 >:程式行： x y  
  
 這則警告指出未預期的值用於指定的內容。 這通常被報告未預期的函式當做引數傳遞的值。  
  
## <a name="example"></a>範例  
 下列C++程式碼會產生這個警告，因為 DoSomething 必須為 null 值，但可能會收到可能非 null 值：  
  
```cpp  
  
#include <string.h>  
#include <malloc.h>  
#include <sal.h>  
  
void DoSomething( _Pre_ _Null_ void* pReserved );  
  
void f()  
{  
    void* p = malloc( 10 );  
    DoSomething( p );  // Warning C6388  
    // code...  
    free(p);  
}  
```  
  
 若要更正這個警告，請使用下列的範例程式碼：  
  
```cpp  
  
#include <string.h>  
#include <malloc.h>  
#include <sal.h>  
  
void DoSomething( _Pre_ _Null_ void* pReserved );  
void f()  
{  
    void* p = malloc( 10 );  
    if (!p)  
    {  
        DoSomething( p );    
    }  
    else  
    {  
        // code...  
        free(p);  
    }  
}  
```  
  
 請注意使用 malloc 和自由，有許多方面的記憶體流失和例外狀況的陷阱。 若要避免這種流失和例外狀況的問題完全，使用所提供的機制C++標準範本庫 (STL)。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，並[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)並[ C++ Standard 程式庫](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。</c0>
