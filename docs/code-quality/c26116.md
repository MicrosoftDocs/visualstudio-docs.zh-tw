# <a name="c26116"></a>C26116
警告 C26116：無法在 \<func > 中取得或保留鎖定 \<lock >。

 語言不會執行 C/C++ program 中的語法範圍鎖定*取得*和鎖定*釋放*配對。 函式可能會藉由對並行狀態進行可觀察的修改來引入鎖定副作用。 例如，鎖定包裝函式會針對指定的鎖定，遞增鎖定數目或鎖定計數。您可以分別使用 `_Acquires_lock_` 或 `_Requires_lock_held`，標注具有鎖定取得或鎖定釋放副作用的函式。 如果沒有這類注釋，函式就不會在傳回之後變更任何鎖定計數。 如果取得和發行未平衡，則視為*孤立*。 當函式已標注 `_Acquires_lock_`，但未取得鎖定，或以 `_Requires_lock_held` 標注函式並釋放鎖定時，就會發出警告 C26116。

## <a name="example"></a>範例
 下列範例會產生警告 C26116，因為函數 `DoesNotLock` 已標注 `_Acquires_lock_`，但未取得。 函式 `DoesNotHoldLock` 會產生警告，因為它是以 `_Requires_lock_held` 標注，而且不會保存它。

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

_Acquires_lock_(p->cs) void DoesLock(DATA* p)
{
    EnterCriticalSection(&p->cs); // OK
}

_Acquires_lock_(p->cs) void DoesNotLock(DATA* p)
{
    // Warning C26116
}

_Requires_lock_held_(p->cs) void DoesNotHoldLock(DATA* p)
{
    LeaveCriticalSection(&p->cs); // Warning C26116
}
```
