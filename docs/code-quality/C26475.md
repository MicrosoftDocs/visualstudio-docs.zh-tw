# <a name="c26475-no_function_style_casts"></a>C26475 NO_FUNCTION_STYLE_CASTS

「不要使用函式樣式 C 轉換」。

核心指導方針： **C++** ES. 49：如果您必須使用轉換，請使用已命名的 cast

函式樣式轉換（例如，`int(1.1)`）是 C 樣式轉型的另一個化身（例如 `(int)1.1`），這類轉換具有可疑的安全性。 具體而言，編譯器不會嘗試檢查是否有任何資料遺失發生在 C 轉換或函數轉換中。 在這兩種情況下，最好是盡可能避免轉換或使用大括弧初始化運算式。 如果沒有作用，靜態轉換可能很適合，但最好是使用指導方針支援程式庫中的公用程式：

- gsl：：窄幅可確保不失真的轉換，並會導致執行時間損毀（如果不可行）。
- gsl：： narrow_cast 清楚指出轉換可能會遺失資料，而且是可接受的。

## <a name="remarks"></a>備註

- 此規則只會針對基本類型的常數引發。 在使用大括弧初始化運算式的情況下，編譯器可以清楚地偵測資料遺失併發出錯誤。 需要執行時間執行的情況會以 C26493 NO_CSTYLE_CAST 來標示。
- 預設初始化運算式不會加上旗標（例如 `int()`）。

## <a name="example"></a>範例

危險的轉換

```cpp
constexpr auto planck_constant = float( 6.62607004082e-34 ); // C26475
```

```cpp
dangerous conversion – detecting potential data loss
constexpr auto planck_constant = float{ 6.62607004082e-34 }; // Error C2397
```

```cpp
dangerous conversion – corrected
constexpr auto planck_constant = double{ 6.62607004082e-34 };
```
