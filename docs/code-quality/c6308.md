# <a name="c6308"></a>C6308
警告 C6308: 'realloc' 可能會傳回 null 指標： 指派了 null 指標給\<變數 >，做為引數傳遞至 'realloc'，會造成原始記憶體區塊遺漏

 這則警告表示記憶體流失，是使用不正確的配置函式的結果。 堆積配置函式請勿釋放的傳遞的緩衝區，如果重新配置失敗。 若要更正該缺失，重新配置函式的結果指派給暫存，並取代原始指標重新配置成功之後。

## <a name="example"></a>範例
 下列範例程式碼會產生這個警告：

```cpp
#include <malloc.h>
#include <windows.h>

void f( )
{
  char *x;
  x = (char *) malloc(10);
  if (x != NULL)
  {
    x = (char *) realloc(x, 512);
    // code...
    free(x);
  }
}
```

 若要更正這個警告，請使用下列程式碼：

```cpp
#include <malloc.h>
#include <windows.h>

void f()
{
  char *x, *tmp;

  x = (char *) malloc(10);

  if (x != NULL)
  {
    tmp = (char *) realloc(x,512);
    if (tmp != NULL)
    {
      x = tmp;
    }
    free(x);
  }
}
```

 這個警告可能會產生雜訊，如果當時重新配置函式的結果指派的即時的別名，以配置緩衝區。

 若要完全避免這類的問題，請使用所提供的 c + + 標準範本庫 (STL) 的機制。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，並[向量](/cpp/standard-library/vector)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](/cpp/cpp/smart-pointers-modern-cpp)並[c + + 標準程式庫](/cpp/standard-library/cpp-standard-library-reference)。
