# <a name="ca5402-use-createencryptor-with-the-default-iv"></a>CA5402：使用 CreateEncryptor 搭配預設的 IV

|||
|-|-|
|CheckId|CA5402|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

`rgbIV`使用時，可能為非預設值 <xref:System.Security.Cryptography.SymmetricAlgorithm.CreateEncryptor%2A?displayProperty=fullName> 。

## <a name="rule-description"></a>規則描述

對稱式加密應一律使用不可重複的初始化向量來防止字典攻擊。

此規則類似于[CA5401](ca5401.md)，但分析無法判斷初始化向量是否絕對為預設值。

## <a name="how-to-fix-violations"></a>如何修正違規

明確使用預設 `rgbIV` 值，也就是使用 <xref:System.Security.Cryptography.SymmetricAlgorithm.CreateEncryptor%2A?displayProperty=fullName> 沒有任何參數的多載。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

在下列情況下，您可以放心地隱藏此規則中的警告：

- `rgbIV`參數是由所產生 <xref:System.Security.Cryptography.SymmetricAlgorithm.GenerateIV%2A?displayProperty=fullName> 。
- 您確定 `rgbIV` 參數確實是隨機且不可重複的。
- 您一定要使用初始化向量。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

```csharp
using System;
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod(byte[] rgbIV)
    {
        AesCng aesCng  = new AesCng();
        Random r = new Random();

        if (r.Next(6) == 4)
        {
            aesCng.IV = rgbIV;
        }

        aesCng.CreateEncryptor();
    }
}
```

### <a name="solution"></a>解決方法

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod()
    {
        AesCng aesCng  = new AesCng();
        aesCng.CreateEncryptor();
    }
}
```
