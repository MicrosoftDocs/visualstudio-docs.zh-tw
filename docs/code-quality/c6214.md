# <a name="c6214"></a>C6214

警告 C6214：在語義不同的整數類型之間轉換： HRESULT 為布林值類型

此警告表示正在將 `HRESULT` 轉換成布林值類型。 @No__t-1 等於0的成功值（`S_OK`）。 不過，0表示布林類型的失敗。 將 `HRESULT` 轉換成布林值類型，然後在測試運算式中使用它，會產生不正確的結果。 有時候，如果 `HRESULT` 是儲存在布林值變數中，就會發生此警告。 使用布林值變數來測試 @no__t 0 成功或失敗的任何比較，可能會導致不正確的結果。

## <a name="example"></a>範例

下列程式碼會產生這個警告：

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);
  if ((BOOL)hr) // warning 6214
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

若要修正這個警告，請使用下列程式碼：

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (SUCCEEDED(hr))
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

針對此警告，`SCODE` 類型相當於 `HRESULT`。

通常應該使用 `SUCCEEDED` 或 @no__t 1 宏來測試 `HRESULT` 的值。
