# <a name="ca5399-definitely-disable-httpclient-certificate-revocation-list-check"></a>CA5399：明確停用 HttpClient 憑證撤銷清單檢查

|||
|-|-|
|CheckId|CA5399|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

使用 <xref:System.Net.Http.HttpClient?displayProperty=fullName> 時，如果提供的平臺特定處理常式（ <xref:System.Net.Http.WinHttpHandler?displayProperty=fullName> 或 <xref:System.Net.Http.HttpClientHandler?displayProperty=fullName> ）的 `CheckCertificateRevocationList` 屬性未設定為，則 `true` 會允許將被視為有效的已撤銷憑證 <xref:System.Net.Http.HttpClient> 。

此規則類似于[CA5400](ca5400.md)，但分析可以判斷 `CheckCertificateRevocationList` 屬性是明確 `false` 或未設定。

## <a name="rule-description"></a>規則描述

已撤銷的憑證已不再受信任。 攻擊者可以使用它來傳遞某些惡意資料，或竊取 HTTPS 通訊中的敏感性資料。

## <a name="how-to-fix-violations"></a>如何修正違規

將 `CheckCertificateRevocationList` 屬性設為 `true`。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則。

## <a name="configurability"></a>可設定性

如果您是從[FxCop 分析器](install-fxcop-analyzers.md)執行此規則（而不是使用舊版分析），此規則是可設定的。

### <a name="excluded-symbol-names"></a>排除的符號名稱

您可以設定程式碼基底中要排除的部分，使其不進行分析。 例如，若要指定規則不應在名為之型別內的任何程式碼上執行 `MyType` ，請將下列機碼值組新增至專案中的 editorconfig 檔案：

```ini
dotnet_code_quality.CA5399.excluded_symbol_names = MyType
```

選項值中允許的符號名稱格式（以 ' | ' 分隔）：
  - 僅限符號名稱（包含所有具有名稱的符號，不論包含的類型或命名空間為何）
  - 符號的[檔識別碼格式](https://github.com/dotnet/csharplang/blob/master/spec/documentation-comments.md#id-string-format)中的完整名稱。 每個符號名稱都需要符號種類前置詞，例如 "M：" 前置詞用於方法，"T：" 首碼用於類型，"N：" 前置詞用於命名空間等等。
  - `.ctor`適用于函式和 `.cctor` 靜態的函式

範例：

| 選項值 | 摘要 |
| --- | --- |
|`dotnet_code_quality.CA5399.excluded_symbol_names = MyType` | 符合編譯中名為 ' MyType ' 的所有符號
|`dotnet_code_quality.CA5399.excluded_symbol_names = MyType1|MyType2` | 比對在編譯中名為 ' MyType1 ' 或 ' MyType2 ' 的所有符號
|`dotnet_code_quality.CA5399.excluded_symbol_names = M:NS.MyType.MyMethod(ParamType)` | 與指定的完整簽章相符的特定方法 ' MyMethod '
|`dotnet_code_quality.CA5399.excluded_symbol_names = M:NS1.MyType1.MyMethod1(ParamType)|M:NS2.MyType2.MyMethod2(ParamType)` | 將特定方法 ' MyMethod1 ' 和 ' MyMethod2 ' 與個別的完整簽章進行比對

您可以只針對此規則、所有規則，或此類別中的所有規則（安全性），設定所有這些選項。 如需詳細資訊，請參閱[設定 FxCop 分析器](configure-fxcop-analyzers.md)。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

```csharp
using System.Net.Http;

class ExampleClass
{
    void ExampleMethod()
    {
        WinHttpHandler winHttpHandler = new WinHttpHandler();
        winHttpHandler.CheckCertificateRevocationList = false;
        HttpClient httpClient = new HttpClient(winHttpHandler);
    }
}
```

### <a name="solution"></a>解決方法

```csharp
using System.Net.Http;

class ExampleClass
{
    void ExampleMethod()
    {
        WinHttpHandler winHttpHandler = new WinHttpHandler();
        winHttpHandler.CheckCertificateRevocationList = true;
        HttpClient httpClient = new HttpClient(winHttpHandler);
    }
}
```
