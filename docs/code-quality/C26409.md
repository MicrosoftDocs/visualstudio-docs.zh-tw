# <a name="c26409-nonewdelete"></a>C26409 NO_NEW_DELETE
即使程式碼是乾淨的 malloc 和 free （） 呼叫我們仍建議您考慮較佳選項比明確使用的運算子[新增和刪除](/cpp/cpp/new-and-delete-operators)。

  **C++核心的指導方針**:[R.11:避免新的呼叫，並明確刪除](https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines#r11-avoid-calling-new-and-delete-explicitly)

最終的修正方法是開始使用適當的 factory 函式，使用智慧型指標，例如[std](/cpp/standard-library/memory-functions#make_unique)。

## <a name="remarks"></a>備註
- 「 檢查 」 警告上呼叫任何一種運算子`new`或`delete`： 純量，向量，（全域和類別的特定） 的多載版本，以及在放置版本上。 後者的情況下可能需要一些說明建議的修正方面的核心指南，並且在未來可能會省略。
