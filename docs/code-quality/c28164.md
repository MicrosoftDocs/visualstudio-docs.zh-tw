# <a name="c28164"></a>C28164
警告 C28164：引數傳遞至預期物件指標的函式（不是指標指標）

 當指標呼叫的函式需要物件的指標時，就會報告這個警告。

 函式會在此位置取得 PVOID。 通常，這表示在需要 p*xxx*時，會使用 & p*xxx* 。

 某些多型函式（可以評估為並套用至不同類型的值的函式）會使用採用任何指標類型的 PVOID 引數，在 C 中實*作為函數。* 不過，這可讓程式設計人員撰寫指標的程式碼，而不會造成編譯器錯誤，即使這個類型不適用也一樣。

## <a name="example"></a>範例
 下列程式碼範例會產生此警告：

```
PFAST_MUTEX pFm;
...
KeWaitForSingleObject(&pFm, UserRequest, UserMode, false, NULL);
```

 下列程式碼範例會避免出現警告：

```
PFAST_MUTEX pFm;
...
KeWaitForSingleObject(pFm, UserRequest, UserMode, false, NULL);
```
