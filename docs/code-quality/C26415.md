# <a name="c26415-smartptrnotneeded"></a>C26415 SMART_PTR_NOT_NEEDED
「 智慧型指標參數來只存取包含指標。 使用 T * 或 T& 改用。 」

**C + + Core Guidelines**: R.30： 採用智慧型指標做為參數，才能明確地快速的存留期語意

使用智慧型指標類型，將資料傳遞至函式，以指出目標函式需要管理所包含物件的存留期。 不過，如果函式只會使用智慧型指標，存取所包含的物件，並不會實際呼叫可能會導致其解除配置的任何程式碼 （永遠不會影響其存留期），通常就不需要更加複雜的智慧型指標的介面。 應該是慣用的純文字的指標或參考所包含的物件。

## <a name="remarks"></a>備註
- 這項檢查是也會導致 C26410、 C26415、 C26417 和 C26418 涵蓋大部分的案例。 它會更好清除 SMART_PTR_NOT_NEEDED 第一次，然後切換到共用或唯一指標的邊緣案例。 針對多個著重清除這警告可以停用。
  - 除了標準 std::unqiue_pointer 和 std::shared_pointer 範本，這項檢查可辨識使用者定義型別，這可能要作為智慧型指標。 這種型別會定義下列作業：
  - 多載取值 （dereference） 或成員存取運算子，會是 public 且未標示為已刪除;
  - 公用解構函式的不會刪除或預設值。 這包括解構函式明確地定義空的。
  - 解譯的作業，可能會影響包含物件的存留期相當廣泛，而且包含：
  - 它會接受非常數的智慧型指標; 指標或參考參數的任何函式
  - 複製或移動建構函式或指派運算子;
  - 非常數的函式。
    ## <a name="example"></a>範例
    麻煩的存留期管理

```cpp
bool set_initial_message(
            const std::unique_ptr<message> &m) // C26415, also C26410 NO_REF_TO_CONST_UNIQUE_PTR
{
    if (!m || initial_message_)
        return false;

    initial_message_.reset(m.get());
    return true;
}

void pass_message(const message_info &info)
{
    auto m = std::make_unique<message>(info);
    const auto release = set_initial_message(m);
    // ...
    if (release)
        m.release();
}
```

## <a name="example"></a>範例
麻煩的存留期管理-修改

```cpp
void set_initial_message(std::shared_ptr<message> m) noexcept
{
    if (m && !initial_message_)
        initial_message_ = std::move(m);
}

void pass_message(const message_info &info)
{
    auto m = std::make_shared<message>(info);
    set_initial_message(m);
    // ...
}
```
