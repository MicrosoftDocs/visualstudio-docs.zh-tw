# <a name="c6226"></a>C6226

警告 C6226:語意不相同的整數類型之間發生不當隱含轉換： 將-1 指派給 HRESULT。 請考慮改用 E_FAIL。

這個警告指出`HRESULT`指派或初始化為明確的值為-1。 這個警告通常因為意外造成混淆的整數和`HRESULT`型別。 若要表示作業成功，使用符號常數`S_OK`改。 若要指出失敗，請使用 開頭 E_ 符號常數*常數*，例如`E_FAIL`。

如需詳細資訊，請參閱 < [SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)並[失敗](/windows/desktop/api/winerror/nf-winerror-failed)巨集。

## <a name="example"></a>範例

下列程式碼會產生這個警告：

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

若要更正這個警告，請使用下列程式碼：

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

這個警告，`SCODE`型別會被視為`HRESULT`。

善用`malloc`和`free`（及相關的動態記憶體的 Api） 做為記憶體流失和例外狀況的原因有許多的陷阱。 若要避免這種流失和例外狀況的問題，請使用所提供的指標和容器類別C++標準程式庫。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，並[向量](/cpp/standard-library/vector)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](/cpp/cpp/smart-pointers-modern-cpp)並[ C++ Standard 程式庫](/cpp/standard-library/cpp-standard-library-reference)。</c0>
