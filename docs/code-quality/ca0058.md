# <a name="ca0058"></a>CA0058

無法載入參考的元件。

如果分析工具無法在分析下載入元件所參考的元件，就會發生這個錯誤。 因此，此錯誤可能會導致 CA0001 在其他位置，因為如果發生這種情況，可能會產生許多非預期的狀態。 如果您從 Visual Studio 內使用 managed 程式碼分析工具，以下是可能造成 CA0058 的一些原因：

- 已覆寫輸入元件，而現在指向的元件會參考 Visual Studio 專案檔中參考清單以外的其他元件。

- 如果在 Fxcopcmd.exe 執行之前修改專案以叫用自訂群組建步驟，且自訂群組建程式新增更多對輸入元件的參考，則會產生錯誤 CA0058。

- 在C++中，您可以使用 VCPROJ 中的: vcproj forcedusing 來參考元件，覆寫命令列以傳遞/: vcproj forcedusing，或在來源中新增 #using \<Some .dll >。

在所有情況下，此問題的解決方法是將遺漏的參考新增至 Visual Studio 專案本身。
