# <a name="ca2015-do-not-define-finalizers-for-types-derived-from-memorymanagerlttgt"></a>CA2015：請勿針對衍生自 MemoryManager T 的類型定義完成項 &lt;&gt;

|Item|值|
|-|-|
|CheckId|CA2015|
|類別|Microsoft 可靠性|
|重大變更|不中斷|

## <a name="cause"></a>原因

定義衍生自之類型的完成項 <xref:System.Buffers.MemoryManager%601>

## <a name="rule-description"></a>規則描述

將完成項加入至衍生自的類型 <xref:System.Buffers.MemoryManager%601> 可能是 bug 的表示，因為它會建議可能已在中使用的原生資源 <xref:System.Span%601> 正在清除，而且可能會在仍由使用中 <xref:System.Span%601> 。

> [!NOTE]
> <xref:System.Buffers.MemoryManager%601>類別適用于先進的案例。 大部分的開發人員都不需要使用它。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請移除完成項定義。

```csharp
class DerivedClass <T> : MemoryManager<T>
{
    public override bool Dispose(bool disposing)
    {
        if (disposing)
        {
            _handle.Dispose();
        }
    }

    ...

    // Violation occurs, remove the finalizer to fix the warning.
    ~DerivedClass() => Dispose(false);
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果目的是要為偵錯工具或驗證目的建立完成項，則可以隱藏此規則的違規。
