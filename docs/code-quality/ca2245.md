# <a name="ca2245-do-not-assign-a-property-to-itself"></a>CA2245：請勿將屬性指派給屬性自身

|Item|值|
|-|-|
|CheckId|CA2245|
|類別|Microsoft。使用方式|
|重大變更|不中斷|

## <a name="cause"></a>原因

屬性不小心指派給本身。

## <a name="rule-description"></a>規則描述

C # 編譯器會產生警告 CS1717：指派給相同的變數; 當欄位、本機或參數的符號指派給本身時，[您是否表示指派其他專案](/dotnet/csharp/misc/cs1717)。 當本機、參數或欄位符號的名稱類似範圍中的另一個符號時，這種錯誤很常見。 並不是在指派的左側和右手邊使用不同的符號，而是在兩端使用相同的符號。 這會導致將值的多餘指派給本身，通常表示功能錯誤。

將屬性指派給本身，也是幾乎所有真實世界案例的類似功能錯誤。 不過，在某些極端的情況下，提取屬性值可能會有副作用，而屬性的新值則與原始值不同。 若是如此，屬性的自我指派並不是多餘的，而且無法移除。 這可防止編譯器產生 `CS1717` 屬性自我指派的警告，而不會針對這些情況引進中斷性變更。

規則 `CA2245` 的目的是要填補此缺口。 它會報告屬性自我指派的違規，以協助修正這些功能錯誤。 對於需要自行指派屬性的一小組角落案例， `CA2245` 可以在來源中隱藏違規，並具有適當的對齊批註。

> [!NOTE]
> 舊版分析無法使用規則 CA2245。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.9.8 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規情形，請在指派的左側和右手邊使用不同的符號。 例如，下列程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
public class C
{
    private int p = 0;
    public int P { get; private set; }

    public void M(int p)
    {
        // CS1717: Accidentally assigned the parameter 'p' to itself.
        p = p;

        // CA2245: Accidentally assigned the property 'P' to itself.
        P = P;
    }
}
```

```csharp
public class C
{
    private int p = 0;
    public int P { get; private set; }

    public void M(int p)
    {
        // No violation, now the parameter is assigned to the field.
        this.p = p;

        // No violation, now the parameter is assigned to the property.
        P = p;
    }
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果提取屬性值可能有副作用，而且屬性的新值與原始值不同，就可以隱藏此規則的違規。 若是如此，屬性的自我指派並不會重複。 應該將對齊批註加入至隱藏專案，以將此專案記錄為預期的行為。
