# <a name="ca1838-avoid-stringbuilder-parameters-for-pinvokes"></a>CA1838：避免 `StringBuilder` P/invoke 的參數

|Item|值|
|-|-|
|CheckId|CA1838|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

[P/Invoke](/dotnet/standard/native-interop/pinvoke)具有 <xref:System.Text.StringBuilder> 參數。

## <a name="rule-description"></a>規則描述

的封送處理 `StringBuilder` 一律會建立原生緩衝區複本，導致一個 P/Invoke 呼叫有多個配置。 若要封送處理 `StringBuilder` 做為 P/Invoke 參數，執行時間將會：
- 配置原生緩衝區
- 如果它是 `In` 參數，請將的內容複寫 `StringBuilder` 到原生緩衝區
- 如果它是 `Out` 參數，請將原生緩衝區複製到新配置的受控陣列

根據預設， `StringBuilder` 是 `In` 和 `Out` 。

預設會停用此規則，因為它可能需要以案例的方式分析違規是否感意義，以及可能的非一般重整來解決違規。 使用者可以藉由設定 [分析器規則的嚴重性](use-roslyn-analyzers.md#rule-severity)，明確啟用此規則。

## <a name="how-to-fix-violations"></a>如何修正違規

一般來說，定址違規牽涉到修改 P/Invoke 和其呼叫者使用緩衝區，而不是 `StringBuilder` 。 細節取決於 P/Invoke 的使用案例。

以下是使用 `StringBuilder` 做為輸出緩衝區以供原生函式填入之常見案例的範例：

```csharp
// Violation
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern void Foo(StringBuilder sb, ref int length);

public void Bar()
{
    int BufferSize = ...
    StringBuilder sb = new StringBuilder(BufferSize);
    int len = sb.Capacity;
    Foo(sb, ref len);
    string result = sb.ToString();
}
```

對於緩衝區較小且 `unsafe` 可接受程式碼的使用案例， [stackalloc](/dotnet/csharp/language-reference/operators/stackalloc) 可以用來在堆疊上配置緩衝區：

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern unsafe void Foo(char* buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    unsafe
    {
        char* buffer = stackalloc char[BufferSize];
        int len = BufferSize;
        Foo(buffer, ref len);
        string result = new string(buffer);
    }
}
```

對於較大的緩衝區，可以將新的陣列配置為緩衝區：

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern void Foo([Out] char[] buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    char[] buffer = new char[BufferSize];
    int len = buffer.Length;
    Foo(buffer, ref len);
    string result = new string(buffer);
}
```

經常針對較大的緩衝區呼叫 P/Invoke 時， <xref:System.Buffers.ArrayPool%601> 可以用來避免其所附的重複配置和記憶體壓力：

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Unicode)]
private static extern unsafe void Foo([Out] char[] buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    char[] buffer = ArrayPool<char>.Shared.Rent(BufferSize);
    try
    {
        int len = buffer.Length;
        Foo(buffer, ref len);
        string result = new string(buffer);
    }
    finally
    {
        ArrayPool<char>.Shared.Return(buffer);
    }
}
```

如果在執行時間之前不知道緩衝區大小，可能需要根據大小以不同的方式建立緩衝區，以避免使用來配置大型緩衝區 `stackalloc` 。

前面的範例會使用2個位元組的寬字元 (`CharSet.Unicode`) 。 如果原生函式 () 使用1位元組字元 `CharSet.Ansi` ，則 `byte` 可以使用緩衝區，而不是 `char` 緩衝區。 例如：

```csharp
[DllImport("MyLibrary", CharSet = CharSet.Ansi)]
private static extern unsafe void Foo(byte* buffer, ref int length);

public void Bar()
{
    int BufferSize = ...
    unsafe
    {
        byte* buffer = stackalloc byte[BufferSize];
        int len = BufferSize;
        Foo(buffer, ref len);
        string result = Marshal.PtrToStringAnsi((IntPtr)buffer);
    }
}
```

如果參數也當做輸入使用，則必須以明確新增任何 null 結束字元的字串資料填入緩衝區。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意封送處理的效能影響，請隱藏此規則的違規 `StringBuilder` 。
