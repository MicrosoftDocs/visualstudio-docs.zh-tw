# <a name="ca1832-use-asspan-or-asmemory-instead-of-range-based-indexers-for-getting-readonlyspan-or-readonlymemory-portion-of-an-array"></a>CA1832：使用 AsSpan 或 AsMemory，而不是以範圍為基礎的索引子，以取得陣列的 ReadOnlySpan 或 ReadOnlyMemory 部分

|||
|-|-|
|CheckId|CA1832|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

在陣列上使用範圍索引子，並隱含地將值指派給 <xref:System.ReadOnlySpan%601> 或時 <xref:System.ReadOnlyMemory%601> 。

## <a name="rule-description"></a>規則描述

在陣列上使用範圍索引子並指派給記憶體或範圍類型：上的範圍索引子 <xref:System.Span%601> 是非複製作業 <xref:System.Span%601.Slice%2A?#System_Span_1_Slice_System_Int32_System_Int32_> ，但在陣列的範圍索引子上， <xref:System.Runtime.CompilerServices.RuntimeHelpers.GetSubArray%2A> 會使用方法，而不是 <xref:System.Span%601.Slice%2A?#System_Span_1_Slice_System_Int32_System_Int32_> ，這會產生陣列所要求部分的複本。 當此複製隱含當做或值使用時，通常不需要這麼做 <xref:System.ReadOnlySpan%601> <xref:System.ReadOnlyMemory%601> 。 如果不打算複製，請使用 <xref:System.MemoryExtensions.AsSpan%2A?#System_MemoryExtensions_AsSpan__1___0___> 或 <xref:System.MemoryExtensions.AsMemory%2A?#System_MemoryExtensions_AsMemory__1___0___> 方法來避免不必要的複製。 如果是預定的複本，請先將它指派給區域變數，或加入明確轉換。 只有在範圍索引子作業的結果上使用隱含轉換時，analyzer 才會報告。

### <a name="detects"></a>檢測

隱含轉換：
- `ReadOnlySpan<SomeT> slice = arr[a..b];`
- `ReadOnlyMemory<SomeT> slice = arr[a..b];`

### <a name="does-not-detect"></a>未偵測到

明確轉換：<br>
- `ReadOnlySpan<SomeT> slice = (ReadOnlySpan<SomeT>)arr[a..b];`
- `ReadOnlyMemory<SomeT> slice = (ReadOnlyMemory<SomeT>)arr[a..b];`

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正此規則的違規：請使用 <xref:System.MemoryExtensions.AsSpan%2A?#System_MemoryExtensions_AsSpan__1___0___> 或 <xref:System.MemoryExtensions.AsMemory%2A?#System_MemoryExtensions_AsMemory__1___0___> 擴充方法，以避免建立不必要的資料複本。

```csharp
class C
{
    public void TestMethod(byte[] arr)
    {
        // The violation occurs for both statements below
        ReadOnlySpan<byte> tmp1 = arr[0..2];
        ReadOnlyMemory<byte> tmp3 = arr[5..8];
        ...
    }
}
```

```csharp
class C
{
    public void TestMethod(byte[] arr)
    {
        // The violations fixed with AsSpan or AsMemory accordingly
        ReadOnlySpan<byte> tmp1 = arr.AsSpan()[0..2];
        ReadOnlyMemory<byte> tmp3 = arr.AsMemory()[5..8];
        ...
    }
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在陣列違規上，然後按**Ctrl** + **。** (句點)。 從顯示的選項清單中選擇 [**使用 AsSpan]，而不是陣列上以範圍為基礎的索引子**。
>
> ![CA1832 的程式碼修正-使用 AsSpan 或 AsMemory，而不是以範圍為基礎的索引子來取得陣列的 ReadOnlySpan 或 ReadOnlyMemory 部分](media/ca1832_codefix.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您想要建立複本，則可以放心抑制此規則的違規。 若要隱藏這個警告，您可以直接加入明確轉換。

```csharp
class C
{
    public void TestMethod(byte arr[])
    {
        // The violation occurs
        ReadOnlySpan<byte> tmp1 = arr[0..2];
        ReadOnlyMemory<byte> tmp3 = arr[5..8];
        ...
    }
}
```

```csharp
class C
{
    public void TestMethod(byte arr[])
    {
        // The violation fixed with explicit casting
        ReadOnlySpan<byte> tmp1 = (ReadOnlySpan<byte>)arr[0..2];
        ReadOnlyMemory<byte> tmp3 = (ReadOnlyMemory<byte>)arr[5..8];
        ...
    }
}
```
