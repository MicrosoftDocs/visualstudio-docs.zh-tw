# <a name="ca1712-do-not-prefix-enum-values-with-type-name"></a>CA1712：不要使用類型名稱做為列舉值的前置字元

|||
|-|-|
|TypeName|DoNotPrefixEnumValuesWithTypeName|
|CheckId|CA1712|
|類別|微軟.命名|
|重大變更|中斷|

## <a name="cause"></a>原因
枚舉包含名稱以枚舉的類型名稱開頭的成員。

## <a name="rule-description"></a>規則描述
枚舉成員的名稱不會用類型名稱預綴，因為類型資訊應由開發工具提供。

命名約定為面向通用語言運行時的庫提供了一個常見外觀。 這減少了學習新軟體庫所需的時間，並提高了客戶的信心，即該庫是由具有開發託管代碼的專業知識的人員開發的。

## <a name="how-to-fix-violations"></a>如何修正違規
要修復違反此規則的問題，請從枚舉成員中刪除類型名稱首碼。

## <a name="when-to-suppress-warnings"></a>何時禁止警告
請勿隱藏此規則的警告。

## <a name="configurability"></a>可配置性

如果從[FxCop 分析器](install-fxcop-analyzers.md)運行此規則（而不是使用舊版分析），則可以為此規則配置分析。

您可以配置觸發規則所需的枚舉值數。 例如，要指定在一個或多個枚舉值以枚舉類型名稱開頭時觸發規則，請在專案中將以下鍵值對添加到 *.editorconfig*檔中：

```ini
dotnet_code_quality.CA1712.enum_values_prefix_trigger = AnyEnumValue
```

範例：

| 選項值 | 摘要 |
| --- | --- |
|`dotnet_code_quality.CA1712.enum_values_prefix_trigger = AnyEnumValue` | 如果*任何*枚舉值以枚舉類型名稱開頭，則觸發規則。
|`dotnet_code_quality.CA1712.enum_values_prefix_trigger = AllEnumValues` | 如果*所有*枚舉值都以枚舉類型名稱開頭，則觸發規則。
|`dotnet_code_quality.CA1712.enum_values_prefix_trigger = Heuristic` | 規則使用預設的 FxCop 啟發式觸發（即，當至少 75% 的枚舉值以枚舉類型名稱開頭時）。

您可以僅為此規則、所有規則或此類別中的所有規則（命名）配置所有這些選項。 有關詳細資訊，請參閱配置[FxCop 分析器](configure-fxcop-analyzers.md)。
