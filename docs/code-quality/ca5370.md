# <a name="ca5370-use-xmlreader-for-validating-reader"></a>CA5370：請使用 XmlReader 驗證讀取器

|||
|-|-|
|CheckId|CA5370|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

使用不含物件的類別來驗證不受信任的 XML 輸入 `XmlValidatingReader` `XmlReader` ，可能會導致拒絕服務、資訊洩漏和伺服器端偽造要求。 這些攻擊是由不受信任的 DTD 和 XML 架構處理啟用，可讓您在 XML 中包含 XML 炸彈和惡意的外部實體。 只有 with `XmlReader` 才能夠停用 DTD。 以為基礎的內嵌 XML 架構處理， `XmlReader` `ProhibitDtd` 預設會 `ProcessInlineSchema` 將和屬性設定為， `false` .NET Framework 從版本4.0 開始。

## <a name="rule-description"></a>規則描述

處理不受信任的 DTD 和 XML 架構可能會啟用載入危險的外部參考。 這項危險的載入可以藉由使用 `XmlReader` 安全解析程式或停用 DTD 和 XML 內嵌架構處理來加以限制。 此規則會偵測使用類別的程式碼， `XmlValidatingReader` 但不含 `XmlReader` 做為函式參數。

## <a name="how-to-fix-violations"></a>如何修正違規

- 使用 `XmlValidatingReader(XmlReader)` `ProhibitDtd` ，並 `ProcessInlineSchema` 將屬性設定為 `false` 。
- 從 .NET Framework 2.0 開始， `XmlValidatingReader` 會被視為過時。 您可以使用來具現化驗證讀取器 <xref:System.Xml.XmlReader.Create%2A?displayProperty=nameWithType> 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果 `XmlValidatingReader` 一律用來驗證來自受信任來源的 XML，因而無法遭到篡改，您可能會隱藏這個警告。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="violation"></a>違規

下列虛擬程式碼範例說明此規則偵測到的模式。
的第一個參數類型 `XmlValidatingReader.XmlValidatingReader()` 不是 `XmlReader` 。

```csharp
using System;
using System.IO;
using System.Xml;
...
public void TestMethod(Stream xmlFragment, XmlNodeType fragType, XmlParserContext context)
{
    var obj = new XmlValidatingReader(xmlFragment, fragType, context);
}
```

### <a name="solution"></a>解決方法

```csharp
using System;
using System.Xml;
...
public void TestMethod(XmlReader xmlReader)
{
    var obj = new XmlValidatingReader(xmlReader);
}
```
