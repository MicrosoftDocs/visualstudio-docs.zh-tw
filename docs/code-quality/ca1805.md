# <a name="ca1805-do-not-initialize-unnecessarily"></a>CA1805：請勿在不必要的情況中初始化。

|Item|值|
|-|-|
|CheckId|CA1805|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

類別的欄位會明確初始化為該欄位類型的預設值。

## <a name="rule-description"></a>規則描述

在執行此函式之前，.NET 執行時間會將參考型別的所有欄位初始化為其預設值。 在大部分情況下，將欄位明確初始化為其在函式中的預設值是多餘的，因此增加維護成本並可能降低效能 (例如，) 增加元件大小，而且可以移除明確的初始化。

## <a name="how-to-fix-violations"></a>如何修正違規

在大部分情況下，適當的修正是刪除不必要的初始化。

```csharp
class C
{
    // Violation
    int _value1 = 0;

    // Fixed
    int _value1;
}
```

在某些情況下，刪除初始化可能會導致後續發出的 [CS0649](/dotnet/csharp/misc/cs0649) 警告，因為欄位永遠保留其預設值。  在這種情況下，更好的修正方式可能是完全刪除欄位，或以屬性取代它：

```csharp
class C
{
    // Violation
    private static readonly int s_value = 0;

    // Fixed
    private static int Value => 0;
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

隱藏警告永遠是安全的，因為警告只會反白顯示可能避免的可能不必要程式碼和工作。
