# <a name="c6278"></a>C6278
警告 C6278:\<變數 > 是以陣列 new [] 配置，但以純量 delete 刪除。 將不會呼叫解構函式

這個警告只會顯示在C++程式碼，並指出呼叫的函式不一致已配置的陣列的記憶體**new []** 運算子，但釋放它以純量**刪除**運算子。 這是未定義的行為，根據C++標準和 Microsoft VisualC++實作。 有可能會造成問題的至少三個原因：

- 陣列中的個別物件的建構函式會叫用，但不是會叫用解構函式。

- 如果全域或類別專屬**new 運算子**並**delete 運算子**與不相容**運算子 new []** 並**運算子 delete []**，有可能發生的非預期的結果。

- 它一律是依賴未定義的行為有風險。

這項缺失的實際狀況很難預測。 它可能會導致遺漏類別與執行記憶體取消配置; 的解構函式不一致的行為，對於執行一些語意不重要的作業; 的解構函式類別或記憶體損毀，損毀時已覆寫的運算子。 在其他情況下可能不重要的不同的編譯器和其程式庫的實作不相符。 分析工具永遠無法區別這些情況。

如果陣列配置記憶體**new []**，應該將它通常會釋放陣列**delete []**。

## <a name="example"></a>範例

下列範例程式碼會產生這個警告：

```cpp
class A
{
  // members
};

void f( )
{
  A *pA = new A[5];
  // code ...
  delete pA;
}
```

若要更正這個警告，請使用下列的範例程式碼：

```cpp
void f( )
{
  A *pA = new A[5];
  // code ...
  delete[] pA;
}
```

陣列中的基礎物件是否為基本類型這類`int`， `float`， `enum`，指標，或有任何解構函式呼叫。 在這些情況下，警告[C6283](../code-quality/c6283.md)會改為報告。

使用新增和刪除有記憶體流失和例外狀況的許多問題。 若要避免這種流失和例外狀況的問題完全，使用所提供的機制C++標準範本庫 (STL)。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，並[向量](/cpp/standard-library/vector)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](/cpp/cpp/smart-pointers-modern-cpp)並[ C++ Standard 程式庫](/cpp/standard-library/cpp-standard-library-reference)。</c0>
