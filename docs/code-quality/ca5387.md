# <a name="ca5387-do-not-use-weak-key-derivation-function-with-insufficient-iteration-count"></a>CA5387：請勿使用弱式金鑰衍生函式，但反復專案計數不足

|||
|-|-|
|CheckId|CA5387|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

使用 <xref:System.Security.Cryptography.Rfc2898DeriveBytes.GetBytes%2A?displayProperty=fullName> 與預設反復專案計數，或指定小於100000的反復專案計數。

## <a name="rule-description"></a>規則描述

此規則會檢查是否以小於100000的反復專案計數來產生密碼編譯金鑰 <xref:System.Security.Cryptography.Rfc2898DeriveBytes> 。 較高的反復專案計數有助於減輕嘗試猜測產生的密碼編譯金鑰的字典攻擊。

此規則類似于[CA5388](ca5388.md)，但分析會判斷反復專案計數小於100000。

## <a name="how-to-fix-violations"></a>如何修正違規

呼叫之前，請先設定大於或等於100000的反復專案計數 <xref:System.Security.Cryptography.Rfc2898DeriveBytes.GetBytes%2A> 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您需要使用較小的反復專案計數來與現有的資料相容。

## <a name="configurability"></a>可設定性

如果您是從[FxCop 分析器](install-fxcop-analyzers.md)執行此規則（而不是使用舊版分析），此規則是可設定的。

### <a name="excluded-symbol-names"></a>排除的符號名稱

您可以設定程式碼基底中要排除的部分，使其不進行分析。 例如，若要指定規則不應在名為之型別內的任何程式碼上執行 `MyType` ，請將下列機碼值組新增至專案中的 editorconfig 檔案：

```ini
dotnet_code_quality.CA5387.excluded_symbol_names = MyType
```

選項值中允許的符號名稱格式（以 ' | ' 分隔）：
  - 僅限符號名稱（包含所有具有名稱的符號，不論包含的類型或命名空間為何）
  - 符號的[檔識別碼格式](https://github.com/dotnet/csharplang/blob/master/spec/documentation-comments.md#id-string-format)中的完整名稱。 每個符號名稱都需要符號種類前置詞，例如 "M：" 前置詞用於方法，"T：" 首碼用於類型，"N：" 前置詞用於命名空間等等。
  - `.ctor`適用于函式和 `.cctor` 靜態的函式

範例：

| 選項值 | 摘要 |
| --- | --- |
|`dotnet_code_quality.CA5387.excluded_symbol_names = MyType` | 符合編譯中名為 ' MyType ' 的所有符號
|`dotnet_code_quality.CA5387.excluded_symbol_names = MyType1|MyType2` | 比對在編譯中名為 ' MyType1 ' 或 ' MyType2 ' 的所有符號
|`dotnet_code_quality.CA5387.excluded_symbol_names = M:NS.MyType.MyMethod(ParamType)` | 與指定的完整簽章相符的特定方法 ' MyMethod '
|`dotnet_code_quality.CA5387.excluded_symbol_names = M:NS1.MyType1.MyMethod1(ParamType)|M:NS2.MyType2.MyMethod2(ParamType)` | 將特定方法 ' MyMethod1 ' 和 ' MyMethod2 ' 與個別的完整簽章進行比對

您可以只針對此規則、所有規則，或此類別中的所有規則（安全性），設定所有這些選項。 如需詳細資訊，請參閱[設定 FxCop 分析器](configure-fxcop-analyzers.md)。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="default-iteration-count-violation"></a>預設反復專案計數違規

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod(string password, byte[] salt, int cb)
    {
        var rfc2898DeriveBytes = new Rfc2898DeriveBytes(password, salt);
        rfc2898DeriveBytes.GetBytes(cb);
    }
}
```

### <a name="specify-iteration-count-in-constructor-violation"></a>指定違反函數的反復專案計數

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod(string password, byte[] salt, int cb)
    {
        var rfc2898DeriveBytes = new Rfc2898DeriveBytes(password, salt, 100);
        rfc2898DeriveBytes.GetBytes(cb);
    }
}
```

### <a name="specify-iteration-count-by-property-assignment-violation"></a>依屬性指派違規指定反復專案計數

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod(string password, byte[] salt, int cb)
    {
        var rfc2898DeriveBytes = new Rfc2898DeriveBytes(password, salt);
        rfc2898DeriveBytes.IterationCount = 100;
        rfc2898DeriveBytes.GetBytes(cb);
    }
}
```

### <a name="solution"></a>解決方法

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod(string password, byte[] salt, int cb)
    {
        var rfc2898DeriveBytes = new Rfc2898DeriveBytes(password, salt);
        rfc2898DeriveBytes.IterationCount = 100000;
        rfc2898DeriveBytes.GetBytes(cb);
    }
}
```
