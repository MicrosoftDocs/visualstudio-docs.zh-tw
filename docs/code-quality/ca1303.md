# <a name="ca1303-do-not-pass-literals-as-localized-parameters"></a>CA1303:不要將常值當作已當地語系化的參數傳遞

|||
|-|-|
|CheckId|CA1303|
|類別|Microsoft。全球化|
|重大變更|不中斷|

## <a name="cause"></a>原因

方法會將字串常值當做參數傳遞至 .NET 的函式或方法，而且該字串應該是可當地語系化的。

將常值字串當做值傳遞給參數或屬性，而且有一或多個下列情況成立時，就會引發這個警告：

- <xref:System.ComponentModel.LocalizableAttribute>參數或屬性的屬性設定為 true。

- 傳遞至主控台的字串參數名稱。 Write 或 Console。 WriteLine 方法可以是 "value" 或 "format"。

根據預設，此規則會分析整個程式碼基底，但這是[可](#excluded-type-names-with-derived-types)設定的。

從分析器套件的版本3.0.0 開始，預設不再使用「命名啟發學習法」，但這是[可](#use-naming-heuristic)設定的。

## <a name="rule-description"></a>規則描述

內嵌在原始程式碼中的字串常值很容易當地語系化。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正此規則的違規情形，請將字串常值取代為透過類別的實例所抓取的字串 <xref:System.Resources.ResourceManager> 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果程式碼程式庫不會當地語系化，或字串未向使用者或使用程式碼程式庫的開發人員公開，則可以放心地隱藏此規則的警告。

使用者可以藉由重新具名引數或屬性，或將這些專案標記為條件，來消除不應傳遞當地語系化字串之方法的干擾。

## <a name="configurability"></a>可設定性

如果您是從[FxCop 分析器](install-fxcop-analyzers.md)執行此規則（而不是使用舊版分析），此規則是可設定的。

### <a name="excluded-type-names-with-derived-types"></a>已排除具有衍生類型的類型名稱

您可以設定哪些類型（包括其衍生類型）從分析中排除。 例如，若要指定規則不應在名為的型別和其衍生型別中的任何方法上執行 `MyType` ，請將下列機碼值組新增至專案中的 editorconfig 檔案：

```ini
dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType
```

選項值中允許的符號名稱格式（以 ' | ' 分隔）：

- 僅限類型名稱（包含所有具有名稱的類型，不論包含的類型或命名空間為何）
- 符號的[檔識別碼格式](https://github.com/dotnet/csharplang/blob/master/spec/documentation-comments.md#id-string-format)中的完整名稱，具有選擇性的 `T:` 前置詞。

範例：

| 選項值 | 摘要 |
| --- | --- |
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType` | 比對所有名稱為 ' MyType ' 的類型及其所有的衍生類型（在編譯中）
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType1|MyType2` | 符合所有名稱為 ' MyType1 ' 或 ' MyType2 ' 的類型，以及其在編譯中的所有衍生類型
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = M:NS.MyType` | 符合具有給定完整名稱和其所有衍生類型的特定類型 ' MyType '
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = M:NS1.MyType1|M:NS2.MyType2` | 符合特定類型 ' MyType1 ' 和 ' MyType2 ' 與各自的完整名稱和其所有衍生類型

您可以只針對此規則、所有規則或此類別（全球化）中的所有規則來設定此選項。 如需詳細資訊，請參閱[設定 FxCop 分析器](configure-fxcop-analyzers.md)。

### <a name="use-naming-heuristic"></a>使用命名啟發學習法

您可以設定包含 "Text"、"Message" 或 "Caption" 的參數或屬性名稱是否會觸發此規則。

```ini
dotnet_code_quality.CA1303.use_naming_heuristic = true
```
