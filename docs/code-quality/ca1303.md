# <a name="ca1303-do-not-pass-literals-as-localized-parameters"></a>CA1303：不要將常值當做已當地語系化的參數傳遞

|||
|-|-|
|TypeName|DoNotPassLiteralsAsLocalizedParameters|
|CheckId|CA1303|
|類別|微軟.全球化|
|重大變更|不中斷|

## <a name="cause"></a>原因

方法將字串文本作為參數傳遞給 .NET 構造函數或方法,該字串應可當地語系化。

當文字字串作為值傳遞給參數或屬性,並且以下一個或多個情況為 true 時,將引發此警告:

- 參數<xref:System.ComponentModel.LocalizableAttribute>或屬性的屬性設定為 true。

- 傳遞給 Console.Write 或 Console.WriteLine 方法的字串參數的名稱是"值" 或" 格式"

預設情況下,此規則分析整個代碼庫,但這是[可設定的](#excluded-type-names-with-derived-types)。

從分析器套件的 3.0.0 版本開始,預設情況下不再使用命名啟發式,但這是[可設定的](#use-naming-heuristic)。

## <a name="rule-description"></a>規則描述

嵌入在原始碼中的字串文本很難當地語系化。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修復違反此規則的項,請將字串文本替換為通過<xref:System.Resources.ResourceManager>類的實例檢索的字串。

## <a name="when-to-suppress-warnings"></a>何時禁止警告

如果代碼庫未當地語系化,或者字串未向最終使用者或使用代碼庫的開發人員公開,則可以安全地禁止顯示此規則中的警告。

使用者可以通過重命名參數或屬性,或者將這些項目標記為條件來消除不應傳遞本地化字串的方法的雜訊。

## <a name="configurability"></a>可設定性

如果您從[FxCop 分析器](install-fxcop-analyzers.md)運行此規則(而不是使用舊版分析),則此規則是可配置的。

### <a name="excluded-type-names-with-derived-types"></a>具有衍生型型態的排除類型名稱

您可以設定要從分析中排除哪些類型(包括其派生類型)。 例如,要指定規則不應在命名`MyType`類型及其派生類型中的任何方法上運行,請在專案中將以下鍵值對添加到 .editorconfig 檔案中:

```ini
dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType
```

選項值中允許的符號名稱格式(以"*"分隔):

- 只鍵入名稱(包括名稱的所有類型,而不考慮包含的類型或命名空間)
- 符號[的文件 ID 格式](https://github.com/dotnet/csharplang/blob/master/spec/documentation-comments.md#id-string-format)完全限定的名稱,`T:`帶有可選的前置碼。

範例：

| 選項值 | 摘要 |
| --- | --- |
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType` | 符合編譯中名為「MyType」的所有類型及其所有派生類型
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType1|MyType2` | 符合所有名為「MyType1」或「MyType2」的類型及其在編譯中的所有派生類型
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = M:NS.MyType` | 將特定類型「MyType」與給定的完全限定名稱及其所有派生類型匹配
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = M:NS1.MyType1|M:NS2.MyType2` | 將特定類型「MyType1」和「MyType2」與各自的完全限定名稱及其所有派生類型匹配

您可以僅為此規則、所有規則或此類別中的所有規則(全球化)配置此選項。 有關詳細資訊,請參閱配置[FxCop 分析器](configure-fxcop-analyzers.md)。

### <a name="use-naming-heuristic"></a>使用命名啟發式

您可以設定包含「文本」、「消息」或「標題」的參數或屬性名稱是否會觸發此規則。

```ini
dotnet_code_quality.CA1303.use_naming_heuristic = true
```
