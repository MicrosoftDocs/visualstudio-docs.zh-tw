# <a name="ca1066-implement-iequatable-when-overriding-equals"></a>CA1066：覆寫 Equals 時執行 IEquatable

|||
|-|-|
|TypeName|EquatableAnalyzer|
|CheckId|CA1066|
|類別|Microsoft. Design|
|重大變更|不中斷|

## <a name="cause"></a>原因

實值型別（結構） <xref:System.Object.Equals%2A>會覆寫方法，但<xref:System.IEquatable%601>不會執行。

## <a name="rule-description"></a>規則描述

覆寫<xref:System.Object.Equals%2A>方法的實數值型別表示它支援比較類型的兩個實例以進行值相等。 請考慮執行<xref:System.IEquatable%601>介面，以支援強型別測試是否相等。 這可確保執行相等檢查的呼叫端會叫<xref:System.IEquatable%601.Equals%2A?displayProperty=fullName>用強型別方法，並避免對引數進行裝箱，以改善效能。 如需詳細資訊，請參閱[這裡](/dotnet/api/system.iequatable-1#notes-to-implementers)。

您<xref:System.IEquatable%601.Equals%2A?displayProperty=fullName>的執行應該會傳回與<xref:System.Object.Equals%2A>一致的結果。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請<xref:System.IEquatable%601>執行並<xref:System.Object.Equals%2A>更新覆寫，以叫用這個實方法。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
public struct S
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public override int GetHashCode()
        => _value.GetHashCode();

    public override bool Equals(object other)
        => other is S otherS && _value == otherS._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public override int GetHashCode()
        => _value.GetHashCode();

    public override bool Equals(object other)
        => other is S otherS && Equals(otherS);

    public bool Equals(S other)
        => _value == other._value;
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果執行介面的設計和效能優勢並不重要，就可以隱藏此規則的違規。
