# <a name="ca1829-use-lengthcount-property-instead-of-enumerablecount-method"></a>CA1829：請使用 Length/Count 屬性，不要使用 Enumerable.Count 方法

|||
|-|-|
|CheckId|CA1829|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:System.Linq.Enumerable.Count%2A>LINQ 方法是在支援對等、更有效率或屬性的型別上使用 `Length` `Count` 。

## <a name="rule-description"></a>規則描述

此規則會將 <xref:System.Linq.Enumerable.Count%2A> LINQ 方法呼叫標示為具有相等但較有效率 `Length` 或 `Count` 屬性來提取相同資料的類型集合。 `Length`或 `Count` 屬性不會列舉集合，因此更有效率。

此規則旗標會 <xref:System.Linq.Enumerable.Count%2A> 呼叫下列具有屬性的集合類型 `Length` ：

- <xref:System.Array?displayProperty=fullName>
- <xref:System.Collections.Immutable.ImmutableArray%601?displayProperty=fullName>

此規則旗標會 <xref:System.Linq.Enumerable.Count%2A> 呼叫下列具有屬性的集合類型 `Count` ：

- <xref:System.Collections.ICollection?displayProperty=fullName>
- <xref:System.Collections.Generic.ICollection%601?displayProperty=fullName>
- <xref:System.Collections.Generic.IReadOnlyCollection%601?displayProperty=fullName>

已分析的集合類型可能會在未來延伸，以涵蓋更多案例。

> [!NOTE]
> 舊版分析無法使用規則 CA1829。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.9.6 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請 <xref:System.Linq.Enumerable.Count%2A> 使用 `Length` 或屬性存取來取代方法呼叫 `Count` 。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public int GetCount(int[] array)
        => array.Count();

    public int GetCount(ICollection<int> collection)
        => collection.Count();
}
```


```csharp
using System.Collections.Generic;

class C
{
    public int GetCount(int[] array)
        => array.Length;

    public int GetCount(ICollection<int> collection)
        => collection.Count;
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在違規上，然後按**Ctrl** + **。** (句點)。 從顯示的選項清單中提供時，選擇 [**使用長度/計數] 屬性，而不是 [計數（）** ]。
>
> ![CA1829 的程式碼修正-使用長度/計數屬性，而不是 Count （）（如果有的話）](media/ca1829-codefix.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意不必要的集合列舉計算計數的效能影響，則可以放心抑制此規則的違規。
