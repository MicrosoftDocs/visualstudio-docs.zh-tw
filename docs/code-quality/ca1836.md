# <a name="ca1836-prefer-isempty-over-count-when-available"></a>CA1836：建議使用 IsEmpty，而不要使用 Count (如果有的話)

|Item|值|
|-|-|
|CheckId|CA1836|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

`Count`或 `Length` 屬性或 <xref:System.Linq.Enumerable.Count%60%601%28System.Collections.Generic.IEnumerable%7B%60%600%7D%29> 擴充方法是用來判斷物件是否包含任何專案，其方式是比較值與 `0` 或 `1` ，而且物件具有更有效率的 `IsEmpty` 屬性，可以改用。

## <a name="rule-description"></a>規則描述

此規則 `Count` `Length` 會在 <xref:System.Linq.Enumerable.Count%60%601%28System.Collections.Generic.IEnumerable%7B%60%600%7D%29> <xref:System.Linq.Enumerable.LongCount%60%601%28System.Collections.Generic.IEnumerable%7B%60%600%7D%29> 用來判斷物件是否包含任何專案，且物件具有更有效率的 `IsEmpty` 屬性時，標示和屬性或和 LINQ 方法的呼叫。

此規則的分析原本會與類似的規則 CA1827、CA1828 和 CA1829 重迭;這類規則的分析器會與 CA1836 合併，以在重迭時報告最佳診斷。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請在 <xref:System.Linq.Enumerable.Count%60%601%28System.Collections.Generic.IEnumerable%7B%60%600%7D%29> <xref:System.Linq.Enumerable.LongCount%60%601%28System.Collections.Generic.IEnumerable%7B%60%600%7D%29> `Length` `Count` 判斷物件是否為使用屬性存取權的作業中使用時，取代或方法呼叫或或屬性存取 `IsEmpty` 。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System.Collections.Concurrent;

class C
{
    ConcurrentQueue<int> _queue;
    public bool IsEmpty => _queue.Count == 0;
}
```


```csharp
using System.Collections.Concurrent;

class C
{
    ConcurrentQueue<int> _queue;
    public bool IsEmpty => _queue.IsEmpty;
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在違規上，然後按**Ctrl** + **。** (句點)。 選擇 [比對 ' 計數 ' 的 **' IsEmpty ']，以判斷物件是否包含**所提供選項清單中的任何專案。
>
> ![CA1836 的程式碼修正-偏好「IsEmpty」超過「計數」，以判斷物件是否包含任何專案](media/ca1836-codefix.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意不必要的專案列舉的效能影響來計算計數，則可以安全地隱藏此規則的違規。
