# <a name="c6255"></a>C6255
警告 C6255： _alloca 會引發堆疊溢位例外狀況，以指出失敗。 請考慮改為使用 _malloca

 此警告表示在本機例外狀況處理之外，偵測到 `_alloca` 的呼叫。 應該一律從例外狀況處理常式的受保護範圍內呼叫 `_alloca`，因為它可能會在失敗時引發堆疊溢位例外狀況。 如果可能，請考慮使用 `_malloca`，而不是使用 `_alloca`，這是較安全的 `_alloca` 版本。

## <a name="example"></a>範例
 下列程式碼會產生此警告，因為 `_alloca` 可能會產生例外狀況：

```
#include <windows.h>

void f( )
{
  void *p = _alloca(10);
  // code ...
}
```

 若要更正這個警告，請使用 `_malloca` 並新增例外狀況處理常式，如下列程式碼所示：

```
#include <windows.h>
#include <malloc.h>

void f( )
{
  void *p;
  int errcode;
  __try
  {
    p = _malloca(10);
    // code...
    _freea(p);
  }
  __except( (GetExceptionCode() == STATUS_STACK_OVERFLOW ) ?
              EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
    errcode = _resetstkoflw();
    // code ...
  }
}
```
