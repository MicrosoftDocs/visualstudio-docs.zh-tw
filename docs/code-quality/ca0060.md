# <a name="ca0060"></a>CA0060

無法載入間接參考的元件。

如果分析工具無法載入由分析所間接參考的元件，就會發生此警告。 「間接參考」是指您的其中一個分析元件的元件參考所參考的參考元件。 例如，如果程式碼分析（FxCop）正在分析元件 A，而元件 "A" 參考元件 "B"，而元件 "B" 參考元件 "C"，但 "A" 未參考元件 "c"，則元件 "C" 是間接參考，而元件 "B" 則是直接參考.

無法載入元件可能會導致其他地方發生錯誤 CA0001，因為可能會產生非預期的狀態。 如果您從 Visual Studio 內使用 managed 程式碼分析工具，以下是可能造成 CA0060 警告的一些原因：

- 已覆寫輸入元件，而現在指向的元件會參考 Visual Studio 專案檔中參考清單以外的其他元件。

- 如果在 Fxcopcmd.exe 執行之前修改專案以叫用自訂群組建步驟，且自訂群組建程式新增更多對輸入元件的參考，而且會產生警告 CA0060。

- 在C++中，您可以使用 VCPROJ 中的: vcproj forcedusing 來參考元件，覆寫命令列以傳遞/FU 或新增 #using \<僅限來源中的某些 .dll >。

- 協力廠商元件，具有其他元件的私用參考，而您不需要這麼做，即可編譯和執行程式碼。

在所有情況下，此問題的解決方法是將遺漏的參考新增至 Visual Studio 專案本身。
