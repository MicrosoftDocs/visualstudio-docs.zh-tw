# <a name="ca1416-validate-platform-compatibility"></a>CA1416：驗證平臺相容性

|||
|-|-|
|CheckId|CA1416|
|類別|Microsoft. 互通性|
|重大變更|非中斷|

## <a name="cause"></a>原因

在下列其中一個內容中使用平臺相依 API 來回報違規
- 中使用的平臺特定 API： 
  - 平臺中立的內容。
  - 不同的平臺內容。
- 目前的目標平臺不支援 API。

依預設，只有針對以或更新版本為目標的專案，才會啟用此規則 `net5.0` 。 不過，您可以針對以其他架構為目標的專案 [啟用](#configurability) 它。

## <a name="rule-description"></a>規則描述

在 .NET 5.0 中，我們已新增屬性來標注平臺特定 Api。 其運作方式如下：
- 未標記的 API 會視為可在所有 OS 平臺上運作。
- 標記為的 API `[SupportedOSPlatform("platformName")]` 只會被視為可移植到指定的 OS 平臺 (屬性可透過不同的平臺) 多次套用。
- 標記為的 API `[UnsupportedOSPlatform("platformName")]` 只會被視為不支援指定的作業系統平臺 (屬性可以在不同的平臺) 套用多次。
- 這兩個屬性都可以使用或不含版本號碼作為平臺名稱的一部分來具現化。
- 如果有屬性的組合 `[SupportedOSPlatform] and [UnsupportedOSPlatform]` ，我們會依 OS 平臺識別碼將所有屬性分組：
  - **允許清單**。 如果每個作業系統平臺的最低版本是 `[SupportedOSPlatform]` 屬性，則會將 API 視為僅受列出的平臺支援，而且所有其他平臺都不支援此 API。 此清單的屬性可能具有 `[UnsupportedOSPlatform]` 相同的平臺，但只有較高的版本，表示已從該版本移除 API。
  - **拒絕清單**。 如果每個作業系統平臺的最低版本是 `[UnsupportedOSPlatform]` 屬性，則會將 API 視為只有列出的平臺不支援，而且所有其他平臺都支援此 API。 此清單的屬性可能具有 `[SupportedOSPlatform]` 相同的平臺，但只有較高的版本，表示 API 已從該版本新增支援。
  - **不一致的清單**。 如果某些平臺的最低版本是 `[SupportedOSPlatform]` 在 `[UnsupportedOSPlatform]` 其他平臺上，則會被視為不一致，而且會忽略 API 上的一些批註。 我們打算引進另一個分析器，以在未來發生不一致的情況時產生警告。

從不同的平臺內容存取以上述屬性批註的 Api 可能會導致下列違規。

### <a name="violations"></a>違規
- 存取僅在指定平臺上支援的 API (`[SupportedOSPlatform("platformName")]`) 從其他平臺上可存取的程式碼將會產生違規： `'API' is supported on 'platformName'.`

  ```csharp
  // an API supported only on linux
  [SupportedOSPlatform("linux")]
  public void LinuxOnlyApi() { }

  // an API supported on windows and ios from version 14.0
  [SupportedOSPlatform("windows")]
  [SupportedOSPlatform("ios14.0")]
  public void SupportedOnWindowsAndIos14() { }

  public void Caller()
  {
      LinuxOnlyApi(); // warns CA1416: 'LinuxOnlyApi' is supported on 'linux'
    
      SupportedOnWindowsAndIos14(); // warns CA1416:'SupportedOnWindowsAndIos14' is supported on 'windows'  
                                    // warns CA1416: 'SupportedOnWindowsAndIos14' is supported on 'ios' 14.0 and later
  }

  ```

  > [!NOTE]
  > 只有當專案未以支援的平臺為目標時，才會發生違規 (`net5.0-differentPlatform`) 。 這也適用于多目標專案 (`net5.0`) 。 如果專案以指定的平臺為目標，則不會發生任何違規 (`net5.0-platformName`) 。

- 從以平臺為目標的內容中存取以屬性化的 API `[UnsupportedOSPlatform("platformName")]` `platformName` 可能會產生違規： `'API' is unsupported on 'platformName'.`

  ```csharp
  // an API not supported on android but supported on all other
  [UnsupportedOSPlatform("android")] 
  public void DoesNotWorkOnAndroid() { }

  // an API was unsupported on windows until version 10.0.1903. The API is considered supported everywhere else without constraints.
  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void StartedWindowsSupportFromCertainVersion();

  public void Caller()
  {
      DoesNotWorkOnAndroid(); // warns CA1416:'DoesNotWorkOnAndroid' is unsupported on 'android'
    
      StartedWindowsSupportFromCertainVersion(); // warns CA1416:'StartedWindowsSupportFromCertainVersion' is unsupported on 'windows'  
                                                 // warns CA1416:'StartedWindowsSupportFromCertainVersion' is supported on 'windows' 10.0.1903 and later
  }
  ```

  > [!NOTE]
  > 如果您要建立的應用程式不是以不受支援的平臺為目標，則不會出現任何違規。 違規只會在下列情況下發生：
  > - 如果專案的目標平臺設定為不支援或
  > - `platformName`包含在 [預設專案] 群組中， `MSBuild` `<SupportedPlatform>` 或
  > - `platformName`在 MSBuild items 群組中手動包含 `<SupportedPlatform>` ：

  ```XML
  <ItemGroup>
      <SupportedPlatform Include="platformName" />
  </ItemGroup>
  ```

## <a name="how-to-fix-violations"></a>如何修正違規

處理這些違規的建議方法是，確定您只在適當的平臺上執行時才呼叫這些 Api。 若要達成此目的，您可以使用 #if 和多重目標，或在執行時間有條件地呼叫程式碼，在組建階段排除程式碼。 分析器將會辨識我們新增的平臺防護， <xref:System.OperatingSystem> 以及 <xref:System.Runtime.InteropServices.RuntimeInformation.IsOSPlatform%2A?displayProperty=fullName> 您可以用來檢查的傳統平臺。 

- 以平臺防護方法圍繞呼叫位置來隱藏違規

  ```csharp
  // an API supported only on linux
  [SupportedOSPlatform("linux")]
  public void LinuxOnlyApi() { }  

  // an API supported on windows and ios from version 14.0
  [SupportedOSPlatform("windows")]
  [SupportedOSPlatform("ios14.0")]
  public void SupportedOnWindowsAndIos14() { }  
                                              
  public void Caller()
  {
      LinuxOnlyApi(); // warns CA1416: 'LinuxOnlyApi' is supported on 'linux'
    
      SupportedOnWindowsAndIos14(); // warns CA1416:'SupportedOnWindowsAndIos14' is supported on 'windows'  
                                    // warns CA1416: 'SupportedOnWindowsAndIos14' is supported on 'ios' 14.0 and later
  }

  // The diagnostics fixed using platform guard methods
  public void Caller()
  {
      if (OperatingSystem.IsLinux())
      {
          LinuxOnlyApi(); // no diagnostic
      }

      if (OperatingSystem.IsIOSVersionAtLeast(14))
      {
          SupportedOnWindowsAndIos14(); // no diagnostic
      }

      if(RuntimeInformation.IsOSPlatform(OSPlatform.Windows))
      {
          SupportedOnWindowsAndIos14(); // no diagnostic
      }
  }

  // an API not supported on android but supported on all other
  [UnsupportedOSPlatform("android")]  
  public void DoesNotWorkOnAndroid() { }

  // an API was unsupported on windows until version 10.0.1903. The API is considered supported everywhere else without constraints.
  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void StartedWindowsSupportFromCertainVersion();

  public void Caller()
  {
      DoesNotWorkOnAndroid(); // warns CA1416:'DoesNotWorkOnAndroid' is unsupported on 'android'
    
      StartedWindowsSupportFromCertainVersion(); // warns CA1416:'StartedWindowsSupportFromCertainVersion' is unsupported on 'windows'  
                                                 // warns CA1416:'StartedWindowsSupportFromCertainVersion' is supported on 'windows' 10.0.1903 and later
  }

  public void Caller()
  {
      if (!OperatingSystem.IsAndroid())
      {
          DoesNotWorkOnAndroid(); // no diagnostic
      }

      // Can use &&, || logical operators to guard combined attributes
      if (!OperatingSystem.IsWindows() || OperatingSystem.IsWindowsVersionAtLeast(10, 0, 1903))
      {
          StartedWindowsSupportFromCertainVersion(); // no diagnostic
      }
  }

  ```

- 分析器也可 <xref:System.Diagnostics.Debug.Assert%2A?displayProperty=fullName> 作為防止程式碼在不支援的平臺上到達的方法。 使用 `Debug.Assert` 可讓您視需要將檢查從發行組建中去除。

  ```csharp
  // an API supported only on linux
  [SupportedOSPlatform("linux")]
  public void LinuxOnlyApi() { } 

  public void Caller() 
  {
      Debug.Assert(OperatingSystem.IsLinux());

      LinuxOnlyApi(); // No diagnostic
  } 
  ```

- 您可以選擇將自己的 Api 標記為平臺專屬的 Api，如此一來，就能有效地將需求轉送至您的呼叫端。 您可以將平臺屬性套用至下列任一項：
  - 類型
  - 成員 (方法、欄位、屬性和事件) 
  - 組件

  ```csharp
  [SupportedOSPlatform("windows")]
  [SupportedOSPlatform("ios14.0")]
  public void SupportedOnWindowsAndIos14() { }  

  [SupportedOSPlatform("ios15.0")] // call site version should be equal to or higher than the API version
  public void Caller() 
  { 
      SupportedOnWindowsAndIos14(); // No diagnostics
  } 

  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void StartedWindowsSupportFromCertainVersion();

  [UnsupportedOSPlatform("windows")]
  [SupportedOSPlatform("windows10.0.1903")]
  public void Caller() 
  { 
      StartedWindowsSupportFromCertainVersion(); // No diagnostics
  } 
  ```

- 套用元件或類型層級屬性時，會將元件或類型中的所有成員視為平臺特定。

  ```csharp
  [assembly:SupportedOSPlatform("windows")]
  public namespace ns
  {
      public class Sample
      {
          public void SupportedOnWindows() { }  
      
          public void Caller() 
          { 
              SupportedOnWindows(); // No diagnostic as call site and calling method both windows only
          }
      }
  } 
  ```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

不建議在不使用適當平臺內容/防護的情況下參考平臺特定 Api。 不過，如有需要，您可以使用一般方式來隱藏這些診斷 (`<NoWarn>` 、編輯器設定或 #pragma) ：

```csharp
[SupportedOSPlatform("linux")]
public void LinuxOnlyApi() { } 

public void Caller() 
{ 
#pragma warning disable CA1416
    LinuxOnlyApi(); 
#pragma warning restore CA1416
} 
```

## <a name="configurability"></a>性

根據預設，分析器只會針對目標或更高版本的專案啟用 `net5.0` ，而 [AnalysisLevel](https://docs.microsoft.com/dotnet/core/project-sdk/msbuild-props#analysislevel) 專案) 的預設值為 5 (預設值 `net5.0` 。 您可以 `net5.0` 將下列索引鍵/值組新增至專案中的 editorconfig 檔案，以針對低於的目標 framework 啟用它：

```ini
dotnet_code_quality.enable_platform_analyzer_on_pre_net5_target=true
```

如需詳細資訊，請參閱 [設定 .net 程式碼品質分析器](configure-fxcop-analyzers.md)。
