# <a name="ca2246-do-not-assign-a-symbol-and-its-member-in-the-same-statement"></a>CA2246：請勿在相同的陳述式中指派符號及其成員

|Item|值|
|-|-|
|CheckId|CA2246|
|類別|Microsoft。使用方式|
|重大變更|不中斷|

## <a name="cause"></a>原因

已在相同的語句中指派符號和其成員。 例如：

```csharp
// 'a' and 'a.Field' are assigned in the same statement
a.Field = a = b;
```

## <a name="rule-description"></a>規則描述

不建議在相同的語句中指派符號和其成員（也就是欄位或屬性）。 如果成員存取預定要在指派之前使用符號的舊值，或在此語句中指派新的值，則不會很清楚。 為了清楚起見，多指派語句必須分割成兩個或多個簡單的指派語句。

> [!NOTE]
> 舊版分析無法使用規則 CA2246。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.9.6 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請將多個指派語句分割成兩個或多個簡單的指派語句。 例如，下列程式碼片段顯示違反規則的情況，以及根據使用者意圖修正的幾種方法：

```csharp
public class C
{
    public C Field;
}

public class Test
{
    public void M(C a, C b)
    {
        // Let us assume 'a' points to 'Instance1' and 'b' points to 'Instance2' at the start of the method.
        // It is not clear if the user intent in the below statement is to assign to 'Instance1.Field' or 'Instance2.Field'.        
        // CA2246: Symbol 'a' and its member 'Field' are both assigned in the same statement. You are at risk of assigning the member of an unintended object.
        a.Field = a = b;
    }
}
```

```csharp
public class C
{
    public C Field;
}

public class Test
{
    public void M(C a, C b)
    {
        // Let us assume 'a' points to 'Instance1' and 'b' points to 'Instance2' at the start of the method.
        // 'Instance1.Field' is intended to be assigned.
        var instance1 = a;
        a = b;
        instance1.Field = a;
    }
}
```

```csharp
public class C
{
    public C Field;
}

public class Test
{
    public void M(C a, C b)
    {
        // Let us assume 'a' points to 'Instance1' and 'b' points to 'Instance2' at the start of the method.
        // 'Instance2.Field' is intended to be assigned.
        a = b;
        b.Field = a; // or 'a.Field = a;'
    }
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則的違規。
