# <a name="ca2208-instantiate-argument-exceptions-correctly"></a>CA2208:必須正確執行個體化引數例外狀況

|||
|-|-|
|CheckId|CA2208|
|類別|Microsoft。使用方式|
|重大變更|不中斷|

## <a name="cause"></a>原因

當方法具有參數，而且它擲回例外狀況類型（也就是衍生自）時， <xref:System.ArgumentException> 它應該會呼叫可正確接受參數的函式 `paramName` 。 可能的原因包括下列情況：

- 會對具有接受參數之函式的例外狀況類型，呼叫預設（無參數）的函式（此為或衍生自） <xref:System.ArgumentException> `paramName` 。

- 不正確的字串引數會傳遞給例外狀況類型（也就是衍生自）的參數化函式 <xref:System.ArgumentException> 。

- 其中一個參數名稱會針對例外狀況型別（ `message` 也就是衍生自）的函式的引數傳遞 <xref:System.ArgumentException> 。

## <a name="rule-description"></a>規則描述

請呼叫其中一個可提供更有意義的例外狀況訊息的其中一個函式多載，而不是呼叫預設的函式。 例外狀況訊息應以開發人員為目標，並清楚說明錯誤情況，以及如何更正或避免例外狀況。

和其衍生類型的一個和兩個字串構造函式的簽章， <xref:System.ArgumentException> 與位置 `message` 和參數不一致 `paramName` 。 請確定使用正確的字串引數呼叫這些函式。 簽章如下所示：

- <xref:System.ArgumentException>（字串 `message` ）
- <xref:System.ArgumentException>（字串 `message` ，字串 `paramName` ）
- <xref:System.ArgumentNullException>（字串 `paramName` ）
- <xref:System.ArgumentNullException>（字串 `paramName` ，字串 `message` ）
- <xref:System.ArgumentOutOfRangeException>（字串 `paramName` ）
- <xref:System.ArgumentOutOfRangeException>（字串 `paramName` ，字串 `message` ）
- <xref:System.DuplicateWaitObjectException>（字串 `parameterName` ）
- <xref:System.DuplicateWaitObjectException>（字串 `parameterName` ，字串 `message` ）

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正此規則的違規，請呼叫接受訊息、參數名稱或兩者的函式，並確定引數對於所呼叫的類型而言是正確的 <xref:System.ArgumentException> 。

> [!TIP]
> 程式碼修正適用于 Visual Studio 中錯誤定位的參數名稱。 若要使用它，請將游標放在警告資料列上，然後按**Ctrl** + **。** (句點)。 從顯示的選項清單中選擇 [**交換引數順序**]。
>
> ![CA2208 的程式碼修正-交換引數](media/ca2208-codefix_swap.png)
>
> 如果 <xref:System.ArgumentException> （字串）多載 `message` 不小心使用參數名稱，則 fixer 會提供選項來搭配訊息使用兩個引數的函式 `null` 。
> 
> ![CA2208 的程式碼修正-交換引數](media/ca2208-codefix_null_msg.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

只有在使用正確的字串引數呼叫參數化的函式時，才可以安全地隱藏此規則的警告。

## <a name="configurability"></a>可設定性

如果您是從[FxCop 分析器](install-fxcop-analyzers.md)執行此規則（而不是使用舊版分析），您可以根據其存取範圍，設定程式碼基底中的哪些部分來執行此規則。 例如，若要指定規則只針對公用 API 介面執行，請將下列機碼值組新增至專案中的 editorconfig 檔案：

```ini
dotnet_code_quality.ca2208.api_surface = public
```
根據預設，CA2208 規則適用于所有 API 介面（公用、內部和私用）。 您可以只針對此規則、所有規則或此類別中的所有規則（設計）設定此選項。 如需詳細資訊，請參閱[設定 FxCop 分析器](configure-fxcop-analyzers.md)。
