# <a name="c26404--dont_delete_invalid"></a>C26404  DONT_DELETE_INVALID
一旦擁有者指標釋放或傳輸其資源，它就會進入「無效」狀態。
刪除這類指標可能會導致立即記憶體損毀，這是因為雙重刪除，或從另一個擁有者指標存取已刪除的資源時的存取違規。

## <a name="example-1-deleting-an-owner-after-transferring-its-value"></a>範例 1：在傳送其值之後刪除擁有者

```cpp
gsl::owner<State*> validState = nullptr;
gsl::owner<State*> state = ReadState();
validState = state;
if (!IsValid(state))
    delete state;   // C26404
```

## <a name="example-2-deleting-an-uninitialized-owner"></a>範例 2：刪除未初始化的擁有者

```cpp
gsl::owner<Message*> message;
if (popLast)
    message = ReleaseMessage();
delete message; // C26404
```
