# <a name="ca2244-do-not-duplicate-indexed-element-initializations"></a>CA2244：不要複製索引的元素初始化

|||
|-|-|
|CheckId|CA2244|
|類別|Microsoft。使用方式|
|重大變更|不中斷|

## <a name="cause"></a>原因

物件初始化運算式有一個以上具有相同常數索引的索引項目目初始化運算式。 除了最後一個初始化運算式之外，所有的都是多餘的。

## <a name="rule-description"></a>規則描述

[物件初始化運算式](/dotnet/csharp/programming-guide/classes-and-structs/object-and-collection-initializers#object-initializers)可讓您在建立期間將值指派給物件的任何可存取欄位或屬性，而不需要叫用後面接著指派語句行的函數。

物件初始化運算式中的索引項目目初始化運算式必須初始化唯一的元素。 重複的索引將會覆寫先前的專案初始化。

> [!NOTE]
> 舊版分析無法使用規則 CA2244。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.9.8 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請移除任何後續專案初始化運算式所覆寫的多餘索引項目目初始化運算式。 例如，下列程式碼片段顯示違反規則和一些可能的修正：

```csharp
using System.Collections.Generic;

class C
{
    public void M()
    {
        var dictionary = new Dictionary<int, int>
        {
            [1] = 1, // CA2244
            [2] = 2,
            [1] = 3
        };
    }
}
```

```csharp
using System.Collections.Generic;

class C
{
    public void M()
    {
        var dictionary = new Dictionary<int, int>
        {
            [2] = 2,
            [1] = 3
        };
    }
}
```

```csharp
using System.Collections.Generic;

class C
{
    public void M()
    {
        var dictionary = new Dictionary<int, int>
        {
            [1] = 1,
            [2] = 2
        };
    }
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在違規上，然後按**Ctrl** + **。** (句點)。 從顯示的選項清單中選擇 [**移除多餘的元素初始化運算式**]。
>
> ![CA2244 的程式碼修正-移除多餘的元素初始化運算式](media/ca2244-codefix.png)


## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則的違規。
