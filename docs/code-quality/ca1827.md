# <a name="ca1827-do-not-use-countlongcount-when-any-can-be-used"></a>CA1827：如果有任何可用的 Count/LongCount，請勿使用

|||
|-|-|
|TypeName|DoNotUseCountWhenAnyCanBeUsedAnalyzer|
|CheckId|CA1827|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

使用<xref:System.Linq.Enumerable.Count%2A>了<xref:System.Linq.Enumerable.LongCount%2A>或方法，其中<xref:System.Linq.Enumerable.Any%2A>方法會更有效率。

## <a name="rule-description"></a>規則描述

此規則會旗<xref:System.Linq.Enumerable.Count%2A>標<xref:System.Linq.Enumerable.LongCount%2A>和 LINQ 方法呼叫，用來檢查集合是否至少有一個元素。 這些方法呼叫需要列舉整個集合來計算計數。 使用<xref:System.Linq.Enumerable.Any%2A>方法時，相同的檢查會更快，因為它可避免列舉集合。

> [!NOTE]
> 舊版分析無法使用規則 CA1827。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.9.6 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請以<xref:System.Linq.Enumerable.Count%2A> <xref:System.Linq.Enumerable.Any%2A>方法<xref:System.Linq.Enumerable.LongCount%2A>取代或方法呼叫。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public string M1(IEnumerable<string> list)
        => list.Count() != 0 ? "Not empty" : "Empty";

    public string M2(IEnumerable<string> list)
        => list.LongCount() > 0 ? "Not empty" : "Empty";
}
```


```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public string M1(IEnumerable<string> list)
        => list.Any() ? "Not empty" : "Empty";

    public string M2(IEnumerable<string> list)
        => list.Any() ? "Not empty" : "Empty";
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在陣列違規上，然後按**Ctrl**+**。** (句點)。 當有任何（）可從顯示的選項清單中使用時，請選擇 [**不要使用 Count （）] 或 [LongCount （）** ]。
>
> ![CA1827 的程式碼修正-當可以使用任何（）時，請勿使用 Count （）或 LongCount （）](media/ca1827-codefix.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意不必要的集合列舉計算計數的效能影響，則可以放心抑制此規則的違規。
