# <a name="c28104"></a>C28104
警告 C28104：未取得函式結束前應取得的資源

 函式，目的是要在結束之前取得資源，而不需取得資源。 此警告表示函式已標注 `__drv_acquiresResource`，但不會傳回實際取得資源。 如果此函式是包裝函式，則透過函式的路徑不會到達已包裝的函式。 如果無法連線到已包裝的函式是因為函數傳回錯誤，而且實際上並未取得資源，您可能需要使用條件式注釋（`__drv_when`）。

 如果此函式實際執行資源的取得，則 PFD 可能無法偵測到資源已取得。 在此情況下，請使用 `#pragma` 警告來隱藏錯誤。 您可能會將 `#pragma` 放在開頭為函式主體的 `{` 前面的那一行。 呼叫函式仍然需要注釋，但程式碼分析工具將無法偵測是否已取得資源。

## <a name="example"></a>範例

```
__drv_acquireResourceGlobal(HWLock, lockid)
void GetHardwareLock(lockid)
#pragma warning (suppress: 28104)
{
   // code to implement a hardware lock (which the Code Analysis tool can't recognize)
}
```
