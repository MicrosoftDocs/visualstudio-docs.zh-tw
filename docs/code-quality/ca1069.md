# <a name="ca1069-enums-should-not-have-duplicate-values"></a>CA1069：列舉不能有重複的值

|||
|-|-|
|CheckId|CA1069|
|類別|Microsoft. Design|
|重大變更|中斷|

## <a name="cause"></a>原因

[列舉](/dotnet/csharp/language-reference/builtin-types/enum)有多個成員明確指派相同的常數值。

## <a name="rule-description"></a>規則描述

每個列舉成員都應該有唯一的常數值，或使用列舉中的先前成員明確地指派，以指示共用值的明確意圖。 例如：

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 2,   // CA1069: This is not fine. Either assign a different constant value or 'Field2' to indicate explicit intent of sharing value.
}
```

此規則有助於攔截從下列案例引進的功能 bug：
- 意外輸入錯誤，使用者不小心為多個成員輸入相同的常數值。
- 複製貼上錯誤，其中使用者複製了現有的成員定義，然後將該成員重新命名，但忘了變更此值。
- 來自多個分支的合併解析，其中新的成員是以不同的名稱加入，但在不同的分支中具有相同的值。

> [!NOTE]
> 舊版分析無法使用規則 CA1069。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本3.0.0 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請指派新的唯一常數值，或使用列舉中的先前成員來指派，以指示共用相同值的明確意圖。 例如，下列程式碼片段顯示違反規則的情況，以及修正違規的幾種方式：

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 2,   // CA1069: This is not fine. Either assign a different constant value or 'Field2' to indicate explicit intent of sharing value.
}
```

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = 3,   // This is now fine
}
```

```csharp
enum E
{
   Field1 = 1,
   AnotherNameForField1 = Field1,   // This is fine
   Field2 = 2,
   Field3 = Field2,   // This is also fine
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則的違規。
