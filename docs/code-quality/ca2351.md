# <a name="ca2351-ensure-datasetreadxmls-input-is-trusted"></a>CA2351：請確認 DataSet.ReadXml() 的輸入是受信任的

|Item|值|
|-|-|
|CheckId|CA2351|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:System.Data.DataSet.ReadXml%2A?displayProperty=nameWithType>已呼叫或參考方法，而不是在自動產生的程式碼中。

此規則會依下列方式分類自動產生的程式碼：
- 位於名為的方法內 `ReadXmlSerializable` 。
- `ReadXmlSerializable`方法具有 <xref:System.Diagnostics.DebuggerNonUserCodeAttribute?displayProperty=nameWithType> 。
- `ReadXmlSerializable`方法位於具有的類型內 <xref:System.ComponentModel.DesignerCategoryAttribute?displayProperty=nameWithType> 。

[CA2361](ca2361.md)是類似的規則，當出現在自動產生的程式 <xref:System.Data.DataSet.ReadXml%2A?displayProperty=nameWithType> 代碼內時。

## <a name="rule-description"></a>規則描述

當還原序列化為 <xref:System.Data.DataSet> 不受信任的輸入時，攻擊者可以製作惡意的輸入來執行阻絕服務攻擊。 可能有未知的遠端程式碼執行弱點。

如需詳細資訊，請參閱[資料集和 DataTable 安全性指引](https://go.microsoft.com/fwlink/?linkid=2132227)。

## <a name="how-to-fix-violations"></a>如何修正違規

- 可能的話，請使用[Entity Framework](https://docs.microsoft.com/ef/) ，而不是 <xref:System.Data.DataSet> 。
- 將序列化的資料進行篡改。 序列化之後，以密碼編譯方式簽署序列化的資料。 在還原序列化之前，請驗證密碼編譯簽章。 保護密碼編譯金鑰免于洩漏，並設計金鑰輪替。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="violation"></a>違規

```csharp
using System.Data;

public class ExampleClass
{
    public DataSet MyDeserialize(string untrustedXml)
    {
        DataSet dt = new DataSet();
        dt.ReadXml(untrustedXml);
    }
}
```
