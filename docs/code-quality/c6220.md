# <a name="c6220"></a>C6220
警告 C6220-語義不同整數類型之間的隱含轉換：比較 HRESULT 與-1。 請考慮改為使用 SUCCEEDED 或 FAILED 宏

 此警告表示正在將 `HRESULT` 與明確的非 `HRESULT` 值（不是格式正確的 `HRESULT`）進行比較。 @No__t-0 （`E_FAIL`）中的失敗不是以-1 表示。 因此，從 `HRESULT` 到整數的隱含轉換將會產生不正確的值，而且可能會導致錯誤的結果。

## <a name="example"></a>範例
 在大部分的情況下，此警告是由程式碼錯誤地預期應該傳回 `HRESULT` 的函式會傳回整數，方法是使用-1 做為失敗值。 下列程式碼範例會產生這個警告：

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == -1)
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 最佳做法是使用 `SUCCEEDED` 或 `FAILED` 宏來測試 `HRESULT` 的值。 若要修正這個警告，請使用下列程式碼：

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 針對此警告，`SCODE` 類型相當於 `HRESULT`。

 明確比較適用于檢查特定的 `HRESULT` 值，例如，`E_FAIL`。 否則，請使用 `SUCCEEDED` 或 `FAILED` 宏。

 如需詳細資訊，請參閱[SUCCEEDED 宏](http://go.microsoft.com/fwlink/?LinkId=92738)和[FAILED 宏](http://go.microsoft.com/fwlink/?LinkId=180875)。

 請注意，在記憶體流失和例外狀況方面，使用 malloc 和 free （和相關的動態記憶體配置 Api）有許多陷阱。 若要避免這類流失和例外狀況問題，請使用C++標準範本程式庫（STL）提供的機制。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)、 [unique_ptr](/cpp/standard-library/unique-ptr-class)和[vector](/cpp/standard-library/vector)。 如需詳細資訊，請參閱[智慧型指標](/cpp/cpp/smart-pointers-modern-cpp)和[ C++標準程式庫](/cpp/standard-library/cpp-standard-library-reference)。
