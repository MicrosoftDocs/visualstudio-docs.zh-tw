# <a name="ca5371-use-xmlreader-for-schema-read"></a>CA5371：請使用 XmlReader 讀取結構描述

|Item|值|
|-|-|
|CheckId|CA5371|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

不使用物件具現化來處理不受信任的 XML 輸入， <xref:System.Xml.Schema.XmlSchema.Read%2A?displayProperty=nameWithType> `XmlReader` 可能會導致拒絕服務、資訊洩漏和伺服器端偽造要求攻擊。 這些攻擊是由不受信任的 DTD 和 XML 架構處理啟用，可讓您在 XML 中包含 XML 炸彈和惡意的外部實體。 只有 with `XmlReader` 才能夠停用 DTD。 中的內嵌 XML 架構處理，在 `XmlReader` `ProhibitDtd` `ProcessInlineSchema` 4.0 版開始的 .NET Framework 中，預設會將和屬性設定為 false。 其他選項（例如 `Stream` 、 `TextReader` 和） `XmlSerializationReader` 無法停用 DTD 處理。

## <a name="rule-description"></a>規則描述

處理不受信任的 DTD 和 XML 架構可能會啟用載入危險的外部參考。 使用搭配 `XmlReader` 安全解析程式或已停用 DTD 和 XML 內嵌架構處理時，會限制這種情況。 此規則會偵測使用方法的程式碼， <xref:System.Xml.Schema.XmlSchema.Read%2A?displayProperty=nameWithType> 而不需要 `XmlReader` 做為參數。

## <a name="how-to-fix-violations"></a>如何修正違規

使用多載 `XmlSchema.Read(XmlReader, *)` 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果 <xref:System.Xml.Schema.XmlSchema.Read%2A?displayProperty=nameWithType> 一律使用方法來處理來自受信任來源的 XML，因而無法遭到篡改，您可能會隱藏這個警告。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="violation"></a>違規

下列虛擬程式碼範例說明此規則偵測到的模式。
的第一個參數類型 <xref:System.Xml.Schema.XmlSchema.Read%2A?displayProperty=nameWithType> 不是 `XmlReader` 。

```csharp
using System.IO;
using System.Xml.Schema;
...
public void TestMethod(Stream stream, ValidationEventHandler validationEventHandler)
{
    XmlSchema.Read(stream, validationEventHandler);
}
```

### <a name="solution"></a>解決方案

```csharp
using System.IO;
using System.Xml.Schema;
...
public void TestMethod(XmlReader reader, ValidationEventHandler validationEventHandler)
{
    XmlSchema.Read(reader, validationEventHandler);
}
```
