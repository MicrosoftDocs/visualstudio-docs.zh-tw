# <a name="ca1716-identifiers-should-not-match-keywords"></a>CA1716：識別項名稱不應該和關鍵字相符

|||
|-|-|
|TypeName|IdentifiersShouldNotMatchKeywords|
|CheckId|CA1716|
|類別|Micros. 命名|
|重大變更|中斷|

## <a name="cause"></a>原因

命名空間、類型或虛擬或介面成員的名稱與程式設計語言中的保留關鍵字匹配。

預設情況下,此規則僅檢視外部可見的命名空間、類型和成員,但您可以[設定可見性](#api-surface)和[符號類型](#analyzed-symbol-kinds)。

## <a name="rule-description"></a>規則描述

命名空間、類型以及虛擬和介面成員的標識符不應與由面向通用語言運行時的語言定義的關鍵字匹配。 根據所使用的語言和關鍵字,編譯器錯誤和模稜兩可會使庫難以使用。

此規則針對以下語言的關鍵字進行檢查:

- Visual Basic
- C#
- C++/CLI

不區分大小寫的比較用於 Visual Basic 關鍵字,並且區分大小寫的比較用於其他語言。

## <a name="how-to-fix-violations"></a>如何修正違規

選擇關鍵字清單中未顯示的名稱。

## <a name="when-to-suppress-warnings"></a>何時禁止警告

如果您確信識別碼不會混淆 API 的使用者,並且庫可用於 .NET 中的所有可用語言,則可以禁止顯示此規則的警告。

## <a name="configurability"></a>可設定性

### <a name="api-surface"></a>Api 表面

如果從[FxCop 分析器](install-fxcop-analyzers.md)運行此規則(而不是使用舊版分析),則可以根據代碼庫的可存取性配置要運行此規則的哪些部分。 例如,要指定規則應僅針對非公共 API 曲面運行,請在專案中將以下鍵值對新增到 .editorconfig 檔案中:

```ini
dotnet_code_quality.ca1716.api_surface = private, internal
```

您可以僅為此規則、所有規則或此類別中的所有規則(命名)設定此選項。 有關詳細資訊,請參閱配置[FxCop 分析器](configure-fxcop-analyzers.md)。

### <a name="analyzed-symbol-kinds"></a>剖析符號類型

您可以配置將由此規則分析的成員類型。 使用`Namespace`其中一個或多個`NamedType`值`Property``Event``Parameter``Method`、、、、 作為逗號分隔清單。 請注意,預設值包括上述所有成員。

```ini
dotnet_code_quality.CA1716.analyzed_symbol_kinds = Namespace, NamedType, Method, Property, Event
```
