# <a name="c6219"></a>C6219

警告 C6219：語義不同整數類型之間的隱含轉換：比較 HRESULT 為1或 TRUE。 請考慮改為使用[SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)或[FAILED](/windows/desktop/api/winerror/nf-winerror-failed)宏

此警告表示 `HRESULT` 正在與一（1）的明確、非 @no__t 1 值進行比較。 這項比較可能會導致不正確的結果，因為 `HRESULT` （`S_OK`）的一般成功值為0。 如果您將此值與布林類型進行比較，它會隱含地轉換為 false。

## <a name="example"></a>範例

下列程式碼會產生這個警告，因為 `CoGetMalloc` 會傳回 `HRESULT`，這會與 `TRUE` 進行比較：

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (hr == TRUE)
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

在大部分的情況下，這個警告是由將 `HRESULT` 與布林值相比較的程式碼所造成。 最好是使用 `SUCCEEDED` 或 `FAILED` 宏來測試 `HRESULT` 的值。 若要修正這個警告，請使用下列程式碼：

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

針對此警告，會將 `SCODE` 型別視為 `HRESULT`。

使用 `malloc` 和 `free` （以及相關的動態記憶體 Api）會有許多錯誤，因為記憶體流失和例外狀況的原因。 若要避免這類流失和例外狀況問題，請使用C++標準程式庫所提供的指標和容器類別。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)、 [unique_ptr](/cpp/standard-library/unique-ptr-class)和[vector](/cpp/standard-library/vector)。 如需詳細資訊，請參閱[智慧型指標](/cpp/cpp/smart-pointers-modern-cpp)和[ C++標準程式庫](/cpp/standard-library/cpp-standard-library-reference)。
