# <a name="c6320"></a>C6320
警告 C6320：例外狀況-篩選條件運算式是常數 EXCEPTION_EXECUTE_HANDLER。 這可能會遮罩不打算處理的例外狀況

 此警告表示在 __except 區塊中使用 EXCEPTION_EXECUTE_HANDLER 常數的副作用。 在此情況下，@no__t 4.9.0-_except 區塊中的語句一律會執行來處理例外狀況，包括您不想要在特定函數中處理的例外狀況。 建議您先檢查例外狀況，然後再進行處理。

## <a name="example"></a>範例
 下列程式碼會產生此警告，因為 __except 區塊會嘗試處理所有類型的例外狀況：

```
#include <stdio.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;  // might cause access violation exception
      // code ...
   }
   __except(EXCEPTION_EXECUTE_HANDLER) // warning
   {
      puts("in except");
      // code ...
   }
}
```

 若要修正這個警告，請使用 `GetExceptionCode` 來檢查特定的例外狀況，然後進行處理，如下列程式碼所示：

```
#include <stdio.h>
#include <windows.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;    // might cause access violation exception
      // code ...
   }
   __except(GetExceptionCode()==EXCEPTION_ACCESS_VIOLATION ?
               EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH)
   {
      puts("in except");
      // code ...
   }
}
```
