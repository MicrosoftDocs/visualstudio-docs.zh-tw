# <a name="ca2227-collection-properties-should-be-read-only"></a>CA2227：集合屬性應該為唯讀

|||
|-|-|
|TypeName|CollectionPropertiesShouldBeReadOnly|
|CheckId|CA2227|
|類別|微軟.使用|
|重大變更|中斷|

## <a name="cause"></a>原因

外部可見的可寫屬性的類型是實現<xref:System.Collections.ICollection?displayProperty=fullName>。 此規則忽略陣列、索引器(名稱為"Item"的屬性)、不可變集合、唯讀集合和許可權集。

## <a name="rule-description"></a>規則描述

可寫的集合屬性允許使用者用完全不同的集合替換集合。 唯讀屬性阻止替換集合,但仍允許設置單個成員。 如果替換集合是一個目標,則首選設計模式是包含一個從集合中刪除所有元素的方法,以及重新填充集合的方法。 有關此<xref:System.Collections.ArrayList.Clear%2A><xref:System.Collections.ArrayList.AddRange%2A>模式的範例<xref:System.Collections.ArrayList?displayProperty=fullName>, 請參閱類別的和方法。

二進位和 XML 序列化都支援集合的唯讀屬性。 類<xref:System.Xml.Serialization.XmlSerializer?displayProperty=fullName>對<xref:System.Collections.ICollection>實現<xref:System.Collections.IEnumerable?displayProperty=fullName>和可序列化的類型有特定的要求。

## <a name="how-to-fix-violations"></a>如何修正違規

要修復違反此規則的行為,請使屬性為唯讀。 如果設計需要它,則添加方法來清除和重新填充集合。

## <a name="when-to-suppress-warnings"></a>何時禁止警告

如果屬性是[數據傳輸物件 (DTO)](/previous-versions/msp-n-p/ff649585(v=pandp.10))類的一部分,則可以禁止該警告。

否則,不要禁止顯示此規則發出的警告。
