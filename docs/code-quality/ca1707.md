# <a name="ca1707-identifiers-should-not-contain-underscores"></a>CA1707:識別項名稱不應該包含底線

|Item|值|
|-|-|
|CheckId|CA1707|
|類別|Microsoft。命名|
|重大變更|重大-在元件上引發<br /><br /> 非中斷-在類型參數上引發時|

## <a name="cause"></a>原因

識別碼的名稱包含底線 (\_) 字元。

## <a name="rule-description"></a>規則描述

依照慣例，識別碼名稱不包含底線 (\_) 字元。 此規則會檢查命名空間、類型、成員和參數。

命名慣例可針對以 common language runtime 為目標的程式庫提供常見的外觀。 這可減少新軟體程式庫所需的學習曲線，並提高客戶的信賴度，以開發受管理程式碼的專業知識的人員來開發該程式庫。

## <a name="how-to-fix-violations"></a>如何修正違規

移除名稱中的所有底線字元。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏實際執行程式碼的警告。 不過，您可以放心隱藏測試程式碼的這個警告。 您可以藉由將警告的 [嚴重性](use-roslyn-analyzers.md#rule-severity) 設定為 [ **無**]，來隱藏此規則的警告。

若為 Microsoft 程式碼中目前使用底線且無法修改的已知方法，則應該隱藏規則。

## <a name="configurability"></a>性

如果您是從 [FxCop 分析器](install-fxcop-analyzers.md) 執行此規則 (而不是使用舊版分析) ，您可以根據其存取範圍，設定程式碼基底的哪些部分執行此規則。 例如，若要指定規則只能針對非公用 API 介面執行，請將下列機碼值組新增至專案中的 editorconfig 檔案：

```ini
dotnet_code_quality.ca1707.api_surface = private, internal
```

您可以只針對此規則、所有規則或此類別中的所有規則設定此選項， (命名) 。 如需詳細資訊，請參閱 [設定 FxCop 分析器](configure-fxcop-analyzers.md)。
