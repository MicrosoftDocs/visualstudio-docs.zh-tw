# <a name="ca1707-identifiers-should-not-contain-underscores"></a>CA1707:識別項名稱不應該包含底線

|Item|值|
|-|-|
|CheckId|CA1707|
|類別|Microsoft. 命名|
|重大變更|中斷-在元件上引發時<br /><br /> 不中斷-在類型參數上引發時|

## <a name="cause"></a>原因

識別碼的名稱包含底線 (\_) 字元。

## <a name="rule-description"></a>規則描述

依照慣例，識別碼名稱不包含底線 (\_) 字元。 此規則會檢查命名空間、類型、成員和參數。

命名慣例提供以通用語言執行時間為目標之程式庫的常見外觀。 這可減少新軟體程式庫所需的學習曲線，並提高客戶對於開發 managed 程式碼專業知識的人員所開發的信心。

## <a name="how-to-fix-violations"></a>如何修正違規

移除名稱中的所有底線字元。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏實際執行程式碼的警告。 不過，針對測試程式碼隱藏此警告是安全的。 您可以將其 [嚴重性](use-roslyn-analyzers.md#rule-severity) 設定為 [ **無**]，以隱藏來自此規則的警告。

對於目前使用底線且無法修改的 Microsoft 程式碼中的已知方法，應隱藏該規則。

## <a name="configurability"></a>可設定性

如果您是從 [FxCop (分析器](install-fxcop-analyzers.md) 執行此規則，而不是使用舊版分析) ，則可以根據其存取範圍，設定程式碼基底中的哪些部分，以執行此規則。 例如，若要指定規則只針對非公用 API 介面執行，請將下列機碼值組新增至專案中的 editorconfig 檔案：

```ini
dotnet_code_quality.ca1707.api_surface = private, internal
```

您可以只針對此規則設定此選項，針對所有規則，或針對此類別中的所有規則， (命名) 。 如需詳細資訊，請參閱 [設定 FxCop 分析器](configure-fxcop-analyzers.md)。
