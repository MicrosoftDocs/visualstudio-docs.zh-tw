# <a name="ca2247-argument-passed-to-taskcompletionsource-constructor-should-be-taskcreationoptions-enum-instead-of-taskcontinuationoptions-enum"></a>CA2247：傳遞到 TaskCompletionSource 建構函式的引數應為 TaskCreationOptions 列舉，而非 TaskContinuationOptions 列舉

|Item|值|
|-|-|
|CheckId|CA2247|
|類別|Microsoft。使用方式|
|重大變更|不中斷|

## <a name="cause"></a>原因

`System.Threading.Tasks.TaskCompletionSource`使用 <xref:System.Threading.Tasks.TaskContinuationOptions?displayProperty=fullName> 列舉值（而非列舉值）來建立 <xref:System.Threading.Tasks.TaskCreationOptions?displayProperty=fullName> 。
使用 <xref:System.Object.ReferenceEquals%2A?displayProperty=fullName> 方法來測試一或多個數值型別是否相等。

## <a name="rule-description"></a>規則描述

TaskCompletionSource 類型具有接受列舉值的函式 <xref:System.Threading.Tasks.TaskCreationOptions?displayProperty=fullName> ，以及另一個接受的函式 <xref:System.Object> 。  不小心傳遞 <xref:System.Threading.Tasks.TaskContinuationOptions?displayProperty=fullName> 列舉值而不是 <xref:System.Threading.Tasks.TaskCreationOptions?displayProperty=fullName> 列舉值，會導致呼叫以為基礎的函 <xref:System.Object> 式：它會進行編譯和執行，但不會有預期的行為。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請將 <xref:System.Threading.Tasks.TaskContinuationOptions?displayProperty=fullName> 列舉值取代為對應的 <xref:System.Threading.Tasks.TaskCreationOptions?displayProperty=fullName> 列舉值。

```csharp
    // Violation
    var tcs = new TaskCompletionSource<int>(TaskContinuationOptions.RunContinuationsAsynchronously);

    // Fixed
    var tcs = new TaskCompletionSource<int>(TaskCreationOptions.RunContinuationsAsynchronously);
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

違反此規則幾乎一律會反白顯示呼叫程式碼中的 bug，使程式碼不會像開發人員預期般運作，而 TaskCompletionSource 會有效地忽略指定的選項。  只有在開發人員實際打算將以 <xref:System.Threading.Tasks.TaskContinuationOptions?displayProperty=fullName> 物件狀態引數的形式傳遞至 TaskCompletionSource 時，才可以安全地隱藏警告。
