# <a name="c26117"></a>C26117
警告 C26117：釋放解除未保留鎖定 \<lock > in 函數 \<func >。

 語言不會執行 C/C++ program 中的語法範圍鎖定*取得*和鎖定*釋放*配對。 函式可能會藉由對並行狀態進行可觀察的修改來引入鎖定副作用。 例如，鎖定包裝函式會針對指定的鎖定，遞增鎖定數目或鎖定計數。您可以分別使用 `_Acquires_lock_` 或 `_Releases_lock_`，標注具有鎖定取得或鎖定釋放副作用的函式。 如果沒有這類注釋，函式就不會在傳回之後變更任何鎖定計數。 如果取得和發行未平衡，則視為*孤立*。 當尚未以 `_Releases_lock_` 標注的函式釋放不會保留的鎖定時，會發出警告 C26117，因為函式必須擁有鎖定，才能釋放它。

## <a name="example"></a>範例
 下列範例會產生警告 C26117，因為函式 `ReleaseUnheldLock` 會釋出不一定會保留的鎖定，`flag` 的狀態不明確，而且沒有任何指定應該的注釋。

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
        EnterCriticalSection(&p->cs);
    // code ...
    LeaveCriticalSection(&p->cs);
}
```

## <a name="example"></a>範例
 下列程式碼藉由保證也會在相同的情況下取得已釋放的鎖定，藉以修正問題。

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
    {
        EnterCriticalSection(&p->cs);
        // code ...
        LeaveCriticalSection(&p->cs);
    }
}
```
