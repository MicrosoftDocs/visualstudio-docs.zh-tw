# <a name="ca1826-use-property-instead-of-linq-enumerable-method"></a>CA1826：請使用屬性，不要使用 Linq Enumerable 方法

|||
|-|-|
|CheckId|CA1826|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:System.Linq.Enumerable>LINQ 方法是在支援對等、更有效率的屬性的型別上使用。

## <a name="rule-description"></a>規則描述

此規則會將 <xref:System.Linq.Enumerable> LINQ 方法呼叫標示為具有對等的類型集合，但更有效率的屬性會提取相同的資料。

此規則會分析下列集合類型：

- 執行的類型 <xref:System.Collections.Generic.IReadOnlyList%601> ，但不是<xref:System.Collections.Generic.IList%601>

此規則旗標會對這些集合類型呼叫下列方法：

- <xref:System.Linq.Enumerable.Count%2A?displayProperty=fullName> 方法
- <xref:System.Linq.Enumerable.First%2A?displayProperty=fullName> 方法
- <xref:System.Linq.Enumerable.FirstOrDefault%2A?displayProperty=fullName> 方法
- <xref:System.Linq.Enumerable.Last%2A?displayProperty=fullName> 方法
- <xref:System.Linq.Enumerable.LastOrDefault%2A?displayProperty=fullName> 方法

已分析的集合類型和/或方法可能會在未來延伸，以涵蓋更多案例。

> [!NOTE]
> 舊版分析無法使用規則 CA1826。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.9.6 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請將 <xref:System.Linq.Enumerable> 方法呼叫取代為屬性存取。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

class C
{
    public void M(IReadOnlyList<string> list)
    {
        Console.Write(list.First());
        Console.Write(list.Last());
        Console.Write(list.Count());
    }
}
```


```csharp
using System;
using System.Collections.Generic;

class C
{
    public void M(IReadOnlyList<string> list)
    {
        Console.Write(list[0]);
        Console.Write(list[list.Count - 1]);
        Console.Write(list.Count);
    }
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在違規上，然後按**Ctrl** + **。** (句點)。 從顯示的選項清單中選擇 [**使用索引子**]。
>
> ![CA1826 的程式碼修正-使用索引子](media/ca1826-codefix.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意特定方法呼叫的效能影響，則可以放心抑制此規則的違規 <xref:System.Linq.Enumerable> 。
