# <a name="c28250"></a>C28250
警告 C28250：函式的註釋不一致：之前的執行個體有錯誤。

 注意：此函式有數個原型。 這個警告會比較第一個原型和實例編號 \<number >。

 如果使用 `typedef` 來進行宣告，則出現 `typedef` 的那一行比宣告的行更有用。

 此警告會參考注釋中的錯誤，並反映出函式聲明上的注釋必須符合定義上之批註的需求，但牽涉到的函數 `typedef` 除外。 在這種情況下，函數 `typedef` 會同時視為宣告和定義的決定性。

 請注意，注釋通常會實作為宏，而一個宏通常會產生數個低層級的注釋。 針對每個不相符的低層級注釋報告此警告，因此單一不相符的注釋宏可能會產生一些不相符的低層級注釋。 最好只比較宣告和定義原始碼，以確保它們是相同的。 （不會報告注釋順序中的簡單差異）。

 這項比較一律會在找到的第一個宣告與目前的宣告之間進行。 如果有其他宣告，每個宣告都會成對檢查。 目前無法進行成對的比較，不過，您可以識別有兩個以上的宣告/定義。  上方的*文字*欄位包含兩個實例之間不同（相當低的層級）的注釋清單。

 此警告訊息會顯示傳送至編譯器的基礎程式碼文字，而不是用來在原始程式碼中實際插入注釋的宏（就像使用宏時的情況）。 一般來說，您不需要瞭解低層級的注釋，但您應該會發現在錯誤訊息中報告的行號之間，批註的回報不一致。 大部分的情況下，檢查原始程式碼會使其清楚說明為何存在不一致的情況。
