# <a name="ca5365-do-not-disable-http-header-checking"></a>CA5365：請勿停用 HTTP 標頭檢查

|Item|值|
|-|-|
|CheckId|CA5365|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

將 <xref:System.Web.Configuration.HttpRuntimeSection.EnableHeaderChecking> 設定為 `false`。

## <a name="rule-description"></a>規則描述

HTTP 標頭檢查可以編碼 `\r` `\n` 在回應標頭中找到的換行字元和分行符號。 這種編碼方式有助於避免利用回應標頭中包含不受信任資料之應用程式的插入式攻擊。

## <a name="how-to-fix-violations"></a>如何修正違規

將 <xref:System.Web.Configuration.HttpRuntimeSection.EnableHeaderChecking> 設定為 `true`。 或者，請移除的指派， `false` 因為預設值是 `true` 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

HTTP 標頭接續會依賴跨越多行的標頭，並在其中要求新的行。 如果您需要使用標頭接續，您必須將屬性設定 <xref:System.Web.Configuration.HttpRuntimeSection.EnableHeaderChecking> 為 `false` 。 檢查標頭會對效能造成影響。 如果您確定您已經執行正確的檢查，關閉此功能可改善應用程式的效能。 在您停用此功能之前，請確定您已在此區域中採取適當的預防措施。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

```csharp
using System;
using System.Web.Configuration;

class TestClass
{
    public void TestMethod()
    {
        HttpRuntimeSection httpRuntimeSection = new HttpRuntimeSection();
        httpRuntimeSection.EnableHeaderChecking = false;
    }
}
```

### <a name="solution"></a>解決方案

```csharp
using System;
using System.Web.Configuration;

class TestClass
{
    public void TestMethod()
    {
        HttpRuntimeSection httpRuntimeSection = new HttpRuntimeSection();
        httpRuntimeSection.EnableHeaderChecking = true;
    }
}
```
