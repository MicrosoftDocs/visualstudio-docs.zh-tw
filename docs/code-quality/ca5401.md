# <a name="ca5401-do-not-use-createencryptor-with-non-default-iv"></a>CA5401：請勿使用具有非預設 IV 的 CreateEncryptor

|Item|值|
|-|-|
|CheckId|CA5401|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

使用 <xref:System.Security.Cryptography.SymmetricAlgorithm.CreateEncryptor%2A?displayProperty=fullName> 非預設值 `rgbIV` 。

## <a name="rule-description"></a>規則描述

對稱式加密應一律使用不可重複的初始化向量來防止字典攻擊。

此規則類似于[CA5402](ca5402.md)，但分析會判斷初始化向量一定是預設值。

## <a name="how-to-fix-violations"></a>如何修正違規

使用預設 `rgbIV` 值，也就是使用 <xref:System.Security.Cryptography.SymmetricAlgorithm.CreateEncryptor%2A?displayProperty=fullName> 沒有任何參數的多載。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

在下列情況下，您可以放心地隱藏此規則中的警告：

- `rgbIV`參數是由所產生 <xref:System.Security.Cryptography.SymmetricAlgorithm.GenerateIV%2A?displayProperty=fullName> 。
- 您可以確定確實 `rgbIV` 是隨機且不可重複的。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod(byte[] rgbIV)
    {
        AesCng aesCng  = new AesCng();
        aesCng.IV = rgbIV;
        aesCng.CreateEncryptor();
    }
}
```

### <a name="solution"></a>解決方案

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod()
    {
        AesCng aesCng  = new AesCng();
        aesCng.CreateEncryptor();
    }
}
```
