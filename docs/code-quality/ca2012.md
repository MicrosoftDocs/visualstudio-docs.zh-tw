# <a name="ca2012-use-valuetasks-correctly"></a>CA2012：必須正確使用 ValueTasks

|Item|值|
|-|-|
|CheckId|CA2012|
|類別|Microsoft 可靠性|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:System.Threading.Tasks.ValueTask>從成員調用傳回的實例會以可能導致例外狀況、損毀或效能不佳的方式來使用。

## <a name="rule-description"></a>規則描述

<xref:System.Threading.Tasks.ValueTask>從成員調用傳回的實例必須直接等待。  嘗試多次使用 ValueTask，或在已知完成前直接存取一個結果，可能會導致例外狀況或損毀。  忽略這類 ValueTask 可能表示功能錯誤，而且可能會降低效能。

## <a name="how-to-fix-violations"></a>如何修正違規

一般來說，ValueTasks 應該直接等候，而不是捨棄或儲存到其他位置，例如本機變數或欄位。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

對於從任意成員呼叫傳回的 ValueTasks，呼叫端必須假設 ValueTask 只能取用（例如等候）一次。  不過，如果開發人員也控制要叫用的成員，而且完全瞭解其執行方式，則開發人員可能會知道隱藏警告，例如，如果傳回 ValueTask 一律包裝物件，則會很安全。 <xref:System.Threading.Tasks.Task>
