# <a name="ca5385-use-rivestshamiradleman-rsa-algorithm-with-sufficient-key-size"></a>CA5385：使用有足夠金鑰大小的 Rivest–Shamir–Adleman (RSA) 加密演算法

|Item|值|
|-|-|
|CheckId|CA5385|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

以下列其中一種方式使用非對稱式加密演算法 RSA，其金鑰大小低於2048：
- 將的任何子代類別具現化 <xref:System.Security.Cryptography.RSA?displayProperty=fullName> ，並將 `KeySize` 參數指定為小於2048。
- 傳回其型別為之子代的任何物件 <xref:System.Security.Cryptography.RSA?displayProperty=fullName> 。
- 使用 <xref:System.Security.Cryptography.AsymmetricAlgorithm.Create%2A?displayProperty=fullName> 不含參數的，這會建立具有預設金鑰大小1024的 RSA。
- 使用 <xref:System.Security.Cryptography.AsymmetricAlgorithm.Create%2A?displayProperty=fullName> ，並將 `algName` 參數指定為 `RSA` 預設金鑰大小1024。
- 使用 <xref:System.Security.Cryptography.CryptoConfig.CreateFromName%2A?displayProperty=fullName> ，並將 `name` 參數指定為 `RSA` 預設金鑰大小1024。
- 使用 <xref:System.Security.Cryptography.CryptoConfig.CreateFromName%2A?displayProperty=fullName> 和將 `name` 參數指定為 `RSA` ，並將金鑰大小指定為明確地小於 2048 `args` 。

## <a name="rule-description"></a>規則描述

小於2048位的 RSA 金鑰較容易遭受暴力密碼破解攻擊。

## <a name="how-to-fix-violations"></a>如何修正違規

改用至少2048金鑰大小、ECDH 或 ECDsa 演算法的 RSA。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

除非與繼承應用程式和資料相容，否則不建議您隱藏此規則。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="violation"></a>違規

下列虛擬程式碼範例說明此規則偵測到的模式。

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod()
    {
        RSACng rsaCng = new RSACng(1024);
    }
}
```

### <a name="solution"></a>解決方案

```csharp
using System.Security.Cryptography;

class ExampleClass
{
    public void ExampleMethod()
    {
        RSACng rsaCng = new RSACng(2048);
    }
}
```
