# <a name="c6230"></a>C6230

警告 C6230： 語意不相同的整數類型之間發生不當隱含轉換： 在布林內容中使用 HRESULT

這則警告指出，裸機`HRESULT`內容中使用布林值的結果所預期的位置，例如`if`陳述式。 這項測試可能會產生不正確的結果。 例如，一般的成功值`HRESULT`(`S_OK`) 為 false 時則會測試做為布林值。

## <a name="example"></a>範例

下列程式碼會產生這個警告：

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (hr)
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

在大部分情況下， [SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)或是[FAILED](/windows/desktop/api/winerror/nf-winerror-failed)巨集應該用來測試的值`HRESULT`。 若要更正這個警告，請使用下列程式碼：

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

這個警告，`SCODE`型別會被視為`HRESULT`。

善用`malloc`和`free`（及相關的動態記憶體的 Api） 做為記憶體流失和例外狀況的原因有許多的陷阱。 若要避免這種流失和例外狀況的問題，請使用所提供的指標和容器類別C++標準程式庫。 其中包括[shared_ptr](/cpp/standard-library/shared-ptr-class)， [unique_ptr](/cpp/standard-library/unique-ptr-class)，並[向量](/cpp/standard-library/vector)。 如需詳細資訊，請參閱 <<c0> [ 智慧型指標](/cpp/cpp/smart-pointers-modern-cpp)並[ C++ Standard 程式庫](/cpp/standard-library/cpp-standard-library-reference)。</c0>
