# <a name="ca2248-provide-correct-enum-argument-to-enumhasflag"></a>CA2248：提供正確的 enum 引數給 HasFlag

|||
|-|-|
|CheckId|CA2248|
|類別|Microsoft。使用方式|
|重大變更|不中斷|

## <a name="cause"></a>原因

當做引數傳遞至方法呼叫的列舉類型 `HasFlag` ，與呼叫的列舉類型不同。

## <a name="rule-description"></a>規則描述

`Enum.HasFlag`方法預期 `enum` 引數的 `enum` 類型與叫用方法所在的實例相同。 如果這些是不同的 `enum` 類型，則會在執行時間擲回未處理的例外狀況。

> [!NOTE]
> 舊版分析無法使用規則 CA2248。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本3.1.0 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請在引數和呼叫端上使用相同的列舉類型：

```csharp
public class C
{
    [Flags]
    public enum MyEnum { A, B, }

    [Flags]
    public enum OtherEnum { A, }

    public void Method(MyEnum m)
    {
        m.HasFlag(OtherEnum.A); // Enum types are different, this call will cause an `ArgumentException` to be thrown at runtime

        m.HasFlag(MyEnum.A); // Valid call
    }
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則的違規。
