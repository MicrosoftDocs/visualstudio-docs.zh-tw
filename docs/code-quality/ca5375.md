# <a name="ca5375-do-not-use-account-shared-access-signature"></a>CA5375：請勿使用帳戶共用存取簽章

|Item|值|
|-|-|
|CheckId|CA5375|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

使用 `GetSharedAccessSignature` 命名空間下的方法，產生帳戶共用存取簽章（SAS） `Microsoft.WindowsAzure.Storage` 。

## <a name="rule-description"></a>規則描述

帳戶 SAS 可以針對不允許服務 SAS 的 blob 容器、資料表、佇列和檔案共用，委派讀取、寫入和刪除作業的存取權。 不過，它不支援容器層級原則，而且對授與的許可權較不具彈性和控制權。 可能的話，請使用服務 SAS 進行更細緻的存取控制。 如需詳細資訊，請參閱[使用共用存取簽章委派存取](/rest/api/storageservices/delegate-access-with-shared-access-signature)。

## <a name="how-to-fix-violations"></a>如何修正違規

使用服務 SAS 而非帳戶 SAS 來進行更細緻的存取控制和容器層級存取原則。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您確定所有資源的許可權都盡可能受到限制，則隱藏此規則是安全的。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="violation"></a>違規

目前，下列虛擬程式碼範例說明此規則偵測到的模式。

```csharp
using System;
using Microsoft.WindowsAzure.Storage;

class ExampleClass
{
    public void ExampleMethod(SharedAccessAccountPolicy policy)
    {
        CloudStorageAccount cloudStorageAccount = new CloudStorageAccount();
        cloudStorageAccount.GetSharedAccessSignature(policy);
    }
}
```

### <a name="solution"></a>解決方案

請改用服務 SAS，而不是帳戶 SAS。

```csharp
using System;
using Microsoft.WindowsAzure.Storage;
using Microsoft.WindowsAzure.Storage.File;

class ExampleClass
{
    public void ExampleMethod(StorageCredentials storageCredentials, SharedAccessFilePolicy policy, SharedAccessFileHeaders headers, string groupPolicyIdentifier, IPAddressOrRange ipAddressOrRange)
    {
        CloudFile cloudFile = new CloudFile(storageCredentials);
        SharedAccessProtocol protocols = SharedAccessProtocol.HttpsOnly;
        cloudFile.GetSharedAccessSignature(policy, headers, groupPolicyIdentifier, protocols, ipAddressOrRange); 
    }
}
```
