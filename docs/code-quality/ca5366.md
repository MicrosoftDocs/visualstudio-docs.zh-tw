# <a name="ca5366-use-xmlreader-for-dataset-read-xml"></a>CA5366：請使用 XmlReader 為 DataSet 讀取 XML

|Item|值|
|-|-|
|CheckId|CA5366|
|類別|Microsoft.Security|
|重大變更|非中斷|

## <a name="cause"></a>原因

檔案類型定義 (DTD) 定義 XML 檔的結構和合法元素和屬性。 從外部資源參考 DTD 可能會導致 (DoS) 攻擊的潛在阻絕服務。 大部分的讀取器都無法停用 DTD 處理和限制外部參考載入，但除外 <xref:System.Xml.XmlReader?displayProperty=nameWithType> 。 使用這些其他讀取器，透過下列其中一種方法來載入 XML，就會觸發此規則：
- <xref:System.Data.DataSet.ReadXml%2A>
- <xref:System.Data.DataSet.ReadXmlSchema%2A>
- <xref:System.Data.DataSet.ReadXmlSerializable%2A>

## <a name="rule-description"></a>規則描述

使用 <xref:System.Data.DataSet?displayProperty=nameWithType> 來讀取具有不受信任資料的 XML 可能會載入危險的外部參考，而這些參考應透過使用 <xref:System.Xml.XmlReader> 安全解析程式或停用 DTD 處理來限制。

## <a name="how-to-fix-violations"></a>如何修正違規

使用 <xref:System.Xml.XmlReader> 或其衍生類別來讀取 XML。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

處理受信任的資料來源時，隱藏此規則的警告。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="violation"></a>違規

```csharp
using System.Data;
using System.IO;

public class ExampleClass
{
    public void ExampleMethod()
    {
        new DataSet().ReadXml(new FileStream("xmlFilename", FileMode.Open));
    }
}
```

### <a name="solution"></a>解決方法

```csharp
using System.Data;
using System.IO;
using System.Xml;

public class ExampleClass
{
    public void ExampleMethod()
    {
        new DataSet().ReadXml(new XmlTextReader(new FileStream("xmlFilename", FileMode.Open)));
    }
}
```
