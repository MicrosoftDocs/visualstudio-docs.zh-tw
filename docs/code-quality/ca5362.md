# <a name="ca5362-potential-reference-cycle-in-deserialized-object-graph"></a>CA5362：還原序列化物件圖中的可能參考循環

|Item|值|
|-|-|
|CheckId|CA5362|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

以標記的類別 <xref:System.SerializableAttribute?displayProperty=fullName> 具有欄位或屬性，可直接或間接參考包含物件，以允許潛在的參考週期。

## <a name="rule-description"></a>規則描述

如果還原序列化不受信任的資料，則處理已還原序列化之物件圖形的任何程式碼都必須處理參考迴圈，而不會進入無限迴圈。 這同時包含屬於還原序列化回呼一部分的程式碼，以及在還原序列化完成後處理物件圖形的程式碼。 否則，攻擊者可能會使用包含參考週期的惡意資料來執行拒絕服務攻擊。

此規則不一定表示有弱點，但只會標示已還原序列化物件圖形中的潛在參考週期。

## <a name="how-to-fix-violations"></a>如何修正違規

不要序列化類別，並移除 <xref:System.SerializableAttribute> 。 或者，重新設計您的應用程式，以便從可序列化的類別中移除自我參考的成員。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

在下列情況下，您可以放心地隱藏此規則中的警告：
- 您知道輸入是受信任的。 請考慮您應用程式的信任界限和資料流程可能會隨著時間而改變。
- 所有處理還原序列化資料的程式碼都會偵測並處理參考迴圈，而不會進入無限迴圈或使用過多的資源。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="potential-reference-cycle-violation"></a>可能的參考週期違規

```csharp
using System;

[Serializable()]
class ExampleClass
{
    public ExampleClass ExampleProperty {get; set;}

    public int NormalProperty {get; set;}
}

class AnotherClass
{
    // The argument passed by could be `JsonConvert.DeserializeObject<ExampleClass>(untrustedData)`.
    public void AnotherMethod(ExampleClass ec)
    {
        while(ec != null)
        {
            Console.WriteLine(ec.ToString());
            ec = ec.ExampleProperty;
        }
    }
}
```

### <a name="solution"></a>解決方案

```csharp
using System;

[Serializable()]
class ExampleClass
{
    [NonSerialized]
    public ExampleClass ExampleProperty {get; set;}

    public int NormalProperty {get; set;}
}

class AnotherClass
{
    // The argument passed by could be `JsonConvert.DeserializeObject<ExampleClass>(untrustedData)`.
    public void AnotherMethod(ExampleClass ec)
    {
        while(ec != null)
        {
            Console.WriteLine(ec.ToString());
            ec = ec.ExampleProperty;
        }
    }
}
```
