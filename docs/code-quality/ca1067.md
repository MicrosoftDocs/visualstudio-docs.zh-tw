# <a name="ca1067-override-equals-when-implementing-iequatable"></a>CA1067：實作 IEquatable 時覆寫 Equals

|Item|值|
|-|-|
|CheckId|CA1067|
|類別|Microsoft. Design|
|重大變更|不中斷|

## <a name="cause"></a>原因

型別 <xref:System.IEquatable%601> 會執行，但不會覆寫 <xref:System.Object.Equals%2A> 方法。

## <a name="rule-description"></a>規則描述

實作為型別的 <xref:System.IEquatable%601> 介面指出，它支援比較型別的兩個實例是否相等。 您也應該覆寫和方法的基類實作為， <xref:System.Object.Equals%2A> <xref:System.Object.GetHashCode> 讓它們的行為與執行方式一致 <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> 。 詳細資訊請看[這裡](/dotnet/api/system.iequatable-1#notes-to-implementers)。

您的 <xref:System.Object.Equals%2A> 執行應該會傳回與實作為一致的結果 <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> 。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請叫 <xref:System.Object.Equals%2A> 用實作為覆寫並加以執行 <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> 。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;

    public override bool Equals(object obj)
        => obj is S objS && Equals(objS);

    public override int GetHashCode()
        => _value.GetHashCode();
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則的違規。
