# <a name="ca1067-override-equals-when-implementing-iequatable"></a>CA1067：在執行 IEquatable 時覆寫 Equals

|||
|-|-|
|TypeName|EquatableAnalyzer|
|CheckId|CA1067|
|類別|Microsoft. Design|
|重大變更|不中斷|

## <a name="cause"></a>原因

型別會<xref:System.IEquatable%601>執行，但不會<xref:System.Object.Equals%2A>覆寫方法。

## <a name="rule-description"></a>規則描述

實作為型<xref:System.IEquatable%601>別的介面指出，它支援比較型別的兩個實例是否相等。 您也應該覆寫<xref:System.Object.Equals%2A>和<xref:System.Object.GetHashCode>方法的基類實作為，讓它們的行為與<xref:System.IEquatable%601.Equals%2A?displayProperty=fullName>執行方式一致。 詳細資訊請看[這裡](/dotnet/api/system.iequatable-1#notes-to-implementers)。

您<xref:System.Object.Equals%2A>的執行應該會傳回與<xref:System.IEquatable%601.Equals%2A?displayProperty=fullName>實作為一致的結果。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請<xref:System.Object.Equals%2A>叫用實作為<xref:System.IEquatable%601.Equals%2A?displayProperty=fullName>覆寫並加以執行。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;

    public override bool Equals(object obj)
        => obj is S objS && Equals(objS);

    public override int GetHashCode()
        => _value.GetHashCode();
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則的違規。
