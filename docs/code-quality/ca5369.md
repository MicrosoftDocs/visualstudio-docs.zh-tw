# <a name="ca5369-use-xmlreader-for-deserialize"></a>CA5369：請使用 XmlReader 進行還原序列化

|Item|值|
|-|-|
|CheckId|CA5369|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

不使用物件具現化時，還原序列化不受信任的 XML 輸入 <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> `XmlReader` ，可能會導致拒絕服務、資訊洩漏和伺服器端偽造要求攻擊。 這些攻擊是由不受信任的 DTD 和 XML 架構處理啟用，可讓您在 XML 中包含 XML 炸彈和惡意的外部實體。 只有 with `XmlReader` 才能夠停用 DTD。 中的內嵌 XML 架構處理， `XmlReader` `ProhibitDtd` `ProcessInlineSchema` `false` 在 .NET Framework 4.0 和更新版本中預設會將和屬性設定為。 其他選項（例如 `Stream` 、 `TextReader` 和） `XmlSerializationReader` 無法停用 DTD 處理。

## <a name="rule-description"></a>規則描述

處理不受信任的 DTD 和 XML 架構可能會啟用載入危險的外部參考，這應該使用搭配 `XmlReader` 安全解析程式或停用 DTD 和 XML 內嵌架構處理來加以限制。 此規則會偵測使用方法的程式碼 <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> ，而不接受做為函式 `XmlReader` 參數。

## <a name="how-to-fix-violations"></a>如何修正違規

請勿使用 <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader)> 、、或以外的多載 <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.String)> <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.Xml.Serialization.XmlDeserializationEvents)> <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.String,System.Xml.Serialization.XmlDeserializationEvents)> 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果剖析的 XML 來自受信任的來源，因而無法遭到篡改，您可能會隱藏這個警告。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

### <a name="violation"></a>違規

下列虛擬程式碼範例說明此規則偵測到的模式。
的第一個參數類型 <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> 不是 `XmlReader` 或其衍生類別。

```csharp
using System.IO;
using System.Xml.Serialization;
...
new XmlSerializer(typeof(TestClass).Deserialize(new FileStream("filename", FileMode.Open));
```

### <a name="solution"></a>解決方案

```csharp
using System.IO;
using System.Xml;
using System.Xml.Serialization;
...
new XmlSerializer(typeof(TestClass)).Deserialize(XmlReader.Create (new FileStream("filename", FileMode.Open)));
```
