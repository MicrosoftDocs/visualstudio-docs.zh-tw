# <a name="ca1501-avoid-excessive-inheritance"></a>CA1501：避免過度繼承

|||
|-|-|
|TypeName|AvoidExcessiveInheritance|
|CheckId|CA1501|
|類別|微軟.可維護性|
|重大變更|中斷|

## <a name="cause"></a>原因

類型在其繼承階層架構 (Inheritance Hierarchy) 中超過四個層級的深度。

預設情況下,規則僅從`System`命名空間中排除所有類型的,但這是[可配置的](#configurability)。

## <a name="rule-description"></a>規則描述

太深的巢狀類型階層架構可能會難以依循、了解和維護。 此規則將分析限制為同一模組中的層次結構。

## <a name="how-to-fix-violations"></a>如何修正違規

要修復違反此規則的衝突,應從繼承層次結構中較不深的基類型派生類型,或消除某些中間基類型。

## <a name="when-to-suppress-warnings"></a>何時禁止警告

可以安全地禁止顯示此規則的警告。 但是,代碼可能更難維護。 請注意,根據基類型的可見性,解決違反此規則可能會創建重大更改。 例如,刪除公共基類型是一個重大更改。

## <a name="configurability"></a>可設定性

### <a name="inheritance-excluded-type-or-namespace-names"></a>繼承排除的類型或命名空間名稱

您可以設定類型或命名空間的名稱(由"*"分隔),以便類型或類型的命名空間在繼承層次結構樹中不計數。 預設情況下,命名空間中的所有`System.*`類型都排除。 請注意,無論您正在設置哪個值,都將添加此預設值。

| 選項值 | 摘要 |
| --- | --- |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = MyType` | 符合名為「MyType」的所有類型的或包含命名空間包含「MyType」的所有類型的「系統」命名空間中的所有類型 |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = MyType1\|MyType2` | 符合名為「MyType1」或「MyType2」的所有類型,或者其包含命名空間包含「MyType1」或「MyType2」以及「系統」命名空間中的所有類型 |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = T:NS.MyType` | 符合命名空間「NS」中的特定類型「MyType」和「系統」命名空間中的所有類型 |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = T:NS1.MyType1\|T:NS2.MyType2` | 將特定類型「MyType1」和「MyType2」與相應的完全限定名稱以及「系統」命名空間中的所有類型進行匹配 |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = N:NS` | 符合「NS」命名空間中的所有類型以及「系統」命名空間中的所有類型 |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = My*` | 符合名稱以「My」開頭或包含命名空間元件以「我的」開頭的所有類型的類型,以及「系統」命名空間中的所有類型 |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = T:NS.My*` | 符合名稱以命名空間「NS」中的「我的」開頭的所有類型的類型,以及「系統」命名空間中的所有類型 |
|`dotnet_code_quality.CA1501.additional_inheritance_excluded_symbol_names = N:My*` | 符合包含命名空間以「我的」開頭的所有類型,以及「系統」命名空間中的所有類型 |
