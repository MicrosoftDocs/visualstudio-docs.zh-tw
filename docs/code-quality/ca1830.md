# <a name="ca1830-prefer-strongly-typed-append-and-insert-method-overloads-on-stringbuilder"></a>CA1830：偏好 StringBuilder 上的強型別附加和插入方法多載。

|||
|-|-|
|CheckId|CA1830|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

呼叫了 <xref:System.Text.StringBuilder> `Append` 或方法，該 `Insert` 引數是在 `ToString` `Append` 或方法具有專用多載的類型上呼叫的結果 `Insert` 。

## <a name="rule-description"></a>規則描述

<xref:System.Text.StringBuilder.Append%2A>和 <xref:System.Text.StringBuilder.Insert%2A> 提供多個類型的多載 <xref:System.String> 。  可能的話，偏好使用 ToString （）和以字串為基礎之多載的強型別多載。

## <a name="how-to-fix-violations"></a>如何修正違規

從調用中刪除不必要的 `ToString()` 。

```csharp
using System.Text;

class C
{
    int _value;

    // Violation
    public void Log(StringBuilder destination)
    {
        destination.Append("Value: ").Append(_value.ToString()).AppendLine();
    }

    // Fixed
    public void Log(StringBuilder destination)
    {
        destination.Append("Value: ").Append(_value).AppendLine();
    }
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意不必要的字串配置所造成的效能影響，就可以放心抑制此規則的違規。
