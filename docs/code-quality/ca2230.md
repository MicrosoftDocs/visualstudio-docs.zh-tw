# <a name="ca2230-use-params-for-variable-arguments"></a>CA2230:必須使用 params 作為變數引數

|Item|值|
|-|-|
|CheckId|CA2230|
|類別|Microsoft。使用方式|
|重大變更|中斷|

## <a name="cause"></a>原因
公用或受保護的類型包含使用呼叫慣例的公用或受保護的方法 `VarArgs` 。

> [!NOTE]
> 此舊版規則不適用於 FxCop 分析器。 如需詳細資訊，請參閱已[淘汰的規則](fxcop-rule-port-status.md#deprecated-rules)。

## <a name="rule-description"></a>規則描述
`VarArgs`呼叫慣例會搭配採用可變數目參數的特定方法定義使用。 使用 `VarArgs` 呼叫慣例的方法不 Common Language Specification （CLS）相容，而且可能無法跨程式設計語言存取。

在 c # 中， `VarArgs` 當方法的參數清單以關鍵字結尾時，會使用呼叫慣例 `__arglist` 。 Visual Basic 不支援 `VarArgs` 呼叫慣例，而且 c + + 只允許其在使用橢圓形標記法的非受控碼中使用 `...` 。

## <a name="how-to-fix-violations"></a>如何修正違規
若要修正 c # 中此規則的違規情形，請使用[params](/dotnet/csharp/language-reference/keywords/params)關鍵字，而不是 `__arglist` 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機
請勿隱藏此規則的警告。
