# <a name="ca2014-do-not-use-stackalloc-in-loops"></a>CA2014：不要在迴圈中使用 stackalloc

|||
|-|-|
|CheckId|CA2014|
|類別|Microsoft 可靠性|
|重大變更|不中斷|

## <a name="cause"></a>原因

在迴圈內使用 c # stackalloc 關鍵字。

## <a name="rule-description"></a>規則描述

C # stackalloc 關鍵字會從目前的堆疊框架配置記憶體，而且在目前的方法呼叫傳回之前，不會釋放記憶體。  如果 stackalloc 關鍵字用於迴圈中，可能會因為耗盡堆疊記憶體而導致堆疊溢位。

## <a name="how-to-fix-violations"></a>如何修正違規

將 stackalloc 移至方法中所有迴圈的外部。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

當包含的迴圈或迴圈只叫用有限的次數時，可能可以安全地隱藏警告，讓所有 stackalloc 作業所配置的總記憶體量相對較小。
