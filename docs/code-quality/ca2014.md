# <a name="ca2014-do-not-use-stackalloc-in-loops"></a>CA2014：請勿在迴圈中使用 stackalloc

|Item|值|
|-|-|
|CheckId|CA2014|
|類別|Microsoft 可靠性|
|重大變更|非中斷|

## <a name="cause"></a>原因

在迴圈內使用 c # stackalloc 關鍵字。

## <a name="rule-description"></a>規則描述

C # stackalloc 關鍵字會從目前的堆疊框架配置記憶體，而且在目前的方法呼叫傳回之前，可能不會釋放記憶體。  如果在迴圈中使用 stackalloc 關鍵字，它可能會導致堆疊溢位，因為耗盡堆疊記憶體。

## <a name="how-to-fix-violations"></a>如何修正違規

將 stackalloc 移至方法中所有迴圈的外部。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

當只叫用包含迴圈或迴圈的次數有限時，可能可以安全地隱藏警告，如此一來，在所有 stackalloc 作業間配置的總記憶體量就會相對較小。
