# <a name="c26100"></a>C26100
警告 C26100：競爭條件。 變數 \<var > 應該受到 lock \<lock > 的保護。

 程式碼中的 `_Guarded_by_` 注釋會指定要用來保護共用變數的鎖定。 違反 guard 合約時，會產生警告 C26100。

## <a name="example"></a>範例
 下列範例會產生警告 C26100，因為違反了 `_Guarded_by_` 合約。

```
CRITICAL_SECTION gCS;

_Guarded_by_(gCS) int gData;

typedef struct _DATA {
   _Guarded_by_(cs) int data;
   CRITICAL_SECTION cs;
} DATA;

void Safe(DATA* p) {
   EnterCriticalSection(&p->cs);
   p->data = 1; // OK
   LeaveCriticalSection(&p->cs);
   EnterCriticalSection(&gCS);
   gData = 1; // OK
   LeaveCriticalSection(&gCS);
}

void Unsafe(DATA* p) {
   EnterCriticalSection(&p->cs);
   gData = 1; // Warning C26100 (wrong lock)
   LeaveCriticalSection(&p->cs);
}
```

 發生合約違規的原因是函數中使用了不正確的鎖定 `Unsafe`。 在此情況下，`gCS` 是要使用的正確鎖定。

## <a name="example"></a>範例
 有時候，共用變數只需受限於寫入存取，而不是讀取權限。 在此情況下，請使用 `_Write_guarded_by_` 注釋，如下列範例所示。

```

CRITICAL_SECTION gCS;

_Guarded_by_(gCS) int gData;

typedef struct _DATA2 {
   _Write_guarded_by_(cs) int data;
   CRITICAL_SECTION cs;
} DATA2;

int Safe2(DATA2* p) {
   // OK: read does not have to be guarded
   int result = p->data;
   return result;
}

void Unsafe2(DATA2* p) {
   EnterCriticalSection(&gCS);
   // Warning C26100 (write has to be guarded by p->cs)
   p->data = 1;
   LeaveCriticalSection(&gCS);
}
```

 這個範例也會產生警告 C26100，因為它在函數中使用不正確的鎖定 `Unsafe2`。
