# <a name="ca5392-use-defaultdllimportsearchpaths-attribute-for-pinvokes"></a>CA5392：請對 P/Invokes 使用 DefaultDllImportSearchPaths 屬性

|Item|值|
|-|-|
|CheckId|CA5392|
|類別|Microsoft.Security|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:System.Runtime.InteropServices.DefaultDllImportSearchPathsAttribute>未針對[平台叫用（P/Invoke）](/dotnet/standard/native-interop/pinvoke)函數指定。

## <a name="rule-description"></a>規則描述

根據預設，使用探查的 P/Invoke 函式會 <xref:System.Runtime.InteropServices.DllImportAttribute> 有一些目錄，包括要載入之程式庫的目前工作目錄。 這可能是某些應用程式的安全性問題，因而導致 DLL 劫持。 

例如，如果與已匯入之名稱相同的惡意 DLL 放在目前的工作目錄下（依預設會進行搜尋），則可能會載入惡意的 DLL。

如需詳細資訊，請參閱[安全地載入程式庫](https://msrc-blog.microsoft.com/2014/05/13/load-library-safely/)。

## <a name="how-to-fix-violations"></a>如何修正違規

使用 <xref:System.Runtime.InteropServices.DefaultDllImportSearchPathsAttribute> 來明確指定元件或方法的 DLL 搜尋路徑。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

在下列情況下，隱藏此規則是安全的：
- 您確定載入的元件是您想要的。 例如，您的應用程式會在信任的伺服器上執行，而且您完全信任這些檔案。
- 匯入的元件是常用的系統元件（例如 user32.dll），而搜尋路徑策略會遵循[已知的 dll 機制](https://blogs.msdn.microsoft.com/larryosterman/2004/07/19/what-are-known-dlls-anyway/)。

## <a name="pseudo-code-examples"></a>虛擬程式碼範例

```csharp
using System;
using System.Runtime.InteropServices;

class ExampleClass
{
    [DllImport("The3rdAssembly.dll")]
    public static extern int MessageBox(IntPtr hWnd, String text, String caption, uint type);

    public void ExampleMethod()
    {
        MessageBox(new IntPtr(0), "Hello World!", "Hello Dialog", 0);
    }
}
```

### <a name="solution"></a>解決方案

```csharp
using System;
using System.Runtime.InteropServices;

class ExampleClass
{
    [DllImport("The3rdAssembly.dll")]
    [DefaultDllImportSearchPaths(DllImportSearchPath.UserDirectories)]
    public static extern int MessageBox(IntPtr hWnd, String text, String caption, uint type);

    public void ExampleMethod()
    {
        MessageBox(new IntPtr(0), "Hello World!", "Hello Dialog", 0);
    }
}
```
