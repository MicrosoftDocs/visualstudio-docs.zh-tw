# <a name="ca1721-property-names-should-not-match-get-methods"></a>CA1721：屬性名稱不能和其中有 get 的方法名稱相符

|||
|-|-|
|TypeName|PropertyNamesShouldNotMatchGetMethods|
|CheckId|CA1721|
|類別|Micros. 命名|
|重大變更|中斷|

## <a name="cause"></a>原因

成員的名稱以"獲取"開頭,否則與屬性的名稱匹配。 例如,包含名為「GetColor」的方法和名為"Color"的屬性的類型會導致規則衝突。
如果屬性或方法用 標記<xref:System.ObsoleteAttribute>了 ,則此規則不會觸發。

預設情況下,此規則僅檢視外部可見成員和屬性,但這是[可配置的](#configurability)。

## <a name="rule-description"></a>規則描述

"Get" 方法和屬性的名稱應該清楚區別其功能。

命名約定為面向通用語言運行時的庫提供了一個常見外觀。 這種一致性減少了學習新軟體庫所需的時間,並提高了客戶的信心,即該庫是由具有開發託管代碼的專業知識的人開發的。

## <a name="how-to-fix-violations"></a>如何修正違規

更改名稱,使其與預先設定為「Get」的方法的名稱不匹配。

## <a name="when-to-suppress-warnings"></a>何時禁止警告

請勿隱藏此規則的警告。

> [!NOTE]
> 如果「Get」方法是由實現 IExtender 提供程式介面引起的,則可能會排除此警告。

## <a name="configurability"></a>可設定性

如果從[FxCop 分析器](install-fxcop-analyzers.md)運行此規則(而不是使用舊版分析),則可以根據代碼庫的可存取性配置要運行此規則的哪些部分。 例如,要指定規則應僅針對非公共 API 曲面運行,請在專案中將以下鍵值對新增到 .editorconfig 檔案中:

```ini
dotnet_code_quality.ca1721.api_surface = private, internal
```

您可以僅為此規則、所有規則或此類別中的所有規則(命名)設定此選項。 有關詳細資訊,請參閱配置[FxCop 分析器](configure-fxcop-analyzers.md)。
