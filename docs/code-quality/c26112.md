# <a name="c26112"></a>C26112
警告 C26112：呼叫端在呼叫 \<func > 之前，無法保留任何鎖定。

 批註 `_Requires_no_locks_held_` 會強加一個前置條件，讓呼叫端在呼叫函數時，不能保留任何鎖定。 當函式無法在呼叫另一個函式之前釋放所有鎖定時，就會發出警告 C26112。

## <a name="example"></a>範例
 下列範例會產生警告 C26112，因為在鎖定的區段中呼叫 `NoLocksAllowed` 會違反 `_Requires_no_locks_held_` 前置條件。

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

_Requires_no_locks_held_

void NoLocksAllowed(DATA* p)
{
     // Lock sensitive operations here
}

void LocksHeldFunction(DATA* p)
{
    EnterCriticalSection(&p->cs);
    NoLocksAllowed(p); // Warning C26112
    LeaveCriticalSection(&p->cs);
}
```
