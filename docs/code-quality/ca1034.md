# <a name="ca1034-nested-types-should-not-be-visible"></a>CA1034：巢狀類型不應為可見

|||
|-|-|
|TypeName|NestedTypesShouldNotBeVisible|
|CheckId|CA1034|
|類別|微軟.設計|
|重大變更|中斷|

## <a name="cause"></a>原因

外部可見類型包含外部可見類型聲明。 嵌套枚舉、受保護類型和生成器模式不受此規則的約束。

## <a name="rule-description"></a>規則描述
嵌套類型是另一種類型作用域內聲明的類型。 嵌套類型可用於封裝包含類型的私有實現詳細資訊。 因為有這樣的用途，所以巢狀類型不應為外部可見的。

不要將外部可見的嵌套類型用於邏輯分組或避免名稱衝突;而是使用命名空間。

嵌套類型包括成員可訪問性的概念,一些程式師不能清楚地理解。

受保護的類型可以在子類和嵌套類型中用於預先自定義的方案。

## <a name="how-to-fix-violations"></a>如何修正違規
如果不希望嵌套類型在外部可見,則更改類型的可訪問性。 否則,從其父類型中刪除嵌套類型。 如果嵌套的目的是對嵌套類型進行分類,請使用命名空間來創建層次結構。

## <a name="when-to-suppress-warnings"></a>何時禁止警告
請勿隱藏此規則的警告。
