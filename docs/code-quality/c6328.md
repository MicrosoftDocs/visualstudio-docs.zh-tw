# <a name="c6328"></a>C6328

警告 C6328：大小不符： \<當\< \<呼叫\<函式需要類型 > 時，類型 > 傳遞做為參數編號 > >

將類型`char`的引數傳遞至 C 執行時間，以字元為`is<xxx>()`基礎的常式（ `isspace()`例如），可能會有無法預期的結果。 例如，值大於 `char` 且類型為 `0x7F` 的 SBCS 或 MBCS 單一位元組字元為負值。 如果傳遞，則編譯器可能會將值轉換成已簽署`int`或已簽署`long`的。 `char` 編譯器會對此值進行正負號擴充，但產生非預期的結果。 例如， `isspace`接受類型`int`的引數，不過，其輸入引數的有效值範圍為：

`0 <= c <= 255`，加上特殊值 `EOF`。

## <a name="example"></a>範例

根據預設，`char` 在 Visual C++ 中是帶正負號的類型，所以 char 類型之變數的值範圍是 `-128 <= c <= 127`。 因此，如果您執行下列動作， `c`將會以-37 的值延伸`int`至帶正負號的已簽署，這超出[isspace](/cpp/standard-library/locale-functions#isspace)的有效範圍：

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( c );
    // code ...
}
```

若要更正這個問題，您可以使用 `static_cast`，如下列程式碼所示：

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( static_cast<unsigned char> (c) );
    // code ...
}
```

警告 C6328 會特別存在，以攔截此 bug。 若為7位 ASCII 範圍中的字元，則不需要轉換。 該範圍外的字元可能會有無法預期的結果，例如程式錯誤和終止。
