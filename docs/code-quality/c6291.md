# <a name="c6291"></a>C6291
警告 C6291：邏輯結果的位運算：！ 的優先順序高於&#124;。 使用&#124; &#124; or （！（x &#124; y））代替

@No__t-0 運算子會產生布林值結果，而 `|` （位 or）運算子會採用兩個算術引數。 @No__t-0 運算子的優先順序也高於 `|`。

因此，偵測到下列其中一個錯誤：

- 運算式 parenthesised：

   因為 `!` 的結果是布林值（零或一個），所以嘗試測試這兩個變數是否有 bits 集，只會在右側出現最低位的測試：在 `x == 0` 和 `y == 1` 時，`((!x) | y) != (!(x | y))`。

- @No__t-0 運算子不正確，應改為 `~`：

   @No__t-0 運算子有布林值結果，但 @no__t 1 運算子具有算術結果。 即使在布林值（零或一）上操作時，這些運算子也永遠不會互換： `x == 1` 並 `y == 0` 時，`((!x) | y) != ((~x) | y)`。

- 二元運算子 `|` 不正確，應改為 `||`：

   即使 `|` 有時可以與 `||` 交換，它並不相等，因為它會強制評估運算式的右側。 此類型運算式中的某些副作用可以是 terminal： `(!p | (*p == '\0'))`，當 `p == NULL` 時，我們必須將它取值以評估運算式的其他一半。

如果 `!` 運算子位於 `|` 運算子的右邊，就不會報告這個警告，因為這種情況通常只是不正確運算子的相對無害案例。

在不檢查程式碼的情況下，很容易判斷這個問題的嚴重性。 應檢查程式碼，以確保會發生預期的測試。

此警告一律表示使用運算子或運算子優先順序時可能會造成混淆。

## <a name="example"></a>範例

下列程式碼會產生這個警告：

```cpp
void f(int x, int y )
{
  if (!x | y)
  {
    //code
  }
}
```

若要修正這個警告，請使用下列程式碼中所示的其中一個範例：

```cpp
void fC(int x, int y )
{
  /* When checking whether any bits are set in either x or y. */
  if (!(x | y))
  {
    // code
  }
  /* When checking whether bits are set in either */
  /* the complement of x or in y. */
  if ((~x) | y)
  {
    // code
  }
}

#include <windows.h>
void f(int x, BOOL y )
{
  /* When y is a Boolean or Boolean result. */
  if ((!x) || y)
  {
    // code
  }
}
```
