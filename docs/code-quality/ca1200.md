# <a name="ca1200-avoid-using-cref-tags-with-a-prefix"></a>CA1200：請避免使用具有前置詞的 cref 標記

|||
|-|-|
|CheckId|CA1200|
|類別|Microsoft. 檔|
|重大變更|不中斷|

## <a name="cause"></a>原因

XML 檔批註中的[cref](/dotnet/csharp/programming-guide/xmldoc/cref-attribute)標記會使用[前置](/dotnet/csharp/programming-guide/xmldoc/processing-the-xml-file)詞。

## <a name="rule-description"></a>規則描述

XML 檔標記中的[cref](/dotnet/csharp/programming-guide/xmldoc/cref-attribute)屬性工作表示「程式碼參考」。 它會指定標記的內部文字是程式碼項目，例如類型、方法或屬性。 請避免使用具有前置詞 `cref` 的標記，因為它會防止編譯器驗證參考。 它也會防止 Visual Studio 的整合式開發環境（IDE）在重構期間尋找和更新這些符號參考。 建議您使用沒有前置詞的完整語法來參考 cref 標記中的符號名稱。

> [!NOTE]
> 舊版分析無法使用規則 CA1200。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正此規則的違規，請移除標記中的前置詞 `cref` 。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
// Violates CA1200
/// <summary>
/// Type <see cref="T:C" /> contains method <see cref="F:C.F" />
/// </summary>
class C
{
    public void F() { }
}
```

```csharp
// Does not violate CA1200
/// <summary>
/// Type <see cref="C" /> contains method <see cref="F" />
/// </summary>
class C
{
    public void F() { }
}
```

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果程式碼參考必須使用前置詞，則可放心抑制此警告，因為編譯器不會 findable 參考的型別。 例如，如果程式碼參考參考完整架構中的特殊屬性，但檔案是針對可移植架構進行編譯，則您可以隱藏這個警告。
