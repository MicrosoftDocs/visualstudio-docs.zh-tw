# <a name="ca1828-do-not-use-countasynclongcountasync-when-anyasync-can-be-used"></a>CA1828：當可以使用 AnyAsync 時，請勿使用 CountAsync/LongCountAsync

|||
|-|-|
|TypeName|DoNotUseCountWhenAnyCanBeUsedAnalyzer|
|CheckId|CA1828|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.CountAsync%2A>或<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.LongCountAsync%2A>方法會在方法會更<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.AnyAsync%2A>有效率的地方使用。

## <a name="rule-description"></a>規則描述

此規則會旗<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.CountAsync%2A>標<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.LongCountAsync%2A>和 LINQ 方法呼叫，用來檢查集合是否至少有一個元素。 這些方法呼叫需要列舉整個集合來計算計數。 使用<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.AnyAsync%2A>方法時，相同的檢查會更快，因為它可避免列舉集合。

> [!NOTE]
> 舊版分析無法使用規則 CA1828。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.9.6 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請以<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.CountAsync%2A> <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.AnyAsync%2A>方法<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.LongCountAsync%2A>取代或方法呼叫。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System.Linq;
using System.Threading.Tasks;
using static Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions;

class C
{
    public async Task<string> M1Async(IQueryable<string> list)
        => await list.CountAsync() != 0 ? "Not empty" : "Empty";

    public async Task<string> M2Async(IQueryable<string> list)
        => await list.LongCountAsync() > 0 ? "Not empty" : "Empty";
}
```


```csharp
using System.Linq;
using System.Threading.Tasks;
using static Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions;

class C
{
    public async Task<string> M1Async(IQueryable<string> list)
        => await list.AnyAsync() ? "Not empty" : "Empty";

    public async Task<string> M2Async(IQueryable<string> list)
        => await list.AnyAsync() ? "Not empty" : "Empty";
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在違規上，然後按**Ctrl**+**。** (句點)。 當 AnyAsync （）可從顯示的選項清單中使用時，請選擇 [**不要使用 CountAsync （）] 或 [LongCountAsync （）** ]。
>
> ![CA1828 的程式碼修正-當可以使用 AnyAsync （）時，請勿使用 CountAsync （）或 LongCountAsync （）](media/ca1828-codefix.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意不必要的集合列舉計算計數的效能影響，則可以放心抑制此規則的違規。
