# <a name="ca1828-do-not-use-countasynclongcountasync-when-anyasync-can-be-used"></a>CA1828：不要在可使用 AnyAsync 時使用 CountAsync/LongCountAsync

|Item|值|
|-|-|
|CheckId|CA1828|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.CountAsync%2A> 或方法會在 <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.LongCountAsync%2A> 方法會更有效率的地方使用 <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.AnyAsync%2A> 。

## <a name="rule-description"></a>規則描述

此規則會旗標 <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.CountAsync%2A> 和 <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.LongCountAsync%2A> LINQ 方法呼叫，用來檢查集合是否至少有一個元素。 這些方法呼叫需要列舉整個集合來計算計數。 使用方法時，相同的檢查會更快， <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.AnyAsync%2A> 因為它可避免列舉集合。

> [!NOTE]
> 舊版分析無法使用規則 CA1828。 它最初是在 [FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) 版本2.9.6 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請以 <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.CountAsync%2A> 方法取代或 <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.LongCountAsync%2A> 方法呼叫 <xref:Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions.AnyAsync%2A> 。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
using System.Linq;
using System.Threading.Tasks;
using static Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions;

class C
{
    public async Task<string> M1Async(IQueryable<string> list)
        => await list.CountAsync() != 0 ? "Not empty" : "Empty";

    public async Task<string> M2Async(IQueryable<string> list)
        => await list.LongCountAsync() > 0 ? "Not empty" : "Empty";
}
```

```csharp
using System.Linq;
using System.Threading.Tasks;
using static Microsoft.EntityFrameworkCore.EntityFrameworkQueryableExtensions;

class C
{
    public async Task<string> M1Async(IQueryable<string> list)
        => await list.AnyAsync() ? "Not empty" : "Empty";

    public async Task<string> M2Async(IQueryable<string> list)
        => await list.AnyAsync() ? "Not empty" : "Empty";
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在違規上，然後按**Ctrl** + **。** (句點)。 從顯示的選項清單中 **使用 ( AnyAsync 時，請選擇 [不要使用 CountAsync ( # A1 或 LongCountAsync ( # A3** ]。
>

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意不必要的集合列舉計算計數的效能影響，則可以放心抑制此規則的違規。
