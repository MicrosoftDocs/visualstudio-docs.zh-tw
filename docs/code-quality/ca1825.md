# <a name="ca1825-avoid-zero-length-array-allocations"></a>CA1825：請避免長度為零的陣列配置

|Item|值|
|-|-|
|CheckId|CA1825|
|類別|Microsoft。效能|
|重大變更|不中斷|

## <a name="cause"></a>原因

<xref:System.Array>系統會配置不含任何元素的空白。

## <a name="rule-description"></a>規則描述

初始化長度為零的陣列會導致不必要的記憶體配置。 相反地，請藉由呼叫方法，來使用靜態配置的空陣列實例 <xref:System.Array.Empty%2A?displayProperty=nameWithType> 。 記憶體配置會在此方法的所有調用之間共用。

> [!NOTE]
> 舊版分析無法使用規則 CA1825。 它最初是在[FxCop 分析器](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)版本2.6.0 中引進。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正違規，請使用對的呼叫來取代零長度的陣列配置 <xref:System.Array.Empty%2A?displayProperty=nameWithType> 。 例如，下列兩個程式碼片段會顯示規則違規，以及如何修正此問題：

```csharp
class C
{
    public void M1()
    {
        // Violates rule CA1825.
        var a = new int[0];
    }
}
```

```csharp
class C
{
    public void M1()
    {
        // Resolves rule CA1825 violation.
        var a = System.Array.Empty<int>();
    }
}
```

> [!TIP]
> 程式碼修正適用于 Visual Studio 中的這項規則。 若要使用它，請將游標放在陣列配置上，然後按**Ctrl** + **。** (句點)。 從顯示的選項清單中選擇 [**使用 Array. 空白**]。
>
> ![CA1825 的程式碼修正-使用陣列空白](media/ca1825-codefix.png)

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果您不在意額外的記憶體配置，則可以放心抑制此規則的違規。
