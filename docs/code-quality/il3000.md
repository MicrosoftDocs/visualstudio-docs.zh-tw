# <a name="il3000-avoid-using-accessing-assembly-file-path-when-publishing-as-a-single-file"></a>IL3000：以單一檔案發行時，避免使用存取元件檔案路徑

## <a name="cause"></a>原因

例如，當您將專案中的 PublishSingleFile 屬性設定為 true 時，若要 (發行成單一檔案) ， `Assembly.Location` 針對內嵌在單一檔案組合內的元件呼叫屬性，一律會傳回空字串。

## <a name="how-to-fix-violations"></a>如何修正違規

如果應用程式只需要包含單一檔案配套的目錄，請考慮改為使用 `AppContext.BaseDirectory` 屬性。 否則，請考慮完全移除呼叫。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

如果要存取的元件絕對不是在單一檔案配套中，就適合解除此警告的回應。 如果元件是以動態方式從檔案路徑載入，就可能發生這種情況。
