# <a name="ca1052-static-holder-types-should-be-static-or-notinheritable"></a>CA1052：靜態預留位置類型應該是靜態或 NotInheritable

|Item|值|
|-|-|
|CheckId|CA1052|
|類別|Microsoft. 設計|
|重大變更|中斷|

## <a name="cause"></a>原因

非抽象型別只包含靜態成員 () 可能預設的函式，而且不是使用 [static](/dotnet/csharp/language-reference/keywords/static) 或 [Shared](/dotnet/visual-basic/language-reference/modifiers/shared) 修飾詞來宣告。

根據預設，此規則只會查看外部可見的類型，但這是 [可](#configurability)設定的。

## <a name="rule-description"></a>規則描述

規則 CA1052 假設只包含靜態成員的型別不會被繼承，因為型別不提供任何可在衍生類型中覆寫的功能。 不應繼承的類型應該以 `static` c # 中的修飾詞標記，以禁止將其作為基底類型使用。 此外，也應該移除其預設的函式。 在 Visual Basic 中，類別應轉換成 [模組](/dotnet/visual-basic/language-reference/statements/module-statement)。

如果是具有基類的抽象類別或類別，則不會引發此規則。 不過，對於支援空介面的類別，則會引發此規則。

> [!NOTE]
> 在此規則的最新分析器執行中，它也包含 [規則 CA1053](../code-quality/ca1053.md)的功能。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正此規則的違規情形，請將類型標記為 `static` ，並移除預設的函式 (c # ) ，或將它轉換成模組 (Visual Basic) 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

在下列情況下，您可以隱藏違規：

- 此類型是設計成要繼承的。 缺少 `static` 修飾詞會建議類型可做為基底類型。
- 型別會當做型別引數使用。 靜態類型不能當做型別引數使用。

## <a name="configurability"></a>性

您可以根據其存取範圍，設定程式碼基底的哪些部分來執行此規則。 例如，若要指定規則只能針對非公用 API 介面執行，請將下列機碼值組新增至專案中的 EditorConfig 檔案：

```ini
dotnet_code_quality.ca1052.api_surface = private, internal
```

您可以只針對此規則、所有規則或此類別中的所有規則設定此選項， (設計) 。 如需詳細資訊，請參閱 [設定 .net 程式碼品質分析器](configure-fxcop-analyzers.md)。

## <a name="fix-with-the-static-modifier"></a>使用 static 修飾詞修正

下列範例示範如何使用 c # 中的修飾詞來標記類型，以修正此規則的違規 `static` ：

```csharp
public static class StaticMembers
{
    public static int SomeProperty { get; set; }
    public static void SomeMethod() { }
}
```
