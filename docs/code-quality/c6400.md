# <a name="c6400"></a>C6400
警告 C6400：使用 @no__t 0function 的名稱 > 來執行不區分大小寫的比較與常數位串 \<string name >。 在非英文地區設定中產生未預期的結果

 此警告表示會以與地區設定相關的方式來執行與常數位串的區分大小寫比較，但明顯地說，與地區設定無關的比較是有意的。

 這項缺陷的一般結果是非英文說話地區設定中不正確的行為。 例如，在土耳其文中，".gif" 不會符合 "。GIF ";在越南文中，"LogIn" 不會符合 "LOGIN"。

 通常應該使用 `CompareString` 函數來執行字串比較。 若要在 Windows XP 上執行與地區設定無關的比較，第一個參數應該是常數 `LOCALE_INVARIANT`。

## <a name="example"></a>範例
 下列程式碼會產生這個警告：

```
#include <windows.h>
int f(char *ext)
{
  // code...
  return (lstrcmpi(ext, TEXT("gif")) == 0);
}
```

 若要修正這個警告，請執行與地區設定無關的測試，讓 char * ext 符合「gif」是否忽略大寫/小寫的差異，請使用下列程式碼：

```
#include <windows.h>
int f(char *ext)
{
  // code...
  return (CompareString(
                        LOCALE_INVARIANT,
                        NORM_IGNORECASE,
                        ext,
                        -1,
                        TEXT ("gif"),
                        -1) == CSTR_EQUAL);
}
```
