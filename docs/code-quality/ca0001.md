# <a name="ca0001"></a>CA0001

在 managed 程式碼分析工具中引發例外狀況，但未指出預期的錯誤情況。

在下列情況下會產生此錯誤：

- 自訂規則中的瑕疵

     在此情況下，錯誤會報告規則和目標。 範例錯誤看起來會像下面這樣：

     內部錯誤 CA0001：規則 = Microsoft。使用方式 # CA2214，Target = B.。ctor （）：物件參考未設定為物件的實例。

     如需詳細的診斷資訊，請參閱 \obj 資料夾中的 CodeAnalysisReport。 此報表會列出例外狀況類型、堆疊、類型、訊息，以及所有內部例外狀況的堆疊。 先前例外狀況的 CodeAnalysisReport 範例報表看起來類似下列內容：

     ```xml
     <Exception Keyword="CA0001" Kind="Rule" TypeName="DoNotCallOverridableMethodsInConstructors" Category="Microsoft.Usage" CheckId="CA2214" Target="B..ctor()">
       <Type>System.NullReferenceException</Type>
       <ExceptionMessage>Object reference not set to an instance of an object.</ExceptionMessage>
       <StackTrace>   at Microsoft.FxCop.Rules.Usage.DoNotCallOverridableMethodsInConstructors.CheckCallees(Method method, Boolean isCallVirt) in d:\rules\DoNotCallOverridableMethodsInConstructors.cs:line 107 at Microsoft.FxCop.Rules.Usage.DoNotCallOverridableMethodsInConstructors.CheckCallees(Method method, Boolean isCallVirt) in d:\rules\DoNotCallOverridableMethodsInConstructors.cs:line 128 at Microsoft.FxCop.Rules.Usage.DoNotCallOverridableMethodsInConstructors.Check(Member member) in d:\rules\DoNotCallOverridableMethodsInConstructors.cs:line 58 at Microsoft.FxCop.Engines.Introspection.AnalysisVisitor.CheckMember(Member member, NodeBase target) in d:\Engines\Introspection\AnalysisVisitor.cs:line 743</StackTrace>
     </Exception>
     ```

     在上一個報告中，DoNotCallOverridableMethodsInConstructors.cs 檔案的第107行發生例外狀況。 所有其他堆疊追蹤都會顯示導致例外狀況的執行路徑。

- Managed 程式碼分析工具中有未知的瑕疵

     在此情況下，錯誤會出現在 CodeAnalysisReport 中，但不含 Rule、TypeName 或 Category 屬性，而且主控台上會顯示下列訊息：

     內部錯誤 CA0001：物件參考未設定為物件的實例。

     請嘗試重寫回報錯誤的行，或這一行周圍的幾行程式碼。 如果無法運作，請聯絡 Microsoft 產品支援服務。
