# <a name="ca1412-mark-comsource-interfaces-as-idispatch"></a>CA1412:ComSource 介面必須標記為 IDispatch

|||
|-|-|
|CheckId|CA1412|
|類別|Microsoft. 互通性|
|重大變更|中斷|

## <a name="cause"></a>原因

類型是以 <xref:System.Runtime.InteropServices.ComSourceInterfacesAttribute> 屬性標記，而且至少有一個指定的介面未標記為 <xref:System.Runtime.InteropServices.InterfaceTypeAttribute> 屬性設定為 `InterfaceIsDispatch` 值。

## <a name="rule-description"></a>規則描述

<xref:System.Runtime.InteropServices.ComSourceInterfacesAttribute>是用來識別類別公開給元件物件模型（COM）用戶端的事件介面。 這些介面必須公開為 `InterfaceIsIDispatch` ，才能讓 Visual Basic 6 COM 用戶端接收事件通知。 根據預設，如果介面未以屬性標示，則 <xref:System.Runtime.InteropServices.InterfaceTypeAttribute> 會公開為雙重介面。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正此規則的違規，請新增或修改 <xref:System.Runtime.InteropServices.InterfaceTypeAttribute> 屬性，使其值設定為 cominterfacetype.interfaceisidispatch，適用于以屬性指定的所有介面 <xref:System.Runtime.InteropServices.ComSourceInterfacesAttribute> 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

請勿隱藏此規則的警告。
