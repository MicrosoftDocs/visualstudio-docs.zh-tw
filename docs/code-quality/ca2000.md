# <a name="ca2000-dispose-objects-before-losing-scope"></a>CA2000:必須在超出範圍前處置物件

|Item|值|
|-|-|
|CheckId|CA2000|
|類別|Microsoft 可靠性|
|重大變更|非中斷|

## <a name="cause"></a>原因

<xref:System.IDisposable>系統會建立類型的本機物件，但物件不會在物件的所有參考都超出範圍之前處置。

根據預設，此規則會分析整個程式碼基底，但這是 [可](#configurability)設定的。

## <a name="rule-description"></a>規則描述

如果可處置物件在所有的參考都超出範圍之前未明確處置，則當垃圾收集行程執行物件的完成項時，將會在某些不定時間處置物件。 因為可能會發生例外狀況事件而導致物件的完成項無法執行，所以應改為明確地處置物件。

### <a name="special-cases"></a>特殊案例

即使未處置物件，仍不會針對下列類型的本機物件引發規則 CA2000：

- <xref:System.IO.Stream?displayProperty=nameWithType>
- <xref:System.IO.StringReader?displayProperty=nameWithType>
- <xref:System.IO.TextReader?displayProperty=nameWithType>
- <xref:System.IO.TextWriter?displayProperty=nameWithType>
- <xref:System.Resources.IResourceReader?displayProperty=nameWithType>

將其中一個類型的物件傳遞至函式，然後將其指派給欄位，表示會將 *處置擁有權傳送* 至新建立的型別。 也就是說，新建造的型別現在負責處置物件。 如果您的程式碼將其中一種類型的物件傳遞至函式，即使在物件的所有參考都超出範圍之前處置物件，也不會違反規則 CA2000。

## <a name="how-to-fix-violations"></a>如何修正違規

若要修正此規則的違規情形，請 <xref:System.IDisposable.Dispose%2A> 在物件的所有參考都超出範圍之前呼叫物件。

您可以使用 Visual Basic) 中的[ `using` 語句](/dotnet/csharp/language-reference/keywords/using-statement) ([`Using`](/dotnet/visual-basic/language-reference/statements/using-statement) 來包裝執行的物件 <xref:System.IDisposable> 。 以這種方式包裝的物件會在區塊結尾自動處置 `using` 。 但是，下列情況不應或無法使用語句來處理 `using` ：

- 若要傳回可處置的物件，物件必須在 `try/finally` 區塊外的區塊中進行 `using` 。

- 請勿初始化語句的函式中可處置物件的成員 `using` 。

- 只有一個例外狀況處理常式所保護的函式會在 [ `using` 語句](/dotnet/csharp/language-reference/language-specification/statements#the-using-statement)的取得部分中進行嵌套時，外部函式中的失敗可能會導致無法關閉嵌套函式所建立的物件。 在下列範例中，此函式中的失敗 <xref:System.IO.StreamReader> 可能會導致 <xref:System.IO.FileStream> 物件永遠不會關閉。 在此情況下，CA2000 會旗標規則違規。

   ```csharp
   using (StreamReader sr = new StreamReader(new FileStream("C:\myfile.txt", FileMode.Create)))
   { ... }
   ```

- 動態物件應使用陰影物件來執行物件的處置模式 <xref:System.IDisposable> 。

## <a name="when-to-suppress-warnings"></a>隱藏警告的時機

除非下列情況，否則請勿隱藏此規則的警告：

- 您已在呼叫的物件上呼叫方法 `Dispose` ，例如 <xref:System.IO.Stream.Close%2A>
- 引發警告的方法會傳回 <xref:System.IDisposable> 包裝物件的物件。
- 配置的方法沒有處置擁有權;也就是說，處置物件的責任會轉移至在方法中建立並傳回給呼叫端的另一個物件或包裝函式

## <a name="configurability"></a>性

此規則具有下列可設定的選項。

### <a name="excluded-symbol-names"></a>排除的符號名稱

您可以設定程式碼基底的哪些部分排除在分析之外。 例如，若要指定規則不應該在名為的型別內的任何程式碼上執行 `MyType` ，請將下列機碼值組新增至專案中的 editorconfig 檔案：

```ini
dotnet_code_quality.CA2000.excluded_symbol_names = MyType
```

選項值中允許的符號名稱格式 (以 ' | ' 分隔) ：
- 符號名稱僅 (包含名稱的所有符號，不論包含的類型或命名空間為何) 
- 符號 [檔識別碼格式](https://github.com/dotnet/csharplang/blob/master/spec/documentation-comments.md#id-string-format)的完整名稱。 每個符號名稱都需要符號類型前置詞，例如 "M：" 前置詞代表方法、"T：" 前置詞代表類型、"N：" 前置詞（命名空間）等等。
- `.ctor`針對函式和靜態函式 `.cctor`

範例：

| 選項值 | 摘要 |
| --- | --- |
|`dotnet_code_quality.CA2000.excluded_symbol_names = MyType` | 符合編譯中名稱為 ' MyType ' 的所有符號
|`dotnet_code_quality.CA2000.excluded_symbol_names = MyType1|MyType2` | 符合編譯中名稱為 ' MyType1 ' 或 ' MyType2 ' 的所有符號
|`dotnet_code_quality.CA2000.excluded_symbol_names = M:NS.MyType.MyMethod(ParamType)` | 符合特定的方法 ' MyMethod ' 與指定的完整簽章
|`dotnet_code_quality.CA2000.excluded_symbol_names = M:NS1.MyType1.MyMethod1(ParamType)|M:NS2.MyType2.MyMethod2(ParamType)` | 符合特定的方法 ' MyMethod1 ' 和 ' MyMethod2 ' 與個別的完整簽章

您可以只針對此規則、所有規則或此類別中的所有規則設定這些選項， (設計) 。 如需詳細資訊，請參閱 [設定 .net 程式碼品質分析器](configure-fxcop-analyzers.md)。

## <a name="example"></a>範例

如果您要執行的方法會傳回可處置的物件，請在沒有 catch 區塊的情況下使用 try/finally 區塊，以確保物件已處置。 藉由使用 try/finally 區塊，您可允許在錯誤點引發例外狀況，並確保物件已處置。

在 OpenPort1 方法中，開啟 ISerializable 物件 SerialPort 或 SomeMethod 呼叫的呼叫可能會失敗。 這項實行會引發 CA2000 警告。

在 OpenPort2 方法中，會宣告兩個 SerialPort 物件，並將其設定為 null：

- `tempPort`，用來測試方法作業是否成功。

- `port`，用於方法的傳回值。

`tempPort`會在區塊中建立和開啟 `try` ，而任何其他必要的工作則會在相同的 `try` 區塊中執行。 在區塊的結尾，會將 `try` 開啟的埠指派給 `port` 將傳回的物件，並將 `tempPort` 物件設定為 `null` 。

`finally`區塊會檢查的值 `tempPort` 。 如果它不是 null，則方法中的作業已失敗，並已 `tempPort` 關閉以確定已釋放任何資源。 如果方法的作業成功，傳回的埠物件將會包含開啟的 SerialPort 物件，如果作業失敗，則會是 null。

```csharp
public SerialPort OpenPort1(string portName)
{
   SerialPort port = new SerialPort(portName);
   port.Open();  //CA2000 fires because this might throw
   SomeMethod(); //Other method operations can fail
   return port;
}

public SerialPort OpenPort2(string portName)
{
   SerialPort tempPort = null;
   SerialPort port = null;
   try
   {
      tempPort = new SerialPort(portName);
      tempPort.Open();
      SomeMethod();
      //Add any other methods above this line
      port = tempPort;
      tempPort = null;

   }
   finally
   {
      if (tempPort != null)
      {
         tempPort.Close();
      }
   }
   return port;
}
```

```vb
Public Function OpenPort1(ByVal PortName As String) As SerialPort

   Dim port As New SerialPort(PortName)
   port.Open()    'CA2000 fires because this might throw
   SomeMethod()   'Other method operations can fail
   Return port

End Function

Public Function OpenPort2(ByVal PortName As String) As SerialPort

   Dim tempPort As SerialPort = Nothing
   Dim port As SerialPort = Nothing

   Try
      tempPort = New SerialPort(PortName)
      tempPort.Open()
      SomeMethod()
      'Add any other methods above this line
      port = tempPort
      tempPort = Nothing

   Finally
      If Not tempPort Is Nothing Then
         tempPort.Close()
      End If

   End Try

   Return port

End Function
```
