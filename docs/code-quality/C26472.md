# <a name="c26472-no_casts_for_arithmetic_conversion"></a>C26472 NO_CASTS_FOR_ARITHMETIC_CONVERSION

「不要使用 static_cast 進行算術轉換。 使用大括弧初始化、gsl：： narrow_cast 或 gsl：：窄。

核心指導方針：類型1：避免轉換**C++**

此規則有助於找出使用靜態轉換在整數類型之間進行轉換的位置。 這些轉換並不安全，因為如果發生任何資料遺失，編譯器不會發出警告。 大括弧初始化運算式適用于使用常數的情況，而且需要編譯器錯誤。 方針支援程式庫也有公用程式，可協助您清楚地描述意圖：

- gsl：：窄幅可確保不失真的轉換，並會導致執行時間損毀（如果不可行）。
- gsl：： narrow_cast 清楚指出轉換可能會遺失資料，而且是可接受的。

## <a name="remarks"></a>備註

- 此規則只會針對 static_casts 執行。 通常不建議使用 C 樣式的轉換。

## <a name="example"></a>範例

未處理的非預期資料

```cpp
rgb from_24bit(std::uint32_t v) noexcept {
    return {
        static_cast<std::uint8_t>(v >> 16),         // C26472, what if top byte is non-zero?
        static_cast<std::uint8_t>((v >> 8) & 0xFF), // C26472
        static_cast<std::uint8_t>(v & 0xFF)         // C26472
    };
}
```

未處理的非預期資料-更安全的版本

```cpp
rgb from_24bit(std::uint32_t v) noexcept {
    return {
        gsl::narrow<std::uint8_t>(v >> 16),
        gsl::narrow_cast<std::uint8_t>((v >> 8) & 0xFF),
        gsl::narrow_cast<std::uint8_t>(v & 0xFF)
    };
}
```
