# <a name="c6067"></a>C6067
警告 C6067： 參數\<數字 > 呼叫中\<函式 > 必須是字串的位址  
  
 這個警告表示的格式規範與函式參數不符。 即使警告會建議使用字串的位址，您必須檢查函式需要修正問題之前參數的型別。 例如，`%s`規格`printf`需要字串引數，但`%s`規格`scanf`需要字串的位址。  
  
 這項缺失很可能會導致當機或某種形式損毀。  
  
## <a name="example"></a>範例  
 下列程式碼會產生這個警告，因為整數傳遞而非字串：  
  
```  
#include <stdio.h>  
void f_defective( )  
{    
  char *str = "Hello, World!";  
  printf("String:\n %s",1); // warning  
  // code ...  
}  
```  
  
 若要修正警告，請將字串傳遞為參數，以`printf`如下列程式碼所示：  
  
```  
#include <stdio.h>  
void f_corrected( )  
{    
  char *str = "Hello, World!";  
  printf("String:\n %s",str);   
  // code ...  
}  
```  
  
 下列程式碼會產生這個警告，因為指定不正確的層級的間接取值時將參數傳遞至緩衝`scanf`:  
  
```  
#include <stdio.h>  
void h_defective( )  
{  
  int retval;  
  char* buffer = new char(20);  
  if (  buffer )  
  {  
    retval = scanf("%s", &buffer); // warning C6067  
    // code...  
    delete buffer ;  
  }  
}  
```  
  
 若要更正上述警告，請傳遞正確的參數如下列程式碼所示：  
  
```  
#include <stdio.h>  
void h_corrected( )  
{  
  int retval;  
  char* buffer = new char(20);  
  if ( buffer )  
  {  
    retval = scanf("%s", buffer);  
    // code...  
    delete buffer;  
  }  
}  
```  
  
 下列程式碼會使用安全字串操作函數，若要更正這個警告：  
  
```  
#include <stdio.h>  
void f_safe( )  
{  
  char buff[20];  
  int retVal;  
  
  sprintf_s( buff, 20, "%s %s", "Hello", "World!" );  
  printf_s( "String:\n   %s  %s", "Hello", "World!" );  
  retVal = scanf_s("%s", buff, 20);  
}  
```
